## 最不像哲学宝典的 5 处（按严重程度）

**1.【最严重】34/36 道推理选择题的【正确理由】用同一句式开头：`"XX模 第N题 选 X。…"`**  
这是答案册外审引用句式，不是学生宝典讲题口吻。哲学宝典本体的"解析/答"段落是变化着开头的（"先看材料……" / "本题的关键在于……" / "答案 X。理由：……"），不会 34 道题机械重复同一模板。例：p20 朝阳二模第7题、p17 顺义二模第7题、p18 丰台一模第11题、p19 海淀二模第5题……全部以"XX模 第N题 选 X"为引。这是 V17 最显眼的"规则堆叠"指纹。

**2.【次严重】每道选择题被 7 个方括号标签框住，其中【完整题干】【完整选项】【材料触发点】明显是后台 QA schema 的命名，不是学生宝典的命名**  
"完整题干""完整选项"是 V12 patch 为了通过"36/36 必须切分"门禁拆出来的——学生不需要被告知"这是完整的题目"，正常宝典只印题目和 A/B/C/D 即可，最多冠以"典例"或"例题"。"材料触发点"是内容工程师的术语，学生看到只会困惑"谁触发了什么"。整本推理册 36 道题、整本思维册 59 处都用同一套 7 标签套模板（p5、p20、p39、p50 截屏完全同构），把学生宝典做成了后台模板填空表。

**3.【为什么能想到】+【正确理由】+【诱人错项和错因】内容高度重复，是为了凑 70 字下限做的"密度灌水"**  
以 p20 朝阳二模第7题为例：【为什么能想到】已经把 A/B/C/D 四项都分析了一遍；【正确理由】又重新分析了一遍 D 正确以及 A/B/C 越界；【诱人错项和错因】再来第三遍 A/B/C 各自的错因。同一信息以三种框架被讲三次，读起来不是层层递进，而是为了让 V17 患指标"平均 103.1 / 174.3 字"达标的人工膨胀。学生看完一题要扫读三段重复内容，疲劳但学不到新东西——哲学宝典从不三复述。

**4.两本书的末页都"戛然而止"——推理宝典 p56、思维宝典 p35 都以最后一题的【答案落点】结尾，下方约 60–70% 空白，没有"全书完"、没有方法卡总结、没有"祝考场顺利"、没有附录**  
这是数据库导出的特征签名，不是印刷品的结尾。哲学宝典本体（用户给出的 5.2 双终极融合版）按用户表述是会收束的——这里直接以"AI 若说'前后自相矛盾'是对的；但如果说这是违反'确定性要求'，就把同一律和矛盾律混了。"这种**告诉学生 AI 评卷可能怎样错**的句子作全书最后一段（推理册 p56），属于把 QA 内部对话当成全书结尾，完全错位。

**5.推理宝典 TOC 是"标签树"，不是"教学进度表"**  
p3–p4 看：H1"三、三段论、性质判断周延与换质位"下挂 9 条 H2，全部以"三段论"开头：`三段论 / 演绎推理中的三段论 / 三段论补大前提 / 三段论四概念错误 / 三段论大项不当扩大与谬误名称纠错 / 三段论小项不当扩大 / 三段论构造 / 三段论有效结构 / 三段论有效性与前提真实性`。"一、充分条件……"下面 4 条 H2 几乎是同义改写（"假言推理 / 假言判断构造 / 假言判断真假辨析 / 假言推理前提真假辨析"）。这是题目元数据按谬误类型自动聚类生成的目录，不是教师为学生设计的章节进度。哲学宝典 TOC 是叙事友好的（"物质和意识的关系 / 实践第一"），不会出现 9 条都以"三段论"开头的 H2 列表。

## 最像哲学宝典的 5 处

**1.主观题"为什么能想到"段落里，确实能看到"先抓材料动作 → 自然触发知识"的真实笔法**  
例：思维册 p4 顺义一模 19(2)（客观性），先讲"研发团队进入真实养老场景、面对老人居家照护的真实困难"，再说"产品功能被真实对象和真实需求约束，而不是从概念想象中推出，所以触发客观性"——这条"动作 → 触发"链是干净的，不是先背"三性"再贴材料。推理册 p17 顺义二模第7题"先抓住最后那句'只有……才……'"也属此类。这部分确实是用户期望的"哲学宝典味"。

**2.36 道选择题的【完整题干】+【完整选项】基本可读，原题文字大体完整，没有出现历史版本里的题干截断**  
学生能直接做题，这是相对 V8/V9 的实质进步。

**3.两本 PDF 的版式骨架（A4、左斜浅色"飞哥正志讲堂"水印、居中页脚"— N —"、TOC 点线 + 右对齐页码、首页无页眉页脚、黑体标题/楷体署名/正文 YaHei、四标签色 21574C）确实已与用户描述的哲学宝典外壳对齐**  
封面、目录、正文三段字体体系清楚，看起来不像速成的 Markdown 转 PDF。

**4.思维册章前 5 步导引（材料怎么看 / 该写哪个思维方法 / 为什么触发 / 答案句怎么落 / 易错项怎么避）方向上是对的，写法上也尽量在用"卷面语言"而不是术语**  
比如 p4 客观性"答案不能只背'三性'，要把三性分别接到材料动作上"、p13 整体性"看到多个主体不要只写'整体性'，还应写清各主体的具体作用"——是教师讲堂的口吻。

**5.推理册主观题的【答案落点】大多是学生可以直接照抄到卷面上的成句**  
例 p17 顺义二模 7"主张2不符合逻辑。题干只能说明引入智能垃圾箱有助于提升垃圾分类效率，不能由效率提升这一结果倒推出'必须引入智能垃圾箱'这一唯一条件；该主张把后件真误当成前件必真，属于充分条件假言推理方向倒置。"——这一条直接放到答题卡上就是满分句。这是宝典该有的"落点"。

## 建议 Codex 回源核验的点（只列点，不直接改正文）

1. **36 道选择题的答案与北京原卷官方答案逐题对一遍。** V17 patch 只声明"加厚正文密度"，没有声明"回源核验答案"。"34/36 同模板"的事实意味着这些理由是从模板生成的，若答案本身错一两道，会带着错误模板一起印发。重点回源：2024 顺义二模 第7题（选 C，回查是否 C 而非 B/D）、2026 丰台二模 第8题（选 C，脑机接口）、2024 朝阳二模 第7题（选 D，自媒体三段论）、2025 顺义一模 第7题（选 A，谬误名称纠错——manifest 自称这是"V8 补入的硬缺口"）。

5. **【完整题干】是否被截断在题干尾词上而非真正的设问处。** 例如 2024 海淀一模 第6题（围棋）题干末尾以"围棋实战中"结束、2025 丰台期末 第9题题干以"据此，下列说法逻辑正确的是（ ）"结尾——若原卷在此后还有设问语，需补；若原卷就到此为止，需在 QA 里标注"原卷无后续设问"以避免外审再次质疑。

6. **【材料触发点】【完整题干】【完整选项】这三个标签名本身。** 请用户/底稿核：哲学宝典本体是否真的用"完整题干""材料触发点"这种工程化命名，还是用"典例""例题""设问""解析"。若哲学宝典使用后者，V12 把"【完整题干与选项】"再拆成两个"完整X"标签的方向就是错的，需要整体改名。

9. **推理册 TOC 中"三段论""充分条件假言"类同义 H2 是否应合并或重命名为教学进度名（如"三段论的形式有效性""三段论的项扩大错误"），而不是按谬误类型平铺**——这是把目录从"标签树"改回"教学路径"的关键回源决定，需用户或哲学宝典本体作为基准。
