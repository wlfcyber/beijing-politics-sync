# 飞哥选必三逻辑与思维硬性要求记事本

## 本 run 硬规则

- 用户已经否定上一版“哲学对齐”质量：两本都没完全对齐哲学宝典。
- 四标题只是最低格式，不是完成标准。
- 必须以哲学宝典 DOCX 本体为 benchmark，不能只引用旧 `PHILOSOPHY_STYLE_LOCK.md`。
- 思维宝典要像哲学宝典那样按方法节点挂题、讲触发、给卷面句，不做研究稿、速查表或口令卡。
- 推理宝典也要像哲学宝典那样按推理形式节点挂题，讲“为什么由题干想到该推理形式”和“答案如何落”，不能只写公式索引。
- 新版必须重新做 Word/PDF：封面、前言/说明、带页码目录、A4 页面、页边距、标题层级、正文密度、样页视觉 QA。
- 没有通过 `PHILOSOPHY_ALIGNMENT_AUDIT`、真实 GPT Pro / Claude 对齐审核、Governor/Confucius 前，不得称最终版。

## 2026-05-26 用户内容审计追加硬规则

- 用户明确要求：先不要管格式小问题，优先认真审核内容上有没有错判、错挂。
- 若正式细则或 source-lock 锁定多个思维方法，正文只把题挂在一个节点，而把另一个节点只写进答案尾句或说明句，这不是单纯格式问题，而是触发链漏挂。
- 这种题必须按哲学宝典方式拆成多个独立节点条目，每个条目都重写 `材料触发点 / 设问 / 为什么能想到 / 答案落点`，说明该节点为什么能从材料中被想到。
- `2024西城一模 19(5)` 已作为校准例：应挂 `超前思维`，不得再放回科学思维三性。

## 2026-05-26 用户框架纠偏硬规则

- 用户明确指出：`探索性与方法更新`、`整体安排` 根本不是其思维框架中科学思维的节点；不得继续作为学生正文二级标题。
- 科学思维只能按用户 PDF 框架保留三点：`追求认识的客观性`、`结果具有预见性`、`结果具有可检验性`。
- 辩证思维必须严格按用户 PDF 框架重排：先 `整体性`、`动态性`，再进入 `分析与综合`、`矛盾分析法`、`量变与质变`、`适度原则`、`辩证否定`、`认识发展历程` 等节点；不得自创 `整体性与系统观念`、`动态性、质量互变与发展过程` 这类合并标题。

- 后续验收先查目录和 H2 标题是否完全服从上述框架；若存在自创节点或错误合并，直接判 `FRAMEWORK_MISMATCH_HARD_FAIL`，不得发 Word/PDF 给用户审核。
