# DEVELOPMENT_PLAN

1. 读取项目门禁、选必三 skill、硬规则记事本和当前 run 控制文件。
2. 从用户推理框架与旧定位表生成题目定位清单，旧表只作为索引。
3. 对每个唯一题回到桌面原卷、细则、教师版答案页重新抽取并清理：
   - 选择题：完整题干、A-D 或 ①②③④、答案表；
   - 主观题：完整材料/设问、原答案/评分细则中推理规则。
4. 生成两个分离产物：
   - 干净正文：无路径、无 OCR/debug/status；
   - 审计 ledger：路径、抽取方式、证据状态、问题提示。
5. 做硬样本和全量 QA：充分条件、必要条件、三段论、类比、归纳、概念/定义、逻辑规律；另查完整选项、答案可见、正文污染词。
6. 生成桌面短路径 Markdown/DOCX 和审计报告。
