# DEVELOPMENT_PLAN

1. 读取项目三层 SOP、选必三 skill、硬性记事本、V87 撤回记录。
2. 把 V17 推理宝典作为学生化主体，抽取 DOCX 文本并检查：选择题数量、正确理由、诱人错项和错因、污染词、目录/域风险。
3. 用 V87 回源表定位高风险答案源修正项，逐项核对 V17 是否已正确；若不正确，只做定点修补。
4. 生成新的 Markdown 和 DOCX 桌面交付版。若沿用 V17 PDF 且正文无变化，复制 PDF；若正文改动，则只交 DOCX/MD 并明确 PDF pending。
5. 跑文本门禁：禁止 `参考答案`、`题号 | 答案 |`、`评标`、`评分标准`、路径/OCR/debug；选择题必须有答案、正确理由、诱人错项和错因。
6. 写自查验收报告，更新 PROGRESS 与 GOVERNOR。

## Hard Gates

- 若选择题 `正确理由` 或 `诱人错项和错因` 为 0，直接 HARD_FAIL。
- 若正文像原题摘录包或审计表，直接 HARD_FAIL。
- 若 V17 与 V87 源修正项冲突，必须回源修补后再交。
