# 框架 v5 自我评估 — 主干性 + 涵盖性

**评估日期**：2026-05-05
**评估对象**：FRAMEWORK_v5_FINAL.md（24 页 PDF / 70 KB docx）+ SITUATION_BANK_v3.md（14 页 PDF / 53 KB docx）
**评估角度**：①主干性（学生能不能秒抓出最重要的东西）②涵盖性（任何一道新题学生能不能用框架归位）

---

## 1. 涵盖性评估

### 实际命中
- 一模/二模主观题：53 道命中（覆盖率 ~93%）
- 选择题：45 直接命中 + 17 类陷阱模式涵盖了 100+ 类型
- 12 项制度：v5 第八部分全部就位（民事法律关系三要素 / AI 主体资格 / 限制行为人合同 / 格式条款 / 要约承诺 / 物业不可抗力 / 诉讼时效 / 好意同乘 / 遗产管理人 / 商标抢注 / 平等就业 / 竞业限制 / 集体合同 / 起诉状 / 代理人称谓 / 司法建议）

### 仍漏的真实考点
- 监护制度（监护人指定 / 撤销 / 临时监护）— 没有专门情境
- 担保物权完整链（质押实现方式 / 留置 / 抵押权次序）— 主线 3 仅触及
- 征收征用补偿 — 物权域少触及
- 个人信息保护法专门规则（敏感信息处理 / 信息可携带权）— 仅在 S2-01 一带而过
- 劳动报酬专项（拖欠工资追讨程序 / 劳动监察）— 主线 6 较弱
- 网络平台格式条款 + 算法推荐合规 — 散落多处但没有专题
- 家庭暴力 / 婚姻无效与可撤销 — 主线 4 偏继承，婚姻部分较弱

### 涵盖性结论
**80 分**：核心覆盖到位，但还有 7 个边角制度可补。**不是必须 v6 强补**，因为低频。可以放进"开放容器层"作为接口。

---

## 2. 主干性评估（核心问题）

### 现状问题
v5 是**累积式**结构：v3 → v4 加 6 项 → v5 加 12 项 → 第八部分像补丁挂在第七部分后面。结果：

1. **学生读完后不知道"先学什么"**：8 个部分 + 6 张表 + 17 类陷阱 + 12 补充制度 = 100+ 条信息散落，缺少清晰的"读这三件即可上 30 分"的入口。
2. **第八部分游离**：12 项制度本应分别归到七大主线（如"民事法律关系三要素"应放主线之前作为底层；"诉讼时效"应放主线 7 程序救济）。现在它们独立成节，破坏了七大主线作为主干的连贯性。
3. **没有明示"主干高频层 vs 开放容器层"**：xuanbier skill 明确要求"主干高频层 + 开放容器层"双层呈现。v5 七大主线没有这层区分。学生看到从未考过的新题不知道用什么 meta-rule 归位。
4. **横切表 1-6 + 表 3-B + 表 4-A/B 信息密度大但权重不清**：哪个表最重要？学生不知道。
5. **第七部分 10 道题卡**和七大主线**信息重复**：题卡是把主线情境再写一遍，没有在主线之外提供新价值。

### 主干性结论
**60 分**：内容富但未"立住"主干。需要 v6 重组，不是再打补丁。

---

## 3. v6 重组方案

不再增加内容，而是**重组现有内容**让主干清晰：

### 改动 1：开篇 "Part 0 三件最重要的事"
在序言之前插入一节，告诉学生：
- 如果时间不够只学 3 件事就学这 3 件：①双段答题结构 ②横切六表（特别是表 4-5 反例库）③七主线下的"主干高频层"。
- 其他都是辅助和工具。

### 改动 2：把第八部分 12 项融入七大主线
- 民事法律关系三要素 → 已在 §1 续，保留并强化（"开题第一刀"）。
- 限制行为人合同 / 格式条款 / 要约承诺 / 物业不可抗力 → 主线 1 合同
- 好意同乘 → 主线 2 侵权
- 遗产管理人 → 主线 4 婚姻继承
- 商标抢注 → 主线 5 知产
- 平等就业 / 竞业限制 / 集体合同 → 主线 6 消费劳动
- 诉讼时效 / 起诉状 / 代理人称谓 / 司法建议 → 主线 7 程序救济
- 删除独立的"第八部分"，把节内容并入主线对应位置

### 改动 3：每条主线显式分"主干高频层 / 开放容器层"
- **主干高频层**：每条主线下 3-5 个最高频情境（即学生必背的）。
- **开放容器层**：每条主线下"如果你看到一道从未见过的题如何归位"的 meta-rule + 接口（"凡涉及 X 都可归这条主线"）。
- 边界归位规则补到 6-8 条（覆盖网络平台/新业态/跨域题）。

### 改动 4：第七部分 10 题卡精简为 5 道关键案例
- 删除与主线情境重复的题卡（其实是"主线下情境的复读"），只保留最有代表性的 5 道（每道是一个综合性、跨主线的"标杆题"）。

### 改动 5：横切表标权重
- 表 1 要件库 / 表 2 救济库 / 表 3 功能落点 → **核心三表**（写主观题第一段必查）
- 表 4 法律名 / 表 5 答案主体 / 表 6 边界禁区 → **防扣分三表**（自查时必看）
- 标 ★（核心）和 ☆（辅助）让学生知道权重。

### 改动 6：序言"7 痛点"压成"3 痛点"
现在序言列了 7 条痛点，对应 7 件事，但读完反而觉得乱。压成 3 条最关键痛点，对应 3 件最重要的事。
