# GPT-5.5 Pro Framework Council Report

## 1. Diagnosis

V0_SCAFFOLD 可以继续作为起点，但不能把“五域”当主框架。三题显示，评分更稳定奖励的是法律分析动作：定关系、找争点、套要件、落责任/效力/程序，再把意义收束到具体法律关系。

当前证据只覆盖 3 道主观题，其中 2 道为 formal_or_scoring_source，1 道为 reference_answer。因此足以支持 V1_CANDIDATE 的方向调整，但不足以宣布定稿。

核心诊断：

- “一核”和“四步”最稳，应上升为主干高频层。
- “五域”更像题域容器，负责归位，不宜排在学生第一学习层。
- “二线”中的价值表达有用，但必须锚定具体法律关系，否则容易变成宏观空话。
- B01-Q2 只能保留为开放容器或观察样本，不能推动主干升级。

## 2. High-Frequency Trunk

主干高频层建议排序：

1. 法律关系判断  
   三题均出现，formal 支撑 2 次。学生先判主体关系、身份关系、权利义务关系。

2. 争点事实识别  
   三题均需要从材料中找出真正争议，不是泛泛背法条。

3. 事实 -> 要件匹配  
   当前最像评分细则奖励动作，尤其 B01-Q1、B01-Q3 支撑较强。

4. 责任/效力/程序/裁判结果落点  
   答案必须落到支持什么、不支持什么、协议/行为效力如何、责任或程序如何处理。

5. 价值收束锚定具体法律关系  
   可作为“主干答法规则”，暂不宜变成独立大节点。它不是让学生背价值口号，而是要求意义类答案回到劳动、继承扶养、知识产权等具体法律关系中。

## 6. Hard Placement Or Proposition-Path Risks

- B01-Q2 最大风险是 reference_only 被误用为主干证据。它只能提示框架要留新技术容器。
- 若继续“五域优先”，B01-Q1、B01-Q3 会被简单塞进劳动/继承，却丢掉共同评分动作。
- “价值收束”若不锚定具体法律关系，会诱导学生写公平、秩序、法治、德治等空话。
- 对意义类题，不能先写价值再倒贴法律；应先完成法律判断，再从该法律关系推出意义。
- 未来遇到只有“法律、权利、规则”等词但法律关系不稳的题，应标 `proposition_path_uncertain`，不要硬归位。

## 7. Proposed Next Framework

V1_CANDIDATE 建议结构：

一核：以事实为根据、以法律为准绳，定分止争，培养法治思维。

主干高频层：

1. 定关系：主体、身份、权利义务关系。
2. 找争点：争议事实、事实边界、材料关键信号。
3. 套要件：把事实匹配到法定条件、制度规则、权利边界。
4. 落结果：责任、效力、程序、裁判或解决方案。
5. 锚价值：围绕具体法律关系写权利保护、秩序维护、公平正义、德治价值或治理意义。

命题路径层：

- 框架反馈：主干强化 / 容器保留 / 框架缺口 / proposition_path_uncertain

开放容器层：

## 8. Version Upgrade Criteria

允许从 V0_SCAFFOLD 推进到 V1_CANDIDATE，但不允许定稿。升级门槛：

- 主干节点必须有多道 formal_or_scoring_source 支撑，且最好跨不同题域复现。
- 开放容器升主干，必须出现 formal 细则反复奖励同一答题动作，reference_answer 不够。
- 新技术知识产权路径只有在出现正式评分细则同类题后，才能从容器升级。
- “价值收束锚定具体法律关系”若要成为稳定主干规则，需要更多意义类 formal 题验证。
- 每次升级必须提供 coverage/gap delta：哪些题更自然归位，哪些硬塞风险减少，哪些缺口仍保留。
- GPT 和 Claude 的意见只作为框架建议；最终必须回到本地题面和评分细则裁决。
- 无法核验证据事实时，一律写“需本地核验”。
