# Cross-Critique of GPT-5.5 Pro Web Framework Council Report

模式确认：本批判基于用户网页端独立 Claude Opus 4.7 Adaptive Thinking 对话生成，未继承其他项目上下文。仅对 GPT-5.5 Pro 报告做交叉批判，不重写自己的报告。

## 1. 同意的框架变化

**证据等级闸门 A/B/C/D 分级**（GPT 报告 §7.6, B01-N07）。这是 GPT 报告最重要的贡献之一，把"reference_answer 不能升格为 formal scoring source"从我的报告里隐含的守卫变成了显式的、可操作的分级。A 级"至少 3 道 formal 支持 + 跨 2 领域"作为主干门槛是务实的硬约束，能直接防止 N=3 样本被过度解读。同意进入融合稿。

**双视角分离教师后台与学生前台**（§7.2）。教师后台 "命题目标 → 载体选择 → 设问路径 → 评分奖励动作" 与学生前台分开，这与我的报告核心判断一致。GPT 把 V0 的"一核二线三问"消化进双视角的处理是合理的。同意进入融合稿。

**五域降为覆盖索引**（B01-N05, §7.5）。GPT 与我的判断一致，五域不作唯一主干，但保留作为知识抓手与归位索引。同意。

**结果落点扩展为四类（责任/效力/归属/程序）**（B01-N13, §7.4）。我的原报告把"落处理"作为单一动作可能过宽。B01-Q3 的"遗产归属"确实需要在"责任"之外有专门位置。同意作为**教师后台分类工具**进入融合稿，但学生侧不应作为四个并列记忆点（理由见下）。

**双向反偏测试（防法考化 + 防宏观法治化）**（B01-N14, B01-N15, §8.4）。两条警告与我的报告中"套法定要件法考语感""价值收束跳过案件"风险完全一致。同意双向反偏作为框架升级的硬测试。

**开放容器扩列**（§4：程序题、效力题、比较题、选择题、人格权侵权、消费者合同、生态公益、新业态）。我的报告只显式提了"新技术"和"新业态"两个容器，GPT 把覆盖盲区更系统地列出来，并明确每个都需 formal 证据补强。同意，但所有容器都按 GPT 自己的 C 级标准处理，不允许 reference_only 推动升格。

## 2. 反对或要求降级的变化

**反对：把"收价值"列为第五个主干动作并行。** GPT 在 §7.3 把价值收束与定关系、抓争点、对规则、落结果并列为五个主干动作。这有两个问题：第一，价值收束是**条件性触发**的——只有设问明确要求"意义/价值/体现什么"时才需要做；并列写成第五动作会让学生在不问意义的题目里也强行加价值上升，反而稀释得分。第二，B01-Q1 与 B01-Q3 的 formal 证据支持的是"问到意义时不能空泛"，不是"每题都要收价值"。建议降级回我原报告的处理：四步主干 + **价值一规**（条件触发的硬规则），不并列。

**反对：学生前台"载体 → 法律关系 → 争点 → 规则 → 结果 → 价值"六步**（GPT 在 §7.2 学生前台列六节点，又在 §9 问题 2 提议）。六节点与五个主干动作高度重叠，且与"看题三识别（载体→问点→收束）+ 答题四步 + 价值一规"功能等价但更冗长。学生考场上多记一层就多一层调用成本。建议保留我原报告的"识别动作 + 四步 + 一规"分层，不要展开为六步线性流程。

**反对：为比较题、选择题单独建立独立模板**（§6.4 比较题"比较对象→标准→适用→选择→价值"；§6.5 选择题"微型四步"）。这两条会把学生记忆负担从一套四步动作扩张到三套模板。建议降级为**教师后台命题分类**：比较题在内部仍然是"对每个比较对象各跑一次定关系→抓争点→对规则→落结果，再对比"，选择题是"压缩版的同一动作链"。学生侧只学一套，遇到比较/选择是同一套的展开或压缩，不是新模板。

**降级：B01-N03 措辞"对规则/条件/要件/程序要求"**。"要件"二字仍带法考语感，与"防法考化"目标自相矛盾。建议改为"对应规则 / 适用条件 / 程序要求"，把"要件"从学生侧表述中删除，仅在教师后台保留。

**降级：A 级标注的当前状态。** GPT §7.6 把法律关系判断、事实→要件匹配、落点、价值收束四节点全部标为"A 候选"。这个表述容易让 Codex 误读为"已经是 A 级"。按 GPT 自己定义的 A 级门槛（≥3 道 formal + 跨 2 领域），当前 Batch01 中 formal 只有 2 道（B01-Q1 劳动 + B01-Q3 继承），跨 2 领域满足，但样本量未达 3。**当前所有节点最高只能到 B 级候选，没有任何 A 级**。建议在融合稿里把这四个节点显式降为"B 候选 / A 待验证"。

**降级：四类结果落点作为学生记忆点。** 同意作为教师后台分类（如§2 同意条所述），但学生侧不应背"责任/效力/归属/程序"四并列。学生只需在"落结果"这一步记住"判处理结果是什么类型——是钱、是有效无效、是归谁、还是走哪条程序"，作为一个识别提示，不是四个并列分支。

## 4. 必须等待 formal_or_scoring_source 本地核验的建议

以下 GPT 报告中的具体声明在 Codex 回到本地题面与评分细则核验之前**不能进入框架升格决策**：

第一，B01-Q1 "三个从属性材料匹配"是评分细则原文动作还是研究者归纳。GPT 在多处（§2.2、§3.1、§9 问题 4）依赖此措辞做主干推论。需 Codex 核验评分细则原文用词。

第二，B01-Q3 "传统美德/核心价值观"是评分细则原文价值层还是参考答案表述。GPT §3.3、§9 问题 5 依赖此点判断价值收束分层。需核验。

第三，B01-Q3 评分细则是否**单独奖励**"效力判断理由"而非只奖励最终归属（§4.3）。这直接决定纯效力题是否要从开放容器升格。需核验。

第四，所有"formal_or_scoring_source 多题支持"的 A 级判断必须等待 Codex 核验"formal" 等级的原始来源是题面+评分细则原文，还是教师参考答案。当前 Batch01 标注的 evidence_level 字段本身需要溯源核验。

第五，纯程序题、纯效力题、比较题、选择题、人格权侵权、消费者合同、物权相邻、生态公益题型的 formal 样本是否存在，存在的话评分动作是否仍能归入五主干动作。这是 GPT 报告 §4、§6、§8.3 的核心未决问题。

第六，B01-Q2 是否在后续 batch 中出现 formal_or_scoring_source 级证据。在出现之前，"新技术 + 权利保护 + 市场秩序"路径不能进入主干升级决策，无论它在概念上多么自洽。

第七，价值类评分项的具体表述是否惩罚空泛价值表达（B01-N04, B01-N15）。当前是推论，需评分细则原文佐证。

第八，五域中"人格权侵权""消费者合同""物权相邻""生态公益与新业态"是否在历年评分细则中出现，及其评分动作能否仍归入五主干。GPT §4.6 已标注待核验，同意。

## 5. 给 Codex 的最终融合建议

### Accept（直接进入融合稿）

证据等级闸门 A/B/C/D 分级机制（B01-N07 + §7.6），按 GPT 定义的门槛使用，但当前 Batch01 所有候选节点最高只能标 B，不允许任何节点在本 batch 直接标 A。

教师后台 / 学生前台双视角分离（§7.2 + B01-N08, B01-N09）。教师后台保留"命题目标 → 载体 → 设问路径 → 奖励动作"，学生前台保留四步 + 一规 + 知识索引。

五域降为覆盖索引（B01-N05, §7.5）。

防法考化 + 防宏观法治化双向反偏测试作为框架升级硬关卡（B01-N14, B01-N15, §8.4）。

reference_answer 不能推动主干，作为框架的强制证据守卫。B01-Q2 维持开放容器位，不进入主干。

开放容器扩列（程序、效力、比较、选择、人格权侵权、消费者合同、生态公益、新业态），所有容器仅作占位，等待 formal 证据。

### Revise（接受但需修改后融合）

GPT-N03 四步措辞，"对规则/条件/要件/程序要求"改为"对应规则 / 适用条件 / 程序要求"，删"要件"二字。学生侧表述与教材语言对齐。

GPT §7.3 五主干动作：保留"定关系 / 抓争点 / 对规则 / 落结果"作为四步主干，价值收束**降为条件触发的"价值一规"**，不并列为第五动作。

GPT §7.4 四类结果落点（责任/效力/归属/程序）：作为**教师后台分类工具**保留；学生侧"落结果"步合一表述，仅作为识别提示词，不作并列记忆。

GPT §7.6 当前节点 A 级状态：全部降为"B 候选 / A 待验证"，明确表明 Batch01 样本未达 A 级门槛。

GPT 价值规则在多处重复（N10、6.7、7.5、8.4.2）：在融合稿合并为单一"价值一规"条目。

### Defer（暂缓，待证据再议）

比较题独立模板（§6.4）—— 待 formal 比较题样本出现后再决定是否独立成模板，还是作为四步的展开。

选择题"微型四步"（§6.5）—— 待 formal 选择题样本与错因统计后再决定。

纯程序专用分支"选路径 → 说理由 → 明后果"（§9 问题 7）—— 待程序题 formal 样本。

举证责任 / 证据效力 / 事实认定独立得分动作（§9 问题 13）—— 待样本。

"三个从属性材料匹配"是否归为劳动关系专属节点 vs 通用节点（§9 问题 4）—— 待评分细则原文核验后裁决。

"传统美德 / 核心价值观"是否归为继承扶养专属价值 vs 通用规则（§9 问题 5）—— 待核验。

新就业形态劳动关系是否作为年度热点容器独立追踪（§9 问题 15）—— 待跨年份样本。

新技术容器是否优先追踪（§3.2 末段）—— 与其他开放容器同级处理，不优先，待 formal 证据再议。

五域配套"主体关系对训练库"（§9 问题 12）—— 即便要做，也只能作为教师后台资源，不进学生侧。先 defer。

继承扶养题中"个案公平 / 养老司法理念 / 传统美德"的层级关系（§9 问题 14）—— 待评分细则核验。

### Reject（不进入融合稿）

学生前台展开为"载体 → 法律关系 → 争点 → 规则 → 结果 → 价值"六步线性流程（§7.2 学生前台 + §9 问题 2）—— 与四步 + 识别动作 + 一规重叠，纯增学生记忆负担，无证据收益。

把价值收束并列为常规第五主干动作而非条件触发规则（§7.3 第 5 项）—— 风险是把不问意义的题目也强行价值化，违反"价值锚定本案"反偏测试自身的精神。

任何在本 batch 内直接给节点标 A 级的处理 —— 即便 GPT 自己标的"A 候选"也需在融合稿降为 B。

任何把 B01-Q2（reference_answer 级）的"新技术 / 权利保护 / 市场秩序"路径作为优先追踪容器或推动主干的处理。
