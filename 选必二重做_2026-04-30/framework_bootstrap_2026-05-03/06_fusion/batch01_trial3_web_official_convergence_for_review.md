## 3. Main Conflict And Codex Decision

### 3.1 四步还是五步

GPT 初稿倾向“五个动作”：定关系、抓争点、对规则、落结果、收价值。Claude 反对把“收价值”并列为每题必做动作，认为价值是条件触发。

Codex 裁决：采纳 Claude 的学生端处理。

- 学生主干：四步。
- 价值处理：单列“价值一规”，只有设问问意义、认识、价值、启示时触发。
- 教师后台：可记录价值收束动作，但不得让学生每题强行加价值。

### 3.2 “要件”措辞

Claude 建议学生端删除“要件”以防法考化。GPT 反驳认为 `B01-Q1` 的从属性材料匹配说明“条件/要件匹配”仍是核心动作。

Codex 裁决：学生端不用“套法定要件”，改为“对规则/条件/程序要求”；教师后台可以保留“要件匹配”作为分析术语。

### 3.3 A/B/C/D 证据等级

GPT 提出 A/B/C/D 闸门，Claude 接受但要求降级当前状态。

Codex 裁决：采用证据等级闸门。

- A：多道 formal_or_scoring_source 支持，且跨题域、能解释评分动作。
- B：单道或少量 formal_or_scoring_source 支持，可作重点候选。
- C：reference_answer 支持，只能进开放容器。
- D：框架推断，必须写“需本地核验”。

当前 Batch01 中三题全命中的动作可记为 `Trial-A 候选`，但稳定 A 级仍需后续批次、跨题型、跨年份验证。

## 4. Current Student-Facing Trial Framework

### 零步看问法

- 载体：题给了什么法律生活场景。
- 问点：题到底问成立、有效、侵权、谁担责、怎么处理，还是意义。
- 收束：最后要落到处理结果，还是落到社会价值。

### 四步答题

1. 定关系：这是谁和谁，是什么法律关系。
2. 抓争点：材料里真正争的是什么事实或权利义务。
3. 对规则/条件：把材料事实对应教材法律规则、适用条件或程序要求。
4. 落处理：支持谁、为什么，责任、效力、归属、救济路径或裁判理由是什么。

### 价值一规

问意义、认识、社会价值、启示时启用：

> 先说本案是什么法律关系、判了什么、为什么这样处理，再上升到权利保护、市场秩序、公平正义、家庭伦理、劳动保护、创新秩序等价值。

禁止跳过案件直接写宏观法治口号。

### 五域索引

五域只作知识定位和覆盖检查：

- 合同消费者劳动
- 物权相邻继承家庭
- 人格权侵权
- 知识产权不正当竞争
- 纠纷解决生态公益新业态

## 5. Three-Question Placement

### B01-Q1：2024 朝阳二模 第17题

归位：合同消费者劳动 -> 劳动关系认定 -> 新就业形态开放追踪。

命题路径：平台经济载体 -> 合作协议外观与实际用工事实冲突 -> 从属性材料匹配 -> 劳动关系认定 -> 经济补偿与治理意义。

对框架的作用：

- 支持“定关系”。
- 支持“抓争点事实”。
- 支持“对规则/条件”。
- 支持“意义要锚定劳动关系确认与劳动者权益保护”。

待核验点：评分细则原文是否直接使用“三个从属性”表述，还是本地摘要归纳。该点决定“从属性材料匹配”是劳动专属节点，还是通用“事实对应规则/条件”的样本。

### B01-Q2：2024 海淀一模 第19题

归位：知识产权不正当竞争 -> 新技术开放容器。

命题路径：新技术现象 -> 权利保护 -> 市场秩序 -> 创新与规范平衡。

对框架的作用：

- 可证明“知识产权/不正当竞争”需要进入正式覆盖索引。
- 可支持“权利保护 -> 侵权/竞争行为识别 -> 责任/价值收束”的通用动作。
- 不能证明“AI/数字人/新技术”是高频主干。

状态：formal 支持知识产权/不正当竞争；新技术载体仍为 single-case open container。

### B01-Q3：2024 海淀二模 第19题

归位：物权相邻继承家庭 -> 遗赠扶养/继承 -> 协议效力与履行 -> 遗产归属 -> 社会价值。

命题路径：养老与继承生活案例 -> 协议效力与履行争议 -> 裁判确认遗产归属 -> 个案公平与社会价值。

对框架的作用：

- 支持“落处理”不能只写责任，还要容纳效力和归属。
- 支持“价值一规”：社会价值必须从具体判决和法律关系推出。
- 支持五域中“物权相邻继承家庭”的覆盖索引。

待核验点：评分细则原文中“养老司法理念、传统美德、核心价值观”的层级关系，是否单独分层给分。

## 6. Accepted / Revised / Deferred / Rejected

### Accept

- 教师后台 / 学生前台分离。
- 四步作为 Batch01 暂定学生主干。
- 五域降为覆盖索引。
- 证据等级闸门。
- reference_answer 不能推动主干。
- B01-Q2 进入知识产权/不正当竞争索引，并把虚拟数字人/AI 放入新技术开放容器。
- 防法考化与防宏观法治化作为升级硬测试。

### Revise

- GPT 的“五个主干动作”改为“四步 + 价值一规”。
- `套法定要件` 改为学生端 `对规则/条件/程序要求`。
- `责任/效力/归属/程序` 不做学生并列记忆点，只作为“落处理”里的识别提示和教师后台分类。
- GPT 报告中的 `A 候选` 不直接当稳定 A；改记为 `Trial-A 候选 / 后续批次验证`。

### Defer

- 比较题独立模板。
- 选择题微型四步。
- 纯程序专用分支。
- 举证责任、证据效力、事实认定是否独立成节点。
- 新就业形态是否独立热点容器。
- 新技术容器是否优先追踪。
- 每个五域是否建立主体关系类型库。
- formal/reference 中间档。
- 区域/命题人偏好元数据。

### Reject

- 把价值收束变成每题必做第五步。
- 把学生前台展开成六步线性流程。
- 本 batch 直接标 A 级。
- 用 B01-Q2 reference_answer 推主干或优先热点。
- 把选必二框架写成法考式细目背诵。
- 把选必二意义题写成脱离个案的必修三宏观法治口号。
