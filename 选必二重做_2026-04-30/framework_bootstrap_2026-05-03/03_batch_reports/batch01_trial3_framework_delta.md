# Batch 01 Trial 3 框架变化候选

## 当前结论

前三题初步支持 v0，但也暴露出必须重排层级：不能只用“五域”做主框架，真正的主干应是答题动作和命题路径。

## 候选主干高频层

### T1 法律关系判断

来源：

- B01-Q1：平台企业与劳动者是否构成劳动关系。
- B01-Q3：遗赠扶养协议与继承争议。
- B01-Q2：著作权人与使用者/竞争者关系，但证据为 reference-only。

升级理由：

前三题全部先要求判断主体关系和权利义务关系。该节点应高于具体五域。

### T2 事实 -> 要件匹配

来源：

- B01-Q1：三个从属性逐项匹配材料。
- B01-Q2：著作权、未经许可使用、不正当竞争构成，但 reference-only。
- B01-Q3：协议有效和扶养义务履行。

升级理由：

这是法律模块区别于必修三宏观法治题的核心动作。

### T3 裁判/处理结果落点

来源：

- B01-Q1：仲裁支持经济补偿。
- B01-Q2：消除影响、赔偿损失。
- B01-Q3：确认协议有效、遗产归属。

升级理由：

学生必须把规则判断落到责任、效力、程序或裁判结果。

### T4 价值收束必须锚定具体法律关系

来源：

- B01-Q1：法律、道德、经济意义。
- B01-Q3：公平正义、助力养老、传统美德和核心价值观。
- B01-Q2：创新活力、市场秩序，但 reference-only。

升级理由：

命题人图片提醒主观题考“法律如何推动法治建设、社会经济发展”，但不能脱离制度本身空写价值。

## 开放容器层候选

- AI/虚拟数字人/新技术知识产权：B01-Q2 暂入开放容器，待 formal 题源补强。
- 典型案例意义三分法：法律意义、道德意义、经济/社会意义。B01-Q1 formal 支撑强，B01-Q3 formal 支撑家庭养老价值，但还需更多题验证是否作为稳定模板。

## 对 v0 的修正方向

v0 的“五域”适合做容器，但不宜做最上层主干。建议下一版框架初步改为：

1. 命题路径层：命题目标 -> 载体 -> 设问路径 -> 细则奖励动作。
2. 法律分析主干层：定关系 -> 找争点 -> 套要件 -> 落责任/效力/程序。
3. 价值收束层：个案公平 -> 类案规则 -> 领域治理 -> 法治/德治价值。
4. 题域容器层：劳动/合同消费者/知识产权竞争/继承家庭/侵权人格/纠纷生态新业态。
