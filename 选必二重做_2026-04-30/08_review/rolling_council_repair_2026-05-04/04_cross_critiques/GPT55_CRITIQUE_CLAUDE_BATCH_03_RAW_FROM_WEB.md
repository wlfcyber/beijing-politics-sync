## 1. 我同意 Claude 的哪些框架变化，为什么

### 1.1 同意把“关键词归域污染”列为 Batch03 第一风险

Claude 指出 Batch03 至少有多道题被“侵权、赔偿、法院、民法典”等关键词误拉入“人格权与侵权责任”域，这个判断方向应采纳，具体题数和最终归属**需本地核验**。题卡中共享单车共有部分、牙膏三倍赔偿、真创新真保护、数据抓取、竞业限制、摄影作品著作权等材料，确实显示“题卡归域”不能直接当事实。 

Codex 应把以下规则写成硬闸门：

| 闸门     | 执行动作                              |
| ------ | --------------------------------- |
| 法律小问入框 | 先判是否有具体法律主体之间的权利义务关系              |
| 题域重判   | 再判是合同消费、物权相邻、人格侵权、劳动职业、创新竞争还是程序救济 |
| 关键词降权  | “侵权”“赔偿”“法院”只作提示，不直接决定题域          |
| 混合题拆分  | 表格题、并列案例题、综合民法题按案例、栏目、小问拆分        |

这个修正对学生很关键。若按关键词归域，学生会把“出现赔偿”误学成“人格权侵权”，会破坏“一核”中“具体法律关系”的判断。需本地核验。

### 1.2 同意“具体人格权类型识别”进入人格侵权卡

Claude 强调肖像权、隐私权、个人信息、身体权、健康权等具体人格利益不能笼统写“人格权”，方向正确。Batch03 题卡中未成年人照片、摄像头拍摄日常出行动态、个人信息处理、身体健康损害等材料，确实支持建立“具体人格权识别卡”。具体哪些题真实采分、是否要求写具体权利名称，**需本地核验**。 

建议学生版表达：

> 先说侵害了哪一种具体人格利益，再说明材料事实如何对应，最后落到责任方式。不能只写“侵犯人格权”。

但 Claude 所说“错字不给分”不能立即全局硬化，见第 2 部分。

### 1.3 同意安全保障义务应作为侵权域高频子卡候选

Claude 将公共场所经营者、管理者安全保障义务列为侵权域子卡，方向可采纳。Batch03 中景区湿滑、摇摆桥项目、公共场所摔倒等材料反复涉及“经营者是否尽到安全保障义务”“合理限度”“受害人自我注意义务”等判断，具体采分稳定性**需本地核验**。 

学生版可形成轻量卡：

| 子卡 | 学生动作                     |
| -- | ------------------------ |
| 主体 | 公共场所经营者、管理者、旅游公司、景区等     |
| 义务 | 合理限度内的安全保障义务             |
| 事实 | 是否有危险、提示、防护、管理、工作人员行为    |
| 边界 | 受害人是否为完全民事行为能力人，是否未尽注意义务 |
| 结果 | 承担或不承担侵权责任，价值后置          |

这样既能防止学生把经营者写成无限责任，也能防止把所有摔倒题都写成“自行承担”。需本地核验。

### 1.4 同意合同责任与侵权责任竞合需要识别

Claude 把合同与侵权竞合列为识别动作，值得保留。Batch03 中运输、旅游、消费、摇摆桥、景区等材料常同时出现合同关系和人身损害或安全保障义务，学生需要知道：有合同关系不排除侵权责任；有人身损害也不意味着只写侵权。具体题目是否奖励双线作答，**需本地核验**。 

Codex 可写成横切动作：

> 先判合同关系是否存在，再判是否侵害人身、人格或其他绝对权；最后按设问选择违约、侵权或双线说明。

### 1.5 同意“见义勇为与受益人适当补偿”可做特殊规则卡候选

Claude 将题 15 见义勇为受益人补偿列为特殊规则卡，有教学价值，但应暂列弱证据。该题能够提醒学生区分“赔偿”和“补偿”，也能体现价值后置，具体是否多题复现、是否进学生主干，**需本地核验**。 

建议处理：

> 作为侵权域下“特殊补偿卡”或“开放小卡”，不进入高频主干第一层。需本地核验。

### 1.6 同意 formal 摘录污染检测应成为流程级硬步骤

Claude 指出题 11、题 12 的评分摘录疑似与设问不对应。Batch03 Evidence Pack 中也可见部分评分摘录含有与题面不一致的内容，如联想思维、土地权利、乡村振兴等，是否真实污染**需本地核验**。 

Codex 应加入流程：

1. 题号对齐。
2. 设问对齐。
3. 分值对齐。
4. 材料主题对齐。
5. 评分点与法律小问对齐。
6. 不对应则标记为 scoring_extract_contaminated，暂不推动主干。

这一点应直接 accept。

### 1.7 同意第四步“结果落点”按域分流

Claude 批评“责任、效力、归属、救济、价值”五并列过满，这一批评成立。Batch03 侵权题更适合落到责任方式或不承担责任；Batch02 物权相邻更适合落到归属、排除妨害、停止妨害、提供便利；Batch01 合同消费更适合落到效力、违约责任、惩罚性赔偿。各批次具体落点仍**需本地核验**。

建议 Codex 废止学生版“五并列落点”，改为“按域选落点表”。

## 2. 我反对或降级 Claude 的哪些变化，为什么

### 2.1 降级“侵权三要件作为硬识别主干”

Claude 把“过错、损害、因果”称为侵权域硬识别主干。方向有用，但“硬主干”表述过满，需降级为**轻量侵权判断链**。原因是：高中政治选必二的侵权题不宜讲成法考式三要件或四要件体系；同时，不同题型可能更看重权利类型、安全保障义务、举证责任、责任边界、补偿与赔偿区分。哪些题真正按“过错、损害、因果”给分，**需本地核验**。

建议修改为：

> 侵权轻量链：行为，损害，因果，过错或特别规则，责任方式。
> 不要求学生系统背侵权构成理论。

这样能保留采分动作，避免法考化。

### 2.2 降级“归责原则三分必须恢复”

Claude 提出一般过错、过错推定、无过错三分应部分恢复为识别动作。这里可接受“识别动作”，但不能接受“归责原则三分”进入学生前台主干。Batch03 中动物园、公共场所、高空抛物、环境污染等选择题题肢确实会出现举证、过错、无过错等陷阱，具体是否稳定为多题采分点**需本地核验**。 

学生版更稳的表达：

| 不建议               | 建议                       |
| ----------------- | ------------------------ |
| 一般过错、过错推定、无过错系统分类 | 看谁负注意义务、谁举证、能否免责         |
| 归责原则三分体系          | 典型场景识别：公共场所、动物园、高空抛物、污染等 |
| 法考式归责口诀           | 错项排除：无过错、举证倒置、必然赔偿、全部住户等 |

这与此前“反法考化”原则不冲突。恢复的是排错识别，不是体系背诵。需本地核验。

### 2.3 反对“侵权必须独立成域，不能并入人身与家庭”的绝对化

Claude 认为 Batch03 完全否定“人身与家庭”合并方案，这个判断过急。Batch03 只能说明人格侵权和家庭继承的采分动作不同，不能直接决定学生前台五域一定要拆成六域或七域。项目硬要求是一核二线三问四步五域必须保留并解释，不能因一个批次把第五域待定或重画全局五域。Claude 的“五域候选第五域待定”不适合进入 Codex。需本地核验。

更稳的方案：

| 前台五域    | 二级卡                 |
| ------- | ------------------- |
| 交易与财产   | 合同消费卡、物权相邻卡         |
| 人身与家庭   | 人格侵权卡、家庭继承卡         |
| 劳动与职业   | 劳动关系卡、解除补偿卡、竞业限制卡   |
| 创新与竞争   | 著作权卡、不正当竞争卡、数据技术秘密卡 |
| 救济与公共利益 | 程序救济卡、执行救济卡、生态公益卡   |

这样承认“人格侵权”有独立动作，但仍不额外增加学生前台域数。是否采用此五域，需本地核验。

### 2.4 反对“五域第五域待 Batch04 定”

Claude 建议学生五域为“合同与消费、物权与相邻、家庭与继承、人格权与侵权、第五域待定”。这会造成学生版结构不闭合，也会临时悬空劳动、知识产权、不正当竞争、程序救济。Batch03 只讨论人格侵权，不能用本批决定全量五域。需本地核验。

Codex 不应保留“第五域待定”。可以后台继续核验，但学生前台框架必须闭合。

### 2.5 降级“具体人格权错字不给分”全局硬规则

Claude 把隐私权错字不给分提升为具体人格权名称必须写对的硬规则。这个提醒很重要，但不能未经核验推广到全部人格侵权题。题 8 细则若确有“隐私权字写错不给分”，只能说明该题或该细则口径严格；是否多题普遍如此，**需本地核验**。 

学生版应写成：

> 具体人格权名称要尽量写准，尤其是肖像权、隐私权、个人信息、名誉权、身体权、健康权。若本地细则明确错字不给分，则按严格口径训练。需本地核验。

### 2.6 降级“上诉期具体天数 = 错项陷阱”

Claude 认为三个批次连续出现上诉期天数陷阱，应作为学生识别原则。这个观察可以保留为选择题错项提示，但不能推广为“看到具体天数就错”。有些题可能会正确考查具体期限；是否在本地样本中多数为错项，**需本地核验**。

建议改为：

> 程序期限类题肢不凭记忆硬选。先看题干是否要求程序知识，是否表述为判决后、生效后、裁定后、上诉期等；具体天数不进入学生主干，只做错项警惕。需本地核验。

### 2.7 降级“一核表达进意义题/裁判依据题首句”硬化

Claude 认为 Batch02 和 Batch03 两次出现一核表达，说明应进入首句采分要素。这个结论过满。题 16 若确实出现“以事实为根据、以法律为准绳”，说明该题可用作高分句；但不能推出每道裁判依据题、意义题都必须首句写一核。是否多题明确给分，**需本地核验**。

建议定位为：

> 教师版高分提示；学生版在裁判依据题、判决意义题中可优先使用，但不作为所有题硬动作。需本地核验。

### 2.8 降级“侵权域第四步四种典型责任方式”

Claude 将停止侵害、赔礼道歉、赔偿损失、消除影响列为侵权域第四步四种典型。可以保留为责任方式库，但不能默认每题都从这四个中选。公共场所安全保障义务题可能落到“不承担赔偿责任”；见义勇为题落到“适当补偿”；数据竞争或著作权题可能落到“停止侵权、消除影响、赔偿”等。具体落点**需本地核验**。

建议改为：

> 侵权域第四步落到责任承担方式或责任边界，包括承担、不承担、适当补偿、停止侵害、赔礼道歉、赔偿损失、消除影响等，按设问和材料选择。需本地核验。

## 3. 哪些建议会增加学生负担、过度法考化、过度宏观法治化

### 3.1 会增加学生负担的建议

| 建议                        | 风险                   | 修正                    |
| ------------------------- | -------------------- | --------------------- |
| 五域第五域待定                   | 学生结构不闭合，后续会变成六域、七域   | 固定五域，二级卡分化，需本地核验      |
| 人格侵权、家庭继承、物权相邻、合同消费全部独立上墙 | 前台域数过多               | 前台五域，后台八域，需本地核验       |
| 具体人格权错字不给分全局化             | 训练焦点从法律关系转成字词洁癖      | 作为严格提醒，不作全局采分规则，需本地核验 |
| 每个侵权题都套过错、损害、因果、归责原则      | 会遮蔽人格权类型、安全保障义务、合同竞合 | 改为轻量链和题型触发，需本地核验      |
| 上诉期天数陷阱单列为新排错             | 排错表过长                | 放入程序错位或期限陷阱小提示，需本地核验  |

### 3.2 会过度法考化的建议

1. **归责原则三分体系前台化。**
   可以讲“谁举证、是否合理限度、能否免责”，不讲完整一般过错、过错推定、无过错体系。需本地核验。

2. **侵权三要件硬主干化。**
   可作为轻量链，不做构成要件理论。需本地核验。

3. **个人信息、隐私、肖像、名誉等细分无限扩张。**
   学生需要识别常见类型，不需要背人格权目录。需本地核验。

4. **程序期限天数记忆化。**
   上诉期、举证期限等不进主干，只作错项警惕。需本地核验。

5. **合同侵权竞合讲成民法责任竞合理论。**
   学生只需判断材料中是合同履行问题、人身损害问题，还是二者并存。需本地核验。

### 3.3 会过度宏观法治化的建议

1. **真创新真保护题若仍放人格侵权域。**
   容易让学生跳过技术秘密、不正当竞争、知识产权保护，直接写“公正司法保护创新”。应回到创新与竞争，具体归属需本地核验。

2. **见义勇为题只写核心价值观。**
   应先写保护他人民事权益、受益人适当补偿，再写传统美德。需本地核验。

3. **安全保障义务题直接写社会主义核心价值观。**
   应先写是否尽到合理限度内安全保障义务，再写自我负责、安全文明出行等价值。需本地核验。

4. **数据产品抓取题直接写数字经济治理。**
   应先写数据竞争权益、商业道德、诚实信用、不正当竞争，再写市场秩序。需本地核验。

5. **网络空间法治化作为人格权题开头。**
   应先写肖像、隐私、个人信息等具体权利及责任，再写网络空间治理。需本地核验。
