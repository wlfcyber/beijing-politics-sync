# 夜间心跳巡检协议

## 每轮巡检要做

1. 读取 `MASTER_REQUIREMENTS.md` 与 `THREAD_REGISTRY.md`。
2. 检查三条线最近 30-60 分钟是否有新产物、新日志或新风险。
3. 检查是否出现以下假闭环信号：
   - 只有正文，没有覆盖矩阵。
   - 只有 Markdown PASS，没有 Word/PDF QA。
   - 只有 GPT/Claude 建议摘要，没有真实截图/原始回复/日志。
   - 把参考答案写成评分细则。
   - 学生正文混入路径、debug、PASS、final、rubric、细则位置等审计语。
4. 对每条线给出下一步硬任务，写入 `patch_orders/`。
5. 更新 `snapshots/heartbeat_YYYYMMDD_HHMM.md`。

## 三条线优先级

1. 选必二：若 v13_10 已交付，优先核它是否真满足 GPT/Claude 框架迭代与聪明高三学生全对标准；若不满足，继续逼框架和题目解析修补。
2. 选必一：优先查是否已经覆盖全部考题，是否达到哲学宝典同级的“框架优先 + 逐题挂载 + 答案句可写”密度。
3. 必修四：优先查“旧缺题”和“新增题”是否全补入已认可框架，不能因为 v7/v8 文件存在就停止。

## 禁止动作

- 不得把 Codex 子 agent 的意见当作 GPT Pro 或 Claude Opus。
- 不得让一个总报告覆盖三条线的逐题矩阵。
- 不得把“看起来很好”当作 Confucius 通过。
- 不得覆盖或重写已认可版本本体；所有改动先写补丁指令和候选产物。
