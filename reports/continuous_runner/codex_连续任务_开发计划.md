# Codex 连续任务器开发计划

## 目标

- 搭建一套可复用的 `codex exec` 连续执行机制。
- 用“任务步骤文档 + 任务进度文档”驱动 Codex 循环推进任务。
- 只有当计划文档与进度文档的已完成步骤完全一致，且计划文档不再存在未完成步骤时，脚本才停止。

## 交付物

- [x] DEV-01 明确脚本的输入、输出、停止条件和文档格式
- [x] DEV-02 创建开发计划、开发进度、任务步骤模板、任务进度模板
- [x] DEV-03 实现步骤同步校验脚本，能比较计划文档与进度文档是否一致
- [x] DEV-04 实现 `codex exec` 循环执行脚本，支持持续推进任务并在每轮后重新校验
- [x] DEV-05 完成一次本地自检，确认脚本可运行且文档状态能对齐

## 约束

- 脚本默认面向当前工作目录运行。
- 任务步骤文档中的步骤 ID 必须唯一，推荐使用 `TASK-01` 这种格式。
- 任务步骤文档和任务进度文档中的步骤标题必须保持一致，便于脚本做集合比较。
- 每完成一步，都要同时更新计划侧和进度侧文档。
