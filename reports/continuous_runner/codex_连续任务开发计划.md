# Codex 连续任务开发计划

## 目标

- 在当前工作区搭建一套由 `codex exec` 驱动的连续开发机制。
- 机制必须以`开发计划文档`和`开发进度文档`为唯一执行底板。
- 每轮开发后都要更新进度文档。
- 只有当`进度文档`与`计划文档`中的任务编号完全对应，且全部任务状态为`done`时，循环才允许停止。

## 文档约定

- 计划文档的机器可读核心是 `## 任务清单` 下的 Markdown 表格。
- 进度文档的机器可读核心是 `## 步骤状态` 下的 Markdown 表格。
- 两份文档都必须使用同一组 `ID`。
- 进度状态只允许使用：
  - `todo`
  - `doing`
  - `done`
  - `blocked`
- 停止条件：
  - 计划文档中的全部 `ID` 在进度文档中都存在；
  - 进度文档中不存在计划外 `ID`；
  - 全部状态都是 `done`。
- 中止条件：
  - 任一任务状态变成 `blocked`。

## 给 Codex 的执行规则

- 开始任何开发前，必须先读本文件和进度文档。
- 必须优先完成尚未 `done` 的任务。
- 必须真实开发，不允许只做分析后退出。
- 每完成一轮开发，必须回写进度文档中的：
  - `总体状态`
  - `步骤状态`
  - `迭代记录`
- 如果任务全部完成，最后消息应明确给出 `COMPLETE`。
- 如果遇到无法自行解除的阻塞，必须把对应步骤标成 `blocked`，并在最后消息中明确给出 `BLOCKED`。
