# Codex 总督工令：只做哲学，一套一套闭环

这是本轮最高优先级补充指令，晚于 `TASK_BRIEF.md` 和 `CLAUDECODE_START_PROMPT.md`。若有冲突，以本文件为准。

## 1. 范围收紧

- 本轮最终产物只要必修四哲学线，不要文化线。
- 文化题、纯文化评分点、传统文化/民族精神/文化自信/文化功能等内容不得进入最终学生版。
- 如果一套卷只有文化题而没有可用哲学触发点，在 `COVERAGE_MATRIX.csv` 和对应套卷报告里标为 `module-boundary-excluded`，不要为了凑数硬塞进哲学框架。
- 允许在审计文件中简短记录“本题/本套为文化边界排除”，但学生版不生成文化章节。

## 2. 工作方式：一套一套做完

- 不准先粗扫一堆目录后直接写总文档。
- 必须按套卷推进：每处理一套卷，都完成“题源定位 -> 题号筛选 -> 评分证据判断 -> 哲学触发链 -> 选择题边界 -> 套卷报告 -> 进度更新”这一整套闭环，再进入下一套卷。
- 每套卷处理完后，在 `suite_reports/` 下生成一个套卷报告，文件名用 `年份_区县_阶段.md`，例如 `2026_海淀_一模.md`。
- 每套卷报告必须写明：
  - 原始试卷文件、答案文件、评分细则/评标/讲评文件；
  - 哲学主观题题号与评分证据等级；
  - 选择题中是否有必修四哲学相关可用题；
  - 本套纳入学生版的条目数；
  - 本套排除的文化题/非哲学题；
  - 本套未解决的 OCR、答案表或评分细则问题；
  - 下一套卷名称。
- `PROGRESS.md` 必须按套卷滚动更新，不准只写笼统“正在扫描”。

## 3. 套卷顺序

优先顺序：2026 -> 2025 -> 2024；同一年内优先 海淀、西城、东城、朝阳、丰台，再处理其他区。若原始目录结构不完全一致，先建立 `SOURCE_LEDGER.csv` 的真实目录索引，再按上述优先级挑下一套。

## 4. 证据硬门槛

- 主观题哲学链只认评分细则、评标/阅卷总结、讲评评分页或用户确认的评分文件。
- 普通参考答案只能做题意辅助，不能冒充评分细则。
- 高风险哲学词如 `辩证否定`、`量变质变`、`两点论与重点论`、`主次矛盾`、`矛盾主次方面`、`价值观导向`，必须有明确评分证据或等值表述，否则不得进学生版。
- 选择题必须有可靠客观答案表。没有答案表就只做题源记录，不生成错项/正确项结论。

## 5. PDF 与 OCR 工具要求

- 当前机器没有 `pdftoppm`。不要因为 `pdftoppm` 缺失就把 PDF 标为读不了。
- 优先使用 Python 的 `fitz` / PyMuPDF 读取 PDF 文本和渲染页面；必要时用系统预览、截图或已有 OCR 工具补读。
- 对扫描 PDF、图片题、漫画题，必须尝试渲染页面并保存审计截图或 OCR 记录；确实失败才标 `ocr-needed`。

## 6. 最终文档标准

- 最终学生版只按哲学原理方法论节点组织。
- 最终学生版不得出现文化章节，不得出现审计话术、来源路径、OCR/debug、`yes/pass/filled/correct_option_chain` 等状态字段。
- 最终要生成 Markdown 和 Word：
  - `outputs/2026北京高考政治哲学宝典---三年模拟全触发全链条_学生版.md`
  - `outputs/2026北京高考政治哲学宝典---三年模拟全触发全链条_学生版.docx`
- 用户已在本机登录并激活 Microsoft Word；最终 `.docx` 生成后必须用 Word 打开、保存并检查版式，不得只以脚本生成成功作为完成。
- 最终验收前必须用 `COVERAGE_MATRIX.csv`、`suite_reports/`、`audit/证据索引.csv` 交叉核验，证明不是漏题后拼接。

## 7. 不许停的条件

- 不要因为工作量大就只交计划。
- 不要因为某套卷有 OCR 或细则问题就整轮停止；记录边界后继续下一套。
- 只有当 `FINAL_ACCEPTANCE_REPORT.md` 写明哲学线已处理完、最终学生版和审计文件都存在、硬样本验收完成后，才可以说任务完成。
