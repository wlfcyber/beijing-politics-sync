### 2.2 框架颗粒度

注意：不能再把“物质决定意识”和“意识能动作用/意识反作用”混写成一个大桶。若细则只支持“物质决定意识/必要性”，学生版只能写客观现实如何决定认识和需要；若题目另有人的主动判断、价值辨析、能动思考，再单独放入“主观能动性 / 意识的能动作用”。

辩证法节点：

### 2.3 单条内容顺序

每一条必须按学生看到题时的思维顺序写：

1. `材料触发点`：至少一个完整句子，指出材料中哪句话、哪个词、哪个关系、哪个主体行动触发该原理。
2. `设问`：主观题必须抄全设问，但设问栏只能放真正的题目问法；材料正文不得混入设问栏，材料只放在“材料触发点”里。若原始字段写成“材料与设问”，必须切出最后的完整问句；评价型设问可连同“有人说/有人认为/这一观点”等被评价对象一起保留。
3. `为什么能想到`：用知识逻辑解释材料和原理的关系，不要写“因为细则提到了所以能放进来”。
4. `答案落点`：必须写成能回答本题设问的答案句或答案方向，不能写“要回应设问”“服务设问”这种空话。

`为什么能想到` 必须从材料里的具体词句、例子、主体行动或关系出发，至少点出一个学生看题时能抓住的材料信号；不能只写“材料强调……”“题目不是……”“这类题要……”这种抽象程序话。

`答案落点` 必须自然包含三段信息：材料给出的真实触发点、对应原理方法论、对设问的直接回答。优先写成学生能放进答案里的句子，例如“材料中……说明……；因为/这体现……原理，所以……”，不能写成“答案可以落在”“把材料事实同原理连接起来”这种元话术。

严禁伪触发点：不能为了安放某个原理，随便从材料里挑一个看似相关的词句当触发点。触发点必须是原题材料中真正支撑该原理的关键词、例子、关系或主体行动，并能解释清楚“为什么正是这句话触发这个原理”。若找不到真实触发点，必须回源查原题、细则或标为待补证据，不能硬编。

### 2.4 复合节点与细则子任务

- 复合标题不能自动展开成整套原理。单条正文必须按细则实际支持的半句来写。细则只支持“物质决定意识/必要性”时，不得补写“意识反作用于物质/意识反作用”。
- `为什么能想到` 和 `答案落点` 必须服从细则的子任务位置。若细则把某原理放在“必要性”里，解释链就要写“为什么客观现实决定必须这样做”；不能泛泛写原理本身，更不能把其他未出现原理塞进答案。
- 同一题出现多个明确原理时，要按框架分别入栏。不能因为一条已经进了 `物质决定意识`，就漏掉同题在 `主观能动性 / 意识的能动作用` 下的独立触发链。
- 用户人工核对到更完整细则时，若本地缓存只抽到简版角度清单，要把本地缓存标为“不完整细分”，按用户指出的完整细则口径修正，并在审计文件里注明边界。

### 2.5 排序

### 4.2 PDF

- 先用 PyMuPDF / `fitz` 提取文本、页码、图片块和页面渲染。
- 若 PDF 是扫描件或嵌图，先用 PyMuPDF 渲染成图片，再用视觉读取、截图、系统预览或其他可用方式补读。

### 4.3 Word / DOCX / DOC

- `.docx` 可用 Python `docx`、解压读取 `word/document.xml`、`textutil`、Quick Look 等方式读取。
- 老 `.doc` 或读取失败文件，先尝试 `textutil`、`strings`、Quick Look、系统预览、截图或其他可用方式，不得一次失败就放弃。
- Microsoft Word 现在已登录激活，可以正常使用。最终 Word 文档可先用 Python `docx` 生成，但必须再用 Microsoft Word 打开、保存并验收。
- Word 验收至少包括：文件能打开无激活/权限/损坏弹窗；首页只留标题和大署名；第二页留前言空位；正文无路径、debug、OCR、审计话术、状态字段；漫画/图片题图片能显示；中文字体和分页不明显错乱。
- 如果可行，用 Word 或系统能力导出一份 PDF/截图作为版式审计附件；若导出失败，必须在 `FINAL_ACCEPTANCE_REPORT.md` 说明已用 Word 打开检查但 PDF 导出边界是什么。

### 4.4 PPT / PPTX

- `.pptx` 可用 Python `pptx` 或解压读取 slide XML。
- 讲评 PPT、阅卷 PPT、评分页若含哲学细则或评分口径，属于重要评分证据；不能因为格式麻烦就跳过。
- 如文本抽取不完整，尝试渲染、截图或图片读取。

### 4.5 图片 / 扫描 / 表格 / 压缩包

- 图片题、扫描页、表格答案、答题卡式答案表都要尝试读取。
- 压缩包先解压到本轮工作目录下的审计或临时目录，记录来源，不要污染原始目录。
- 如果没有 OCR 引擎，仍要尝试通过渲染图、截图、手动视觉读取、系统能力或模型可见图片方式处理。

### 4.6 失败边界

只有在至少尝试两种合理路径后，才能把文件标为 `ocr-needed`、`conversion-needed`、`source-missing` 或 `blocked`。标边界时必须写清：

- 原文件路径；
- 已尝试工具；
- 失败表现；
- 对覆盖矩阵的影响；
