# Failure Patch Manifest

本文件只服务本轮“选必三《逻辑与思维》四线从0重跑”的补丁/Governor 检查，不生成学生版内容，不作为题目结论或证据源。

## 0. 批评转译

用户批评“内容质量没法比、没有穷尽、没有四线”，不是风格意见，而是三类可复现失败：

- `内容质量没法比`：产物没有达到哲学宝典式触发链质量，只停留在概念表、题型总述、制作说明或答案来源摘抄。
- `没有穷尽`：用条目数量或旧稿目录替代 source ledger、coverage matrix、suite/question reports，没有逐套逐题说明进入、排除、缺失与阻塞。
- `没有四线`：把本地 Codex 补丁、reviewer 口头审查或 pending 状态包装成 Codex + ClaudeCode + Claude Opus + GPT-5.5 Pro 已真实闭环。

## 1. 上一版会失败的具体机制

### F01. 用旧稿当隐性母版

失败机制：旧选必三稿件被当成内容来源、结构母版或“已穷尽”的证明，而不是只用于定位源材料、比对遗漏、暴露失败模式。

检查 gate：

- 任一学生版判断、框架条目、题型归类、答案句，必须能回指本轮源证据与本轮 lane 记录。
- 旧稿引用只能出现在审计/定位语境，且必须标明“旧稿仅用于定位/比对，不作证据”。

阻断 PASS：若学生版或最终验收把旧稿结论当作本轮证据，最终 PASS 必须阻断。

### F02. 概念表替代触发链宝典

失败机制：只按“科学思维、辩证思维、创新思维、推理类型”解释概念，缺少材料动作、真实设问、触发逻辑和卷面答案句。

检查 gate：

- 思维部分每个主条目必须具备：材料动作、总帽子、小方法、触发逻辑、答案句。
- 学生阅读顺序必须符合：材料触发点 -> 设问 -> 为什么能想到 -> 答案落点。
- 答案句必须是学生可直接写在卷面上的自然句，不得是“本题要写/可从……角度作答/细则要求”。

阻断 PASS：若思维部分主体仍是概念讲解、摘要或制作说明，最终 PASS 必须阻断。

### F03. 题源穷尽被条目数量冒充

失败机制：用“已有 N 条”“覆盖很多题”替代逐套逐题 coverage，没有 source inventory、coverage matrix、suite report 支撑；上一版“181 条”不能自动等于穷尽。

检查 gate：

- 必须存在本轮 source ledger，记录 source roots、套卷、题面、答案源、评分源、提取状态、缺失/转换/OCR 状态。
- 必须存在本轮 coverage matrix，逐套逐题标明 `included / excluded / missing / blocked` 及理由。
- 每个 excluded、missing、blocked 题必须有边界理由，不能从矩阵中消失。

阻断 PASS：未完成 source inventory 与 coverage matrix 前，不允许声称穷尽；若最终验收仍用数量代替覆盖证据，最终 PASS 必须阻断。

### F04. 同题多方法漏挂

失败机制：同一材料同时触发多个思维方法或给分层级，但只挂一个节点，导致学生少拿分。

检查 gate：

- 每道主观题必须检查是否存在多点位、总说/分说、层级评分、并列角度、同材料多方法。
- 一个题触发多个方法时，必须拆成多个可追溯节点，分别写材料动作、触发逻辑、答案句。

阻断 PASS：硬样本或抽检样本出现同题多方法漏挂，最终 PASS 必须阻断，直到全局同类漏项扫描完成。

### F05. 主观题设问污染

失败机制：把材料正文、背景信息、评标摘录塞进“完整设问”，学生无法识别真实作答任务。

检查 gate：

- `完整设问` 只能写题目要求学生作答的设问句。
- 材料内容必须进入材料触发点/材料动作，不得进入设问栏。

阻断 PASS：若成品中批量存在“设问栏混入材料”，最终 PASS 必须阻断。

### F06. 普通答案冒充评分细则

失败机制：参考答案、教师答案、讲评截图、分类汇编被升级为正式评分细则，导致 A-formal 伪闭环。

检查 gate：

- 主观题证据等级必须区分 `A-formal / A-support / missing`。
- 讲评材料有明确给分口径也只能作为 `A-support`，不得写成正式细则。
- 普通参考答案不得单独支撑主观题给分链。

阻断 PASS：若普通参考答案被冒充正式细则，或证据等级与来源类型不匹配，最终 PASS 必须阻断。

### F07. 选择题陷阱库反推答案

失败机制：只凭旧稿、解释或答案字母写选择题陷阱，没有同时核对题面、选项和可靠客观答案源。

检查 gate：

- 每道选择题必须有题面、完整选项、可靠客观答案源。
- 每个错项必须写清“为什么诱人、为什么错、属于哪类陷阱”。
- 客观答案源冲突时，先标冲突，不入选择题陷阱库。

阻断 PASS：若选择题模块缺完整选项或可靠答案源，或用答案字母反推陷阱，最终 PASS 必须阻断。

### F08. 推理部分只讲分类不挂题

失败机制：推理部分按三段论、假言、选言等写方法说明，但没有把所有题放到对应题型下面。

检查 gate：

- 推理部分必须单独成章。
- 每个题型下面必须挂具体题，写题型触发、规则口令、易错陷阱、解题步骤。
- 未归类题必须在 coverage matrix 中显示 `missing / blocked / other composite`，不能消失。

阻断 PASS：若推理部分只有题型讲解、没有具体题挂载，最终 PASS 必须阻断。

### F09. 模块边界过软

失败机制：看到“辩证、系统、创新、整体”等词就把必修四、选必一、选必二、政治与法治或纯形式逻辑题塞入选必三思维主链。

检查 gate：

- 必须用设问模块、评分来源、答题术语共同裁定归属。
- 纯形式逻辑/推理规则题不得并入思维主链；本轮推理部分单独归类。
- 跨书或边界题必须标 `C-boundary` 或进入推理章/边界记录。

阻断 PASS：若跨书题或纯形式逻辑题被硬塞进思维主链，最终 PASS 必须阻断。

### F10. 四线被本地流程冒充

失败机制：Codex 只当监工、ClaudeCode 没独立生产、Claude Opus/GPT-5.5 Pro 没真实看具体内容，却在验收里写成四线完成。

检查 gate：

- Codex lane 必须有生产证据，不只是 governor 文档。
- ClaudeCode lane B 必须独立从同一原始来源重跑，并留下题源矩阵、entries、suite reports、遗漏/冲突/阻塞。
- Claude Opus 必须在证据锁定后负责学生文稿成品化，不得新增无证据术语。
- GPT-5.5 Pro 必须真实看具体内容并做内容总审稿/压力测试。
- 若真实调用不可用，只能记录 `blocked_advisor` 或 `real_call_pending`，不得标 PASS。

阻断 PASS：任何一条外部/协作 lane 缺真实调用证据而被标为完成，最终 PASS 必须阻断。

### F11. 学生版混入审计话术

失败机制：学生版出现路径、OCR、debug、证据等级、pipeline 字段、评标、参考答案、pass/filled 等内部话术。

检查 gate：

- 学生版必须做污染词扫描。
- 来源、OCR、证据等级、模型分歧、缺失阻塞只能放审计文件。

阻断 PASS：学生版出现批量审计/管线话术，最终 PASS 必须阻断。

### F12. Word/PDF 验收被文件存在冒充

失败机制：脚本生成 docx 后即声称完成，没有 Word 原生打开、目录/页码/标题层级更新、PDF 或截图版式验收。

检查 gate：

- Word 可用时必须打开、更新域、保存，并导出 PDF 或截图验收。
- `soffice` 缺失只能记录 fallback，不能跳过 Word 原生验收。
- 最终验收报告必须记录实际验收方式和失败/修复项。

阻断 PASS：未完成 Word/PDF/截图版式验收，不得最终 PASS。

## 2. 必须阻断最终 PASS 的总 gate

以下任一项为真，最终验收只能是 `BLOCKED` 或 `FAIL`，不得写 `PASS`：

- G-PASS-01：未完成本轮 source inventory。
- G-PASS-02：未完成逐套逐题 coverage matrix，或 excluded/missing/blocked 题消失。
- G-PASS-03：ClaudeCode lane B 未独立生产，或没有可审查产物。
- G-PASS-04：Claude Opus lane 未真实运行且未标 `real_call_pending / blocked_advisor`。
- G-PASS-05：GPT-5.5 Pro gate 未真实运行且未标 `real_call_pending / blocked_advisor`。
- G-PASS-06：思维部分缺少 `材料动作 -> 总帽子 -> 小方法 -> 触发逻辑 -> 答案句` 链条。
- G-PASS-07：推理部分没有按题型挂具体题。
- G-PASS-08：普通参考答案冒充正式评分细则。
- G-PASS-09：选择题缺题面、完整选项或可靠客观答案源。
- G-PASS-10：跨书/纯形式逻辑题混入思维主链且未做边界裁定。
- G-PASS-11：学生版出现审计话术、路径、pipeline 字段或证据等级。
- G-PASS-12：Word/PDF/截图验收未完成。

## 3. Governor 使用要求

- Patcher 发现一个机制性失败时，必须触发同类全局扫描。
- Governor 不接受“已修部分样本”替代全局补丁。
- Governor 不接受“内容看起来多”替代源覆盖证据。
- Governor 不接受“模型建议过”替代真实四线调用记录。
- Governor 最终只能基于本轮证据、lane 产物、融合记录和验收记录裁定 PASS。
