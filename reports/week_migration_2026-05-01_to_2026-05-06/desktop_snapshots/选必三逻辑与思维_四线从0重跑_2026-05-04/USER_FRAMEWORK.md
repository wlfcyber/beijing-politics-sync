# User Framework

## 用户明确要求

- 内容质量必须对标已成功的哲学宝典。
- 思维部分要完全模仿哲学宝典：每一个思维和方法都像哲学的原理方法论一样处理。
- 不能只写框架或摘要，必须穷尽每一道题。
- 推理部分要先分类题型，再把所有题放在对应题型下面。
- 四线流程必须真实执行：Codex、ClaudeCode、Claude Opus、GPT-5.5 Pro。

## 思维部分框架口径

学生阅读顺序：

1. 材料触发点：材料中哪个动作、关系、趋势、冲突、方案、验证、抽象链触发本方法。
2. 设问：只写真实作答任务。
3. 为什么能想到：从材料动作到思维方法的知识逻辑。
4. 答案落点：学生能直接写的卷面答案句。

## 推理部分框架口径

题型分类优先：

- 三段论
- 假言推理
- 选言推理
- 联言推理
- 归纳推理与因果探求
- 类比推理
- 周延判断
- 换质换位
- 逻辑三律与概念规则
- 其他复合推理

每类下面必须放具体题，不能只讲方法。
