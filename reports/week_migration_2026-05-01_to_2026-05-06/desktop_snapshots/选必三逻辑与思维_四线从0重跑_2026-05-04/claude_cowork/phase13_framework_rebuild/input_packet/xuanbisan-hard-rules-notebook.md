# 飞哥选必三逻辑与思维硬性要求记事本

每次重跑、修订、生成 Word、接手 Codex/ClaudeCode 输出、或启动全量跑题前，必须先读本文件。任何新要求只能追加，不能覆盖旧要求。

本文件来自四条真实经验线：

- 必修四宝典：解决框架颗粒度、材料触发、设问、答案落点、学生版/审计版分离、同题多点漏挂等问题。
- 选必一：解决术语不是细则原词、材料触发像摘抄、答案句不像考生卷面作答、总说/分说漏项、后台话术混入正文等问题。
- 选必二：解决硬性要求记事本、证据分级、A/B/C/missing 边界、Patcher/Governor 否决、全量题源与课堂压缩断裂等问题。
- 选必三旧任务：解决“研究稿不等于课稿”、思维抽象与思维具体选择题误选、模块边界混入哲学/法律/国政经/形式逻辑等问题。

## 一、总目标

- 产物是给高三学生用的选必三《逻辑与思维》思维部分满分课，不是工作日志、审计报告、概念背诵表或逐题流水账。
- 内容必须围绕学生拿分链条组织：`材料动作 -> 总帽子 -> 小方法 -> 触发逻辑 -> 答案句`。
- 思维部分和必修四哲学的共性是“触发”：不是先背定义，而是看材料在逼学生调用哪种思维方式。
- 依据只能来自北京区级模拟题、试卷、答案、评分细则、阅卷/评标/讲评等可靠来源。
- 旧稿只能定位和审计，不能当证据。
- 任何套卷如果缺题面、缺答案表、缺正式细则、OCR 不可靠或模块边界不稳，必须明示，不准假装完成。

## 二、证据边界

- 主观题优先使用正式细则、评分参考、评标、阅卷细则、阅卷总结、用户确认可用的评分材料。
- 讲评 PPT 若有明确给分口径，只能标为 `A-support`，不得写成“正式细则要求”。
- 普通参考答案、教师版答案、题面信号、分类汇编、讲评截图不能冒充正式细则。
- 选择题必须同时有题面、选项、可靠客观答案源；不得只靠解释或旧稿反推答案。
- 分类汇编、专题汇编、Notability 汇编只作为补充来源，不是独立套卷；其中题目来自已扫描过的区卷时，不能重复计数为新题。
- `2026石景山期末` 已由用户逐题复核确认没有可用评分细则；除非用户明确提供新的评分细则来源，否则全模块排除。

## 三、运行时日期口径

- 不得把旧稿中“截至 2026-04-23/04-24，2026二模未开考”写成永久规则。
- 每次全量跑题前必须重新扫描本机 source roots。
- 若已出现 2026二模或后续新卷，按同一证据等级纳入裁定。
- 若没有找到新卷，记录“本轮 source roots 未发现”，不要写成客观考试事实。

## 四、模块边界

- 只处理选必三《逻辑与思维》的思维部分。
- 纯形式逻辑、推理规则识别、小项推理类型判断，不并入思维主观题主链；若同题后半明确要求思维方法应用，只收后半。
- 必修四哲学题即使出现 `辩证`、`系统`、`创新`、`整体` 等词，也不能自动并入选必三。
- 选必一、选必二、政治与法治、经济与社会等题，只能作为边界记录，不能硬塞进选必三主链。
- `辩证思维` 与必修四辩证法高度相似，必须用设问模块、评分来源和答题术语共同裁定归属。

## 五、主观题条目必须保留

1. `思维方法/给分点`：评分细则、评标或阅卷总结原词，或被来源明确支持的精确思维方法。
2. `完整设问`：只写题目要求学生作答的完整设问句；材料正文不得混入设问栏。
3. `细则位置`：写清套卷、题号、评分来源、点位、层级、分值或证据等级。
4. `来源`：写清年份、区、考试阶段、题号。
5. `材料动作`：材料里可被学生圈出的动作或关系，如调查、预测、协同、转型、联结、反转、抽象。
6. `总帽子`：科学思维、辩证思维、创新思维，或来源明确支持的总类。
7. `小方法`：客观性、预见性、可检验性、分析与综合、整体性、动态性、质量互变、辩证否定、超前思维、联想、发散、聚合、逆向、迁移和想象、思维抽象与思维具体等。
8. `触发逻辑`：解释为什么材料动作触发该思维方法。
9. `答案句`：写成考生答题纸上能直接落笔的一句话。
10. `框架落点`：进入用户现有框架的哪个节点。
11. `题型标签`：归入哪类可迁移题型。
12. `证据等级`：A-formal、A-support、B-choice-signal、C-boundary、missing。

## 六、材料触发硬规则

- 材料触发不是摘抄材料，也不是后台制作说明。
- 必须写清：题目追问什么、材料在做什么思维动作、为什么这组动作需要该思维方法。
- 必须至少点出一个学生看题时能抓住的材料信号：主体、动作、关系、顺序、冲突、趋势、方案、验证、抽象链等。
- 禁止出现：`材料明确写到`、`先写`、`要写`、`落到`、`采分点`、`v7漏了`、`细则要求`、`关键词最稳`、`本题需要`、`设问要求` 等懒表述或复查话。
- 禁止只写“材料强调科学思维”“题目不是单纯创新题”这种抽象程序话；必须解释材料事实和思维方法之间的关系。

## 七、答案句硬规则

- 答案句就是考生在答题纸上写的那一条答案。
- 必须按“思维方法/细则术语 + 本题材料事实 + 因果/作用/结论”写成卷面作答句。
- 可以短，但必须包含思维方法本身和本题材料事实。
- 不能只复述题目标题，不能只喊口号，不能只有“体现科学思维”“体现创新精神”这类空泛判断。
- 禁止写成制作说明，例如：`要落到……采分点`、`并结合材料说明`、`本题要写`、`细则要求`、`设问要求`。
- 不使用笨前缀 `材料中`。直接把材料事实写成卷面作答。

## 八、选择题硬规则

- 选择题可以依据试卷与可靠客观答案表分析稳定思维信号和错项陷阱。
- 不得用答案字母反推错项；必须看题面、选项和答案三者。
- 客观题答案源冲突时，先记冲突，不入选择题陷阱库。
- 每个错项必须写明为什么诱人、为什么错，以及属于哪类陷阱。
- 每条新增选择题陷阱必须标明来源套卷与题号。
- 选择题信号不能冒充主观题评分链。

## 九、思维抽象与思维具体专项

- `2026通州期末 Q11` 是永久硬样本。
- 口令必须固定为：`杂多现象 -> 抽出核心概念 -> 回到完整整体图景`。
- 看到 `合`、`和`、`融于一身`、`整体`，不能直接选系统整合、分析综合或联想。
- 若题目核心是认识过程从现象上升到本质、再由本质重建整体，优先判 `感性具体 -> 思维抽象 -> 思维具体`。
- 必须单独区分：
  - `分析与综合`：重在拆部分、合整体。
  - `系统整合`：重在多个要素协同形成机制。
  - `联想/类比推理`：重在由此及彼或用一个对象比另一个对象。
  - `发散与聚合`：重在方案生成和收束。
  - `思维抽象与思维具体`：重在认识深化链。

## 十、稳定触发链

- `调查/数据/验证/迭代` -> `科学思维` -> `客观性/可检验性`
- `趋势研判/未来布局/提前规划` -> `科学思维或超前思维` -> `预见性`
- `多主体/多环节/协同治理/一盘棋` -> `辩证思维` -> `分析与综合/整体性`
- `当前与长远/由低到高/渐进推进` -> `辩证思维` -> `动态性/质量互变`
- `旧产业/旧资源/旧模式升级` -> `辩证否定`
- `跨界组合/文旅融合/文创设计` -> `创新思维` -> `联想/迁移和想象`
- `多方案铺开再收束` -> `发散思维 -> 聚合思维`
- `关系、方向、状态反过来` -> `逆向思维`
- `杂多现象 -> 抽出核心概念 -> 回到完整整体图景` -> `思维抽象与思维具体`

## 十一、学生版与审计版分离

- 学生最终版不得出现来源路径、line id、file id、slide、OCR、debug、日志、英文内部编号、`/Users/...`、`C:\...` 等无效信息。
- 学生最终版不得出现 `yes`、`pass`、`filled`、`correct_option_chain`、`A-formal`、`B-choice-signal` 等状态字段作为正文。
- 学生最终版不得出现 `评标`、`参考答案`、`答案写`、`答案/补充`、`答案核`、`可从……角度作答` 等审计或答案来源话术。
- 来源、细则摘录、OCR 状态、证据边界、漏项扫描、模型分歧、证据等级都放审计文件。
- Word 交付前必须渲染检查版式；不能只生成 docx 不看。

## 十二、补丁复查硬规则

- 每轮修一个新要求时，要回头检查旧要求是否仍满足：证据边界、模块边界、完整设问、材料动作、触发逻辑、答案句、学生版干净度、总说/分说、同题多方法。
- 必须做同类漏项扫描：如果一个题型发现了“总帽子没有下沉”“答案句不卷面化”“材料触发是摘抄”，要全局检查同类条目。
- Patcher 必须专查：一个材料对应多个思维方法但只落一个框架点的问题。
- Governor 必须能否决：未分类题目、隐藏 OCR blocker、普通参考答案冒充细则、后台话术进入正文、进度文档假完成、边界题混入主链。

## 十二补一、四要件充分性硬闸口

- 只套用“材料触发点 / 设问 / 为什么能想到 / 答案落点”四个标题，不等于模仿哲学宝典。
- `设问` 必须是真实题目设问句；禁止用 `本题要求结合材料说明其体现的思维方法` 之类模板句冒充。
- `答案落点` 必须是学生能直接写在答题纸上的答案句，结构是 `思维方法/术语 + 本题材料事实 + 因果/作用/结论`。
- 禁止把 `答案落点` 写成制作说明、写作指令或审稿话，例如 `卷面要把...`、`先写...`、`要写...`、`本题需要...`、`设问要求...`、`采分点...`。
- 同一道题多节点挂载时，每个节点都要针对该节点重写四要件；不能把同一套“科学思维 + 辩证思维 + 创新思维”的总表复制到不同框架节点下面。
- 复杂题必须拆出 `材料信号 -> 总帽子 -> 小方法 -> 答案句` 的逐点对应；只列多个方法名属于空心条目。
- 选择题四要件不能只写“圈材料动作再看选项”；必须呈现题干信号、正确项理由、诱人错项理由和陷阱类型。
- 任何学生稿如果大量出现模板设问、制作说明式答案、同题多节点复制粘贴，直接判 `HARD_FAIL_BAD_WORD_CONTENT_GATE`，不得进入 Word/PDF/终稿。

## 十三、选必三试跑硬样本

新 skill 或新规则上线前，至少抽检：

- `2026顺义一模 Q19(2)`：看是否能拆出科学思维的客观性、预见性、可检验性。
- `2025海淀二模 Q20`：看是否能处理辩证思维复合题，区分分析与综合、整体性、动态性/质量互变、辩证否定。
- `2026朝阳期中 Q21(2)`：看是否能处理创新思维复合题，区分超前、联想、逆向、发散聚合。
- `2026通州期末 Q11`：看是否能处理思维抽象与思维具体选择题陷阱。
- `2026东城期末 Q17(2)`：看是否能把纯形式逻辑小问排除在思维主链外。

抽检失败时，不允许进入全量跑题。

## 十四、2026-05-05 纠偏：题量、全量扩容与排序

- 29 题、18 题、8 题这类 source-verified packet 只能算受控候选包，不能叫终稿、最终版、宝典成品，也不能进入 Word/PDF final gate。
- 近 60 套卷的整本选必三重跑，默认目标是全量穷尽有效题，而不是代表例题小册子。若正文题量显著低于 evidence pool 或 source inventory 预期，必须先做 coverage gap，不得继续美化 Word。
- 当前本轮硬样本：Phase05 evidence pool 已有 74 locked evidence rows；若正文只有 29 行，必须判 `SCOPE_TOO_NARROW`，至少处理 74 行中的可修复条目，并回扫 362-row control base/source roots 排查更多有效选必三思维/推理题。
- `same-type index only` 不能替代正文覆盖。若同类题能提供新的材料信号、推理形式、错项陷阱、评分链或可迁移表达，应转正文；若只保留索引，必须写明原因。
- `not represented` 行不得消失。每一行都必须有 `入正文 / 同类索引 / blocked / excluded` 的明确结论和证据理由。
- 推理部分按题型分类后，同类题要尽量全部放在对应题型下面，不得只放一个代表例。常见家庭至少包括三段论、充分条件假言推理、必要条件假言推理、选言推理、归纳推理、类比推理、概念/判断/逻辑规则/逻辑错误等。
- 最终学生文档排序硬规则：主观题在前，选择题在后；每类内部按区排序为海淀、西城、东城、朝阳、丰台、其他区；同一区内部按时间倒序为 2026、2025、2024；同年同区再按考试阶段和题号稳定排序。
- 思维部分继续模仿哲学宝典：每个思维方法当作“原理/方法论”处理，必须讲清材料怎么看、触发哪个方法、为什么触发、答案句怎么落、易错项怎么避。

## 十五、实时纠偏入小本本

- 当用户指出“你之前没有遵守”的新要求时，Codex 必须立即把纠偏写入本文件或当前运行的 `00_飞哥选必三逻辑与思维硬性要求记事本.md`，不能只在聊天里道歉。
- 每条纠偏至少写三件事：之前漏遵守了什么、新硬规则是什么、后续验收怎么查。
- 写完小本本后，再更新当前 run 的 `PROGRESS`、`FINAL_ACCEPTANCE_REPORT`、`MODEL_ADVICE_LOG` 或对应控制文件，保证自动唤醒和新线程不会继续沿用旧口径。
- 对外部 GPT/Claude 审查也要同步最新纠偏。给 GPT/Claude 的 prompt 必须说明新规则，不得让外部模型继续审旧的小包或旧排序。

## 十六、2026-05-05 GPT Phase12 指挥稿永久补丁

- 如果 GPT/用户指出某个 seed 条目机械四要件 PASS 但内容可能漏角度或串线，机械 PASS 不得作为正文安全证明。必须回源核验题面和细则。
- `2024 海淀二模第17题第(1)问` 是新的 source/scope 硬样本。原题设问是“结合材料，说明此次时间利用调查是如何体现科学思维的”，不能把设问改写成科学思维、创新思维、辩证思维三模块并列。
- 该题 028 细则明确支持“全面、真实、准确/客观性”“思路新、方法新/探索性”“分阶段实施、任务明确/整体安排”。正文可写这些来源支持的落点，但必须说明它们是科学思维设问下的细则点，不得冒充原题三模块设问。
- 以后遇到“旧控制稿三角度”和“原题单角度”冲突，优先以原题设问和正式细则裁定；旧控制稿只能作为冲突线索，不能压过原卷。
- Phase12 工作顺序永久化：先冻结小包不终稿，再 74-row 全裁决，再 45 non-body 修源/修推理形式，再 362 control base 回扫，再扩正文和双索引，最后才考虑 Word。

## 十七、选择题完整选项显示硬规则

- 最终学生版选择题必须统一显示完整四个选项，或完整 ①②③④ 四个选项单位。
- 不能有的选择题只写答案与错项陷阱，有的选择题显示完整选项；版式和信息密度必须统一。
- 选择题最终清稿前必须跑选项可见性审计；凡是 `answer_and_trap_visible_but_full_options_missing` 或类似状态，必须回源补齐选项后才能进入 Word。
- 选择题仍要保留正确项理由和错项陷阱，不能为了补选项而删掉解题迁移价值。

## 十八、2026-05-05 Phase12 MUST_FIX_CONTENT 外审补丁

- 77 条 review-only 扩容稿解决了 29 题过少问题，但若外审 verdict=`MUST_FIX_CONTENT`，只能进入补丁审查轮；不得生成 Word/PDF，不得写 final/PASS/终稿/最终稿/宝典成品。

- 需要真实 GPT-5.5 Pro 网页审查时，Codex 默认必须自行上传 packet、自行提交 prompt、自行等待完成并捕获 raw/digest，不能只把路径丢给用户让用户自己找包。

## 二十一、2026-05-06 框架版结构硬规则

- 用户明确否定按“主观题在前、选择题在后，再按地区时间逐题展开”的正文结构；这种排序只能作为审计和检索依据，不能作为宝典最终正文主结构。
- 选必三思维部分必须完全模仿哲学宝典的框架组织法：先写思维类型，再写思维特点/小方法，再把对应模拟题挂在该节点下面。
- 每个思维类型和小方法节点都必须有固定分析流程，至少包括：`材料怎么看`、`该写哪个思维方法`、`为什么触发`、`答案句怎么落`、`易错项怎么避`。
- 科学思维必须下沉到三性与本题来源支持的专项节点：`客观性`、`预见性`、`可检验性`，以及有来源支持时的`探索性与方法更新`、`整体安排`；不得只在正文前列来源清单。
- 创新思维必须下沉到三新和具体方法：`思路新/方法新/结果新`、`联想思维`、`发散思维与聚合思维`、`逆向思维`、`超前思维`、`改变条件/建立新联系`等；每个节点下面要配题。
- 辩证思维必须下沉到`整体性`、`动态性`、`分析与综合`、`质量互变`、`辩证否定`等节点；每个节点下面要配题。
- 认识发展历程必须下沉到`感性具体`、`思维抽象`、`思维具体`，并用`杂多现象 -> 抽出核心概念 -> 回到整体图景`的固定流程带题。
- 推理部分也必须按题型框架组织：题型/规则口令/常见陷阱/解题动作/对应模拟题；不得按试卷题号流水账。
- 同一题若能训练多个框架节点，可以像哲学宝典一样重复挂载，但每次挂载都要服务当前节点，不得只复制题目来源清单。
- 后续验收必须检查：最终学生正文的二级/三级标题是否是框架节点而非地区时间；每个核心节点下面是否至少有模拟题；是否还存在大段“按题目单位展开”的主体结构。
