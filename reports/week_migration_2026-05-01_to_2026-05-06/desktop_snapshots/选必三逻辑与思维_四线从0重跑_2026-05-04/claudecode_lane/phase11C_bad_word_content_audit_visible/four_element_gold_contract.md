# 选必三《逻辑与思维》四要件 Gold Contract（学生版硬合同）

本文件用于约束 Phase11C 之后所有重建/扩展/Opus 改写。它不是建议，是返工硬约束。任何条目如果违反本合同任意一条，必须打回 `HARD_FAIL_FOUR_ELEMENT_GATE`，不得进入 Word/PDF/终稿。

合同同时与 `~/.codex/skills/feige-politics-garden-xuanbisan/references/xuanbisan-hard-rules-notebook.md` 的第六、七、八、九、十一、十二、十二补一、十三条对齐。本合同补强可在 ClaudeCode/Codex/Opus 重写时直接对照打勾的颗粒度。

## 一、四要件标题与角色

每条条目必须有且仅有以下四个标题：

1. `【材料触发点】`：考生看题时能在材料里圈出来的具体信号。
2. `【设问】`：原题题面。
3. `【为什么能想到】`：从信号到方法的知识链条解释。
4. `【答案落点】`：考生在答题纸上能写出的卷面答案句（或选择题正确选项 + 错项分析 + 陷阱类型）。

不允许将这四个标题改名、合并、加副标题、或把「为什么能想到」与「答案落点」合并成一段。

## 二、`【材料触发点】` 硬规则

- 必须包含至少一个学生看材料时能圈出来的具体信号；信号类型可以是：主体、动作、关系、顺序、冲突、趋势、方案、验证、抽象链、数据、引文。
- 至少包含一个原材料关键词或短语，建议用 `“”` 中文双引号摘录；摘录长度不超过 30 个汉字，避免变成材料整段拷贝。
- 同一道题在不同框架节点出现时，材料触发点必须按本节点关注的信号子集重写；不允许节点之间复制粘贴。
- 禁止仅以话题名概括，例如不允许只写「围绕调研活动提建议，材料同时涉及方案设计、方法选择、数据处理、因果判断」这种「topical_label_only」式的概括。
- 不得出现：`材料明确写到`、`材料强调`、`材料是说`、`先写`、`要写`、`本题需要`、`设问要求`、`卷面要把`、`采分点`、`细则要求`、`v7漏了`、`关键词最稳`。
- 出现源信息缺失/OCR 不可靠时，整条标记 `BLOCKED_NEEDS_SOURCE`，不得猜写。

## 三、`【设问】` 硬规则

- 必须是真实题目设问句的逐字摘录。允许做轻微清洗（去 OCR 残字、补正确标点），但不得改动语义、动作词与限定词。
- 必须包含原题中的关键动作词，例如：`分析说明`、`评析`、`运用××思维`、`补全推理`、`判断推理是否成立并说明理由`、`选出错误的是`、`谈谈对……的认识`。
- 子问需注明 `第N题第(M)问` 或 `Q19(2)` 这类完整定位。
- 选择题的设问要把「正问/逆问」「单选/多选/组合」标清楚。例如 `下列分析正确的是` 与 `下列分析错误的是` 必须保留正反面差别。
- 多节点同题：每个节点对应原题的同一句设问，但**节点说明**写在 `【为什么能想到】` 起首，由设问衔接到本节点的视角。设问栏本身不允许合并不同小问。
- 禁止使用以下模板假设问：
  - `本题要求结合材料说明其体现的思维方法，或运用相关思维方法分析、评价、提出做法。`
  - `本题要求依据材料判断正确选项，并排除把思维方法乱套、夸大或错位的选项。`
  - `本题要求识别或运用“XX”规则，判断推理是否成立、错在何处，或补全合乎规则的推理。`
- 题源未确认或子问号不清时，整条标记 `BLOCKED_NEEDS_SOURCE`。

## 四、`【为什么能想到】` 硬规则

- 必须依次解释：本节点视角下材料信号是什么 → 触发哪个总帽子 → 落到哪个小方法 → 为什么是这个小方法而不是相邻方法。
- 必须能让一个零基础学生明白「为什么这条材料一定要落到这种思维方法」，而不是只罗列方法名。
- 多节点同题必须节点专属：进入 `分析与综合` 节点就只解释多主体协同信号 → 整体性 / 分析综合；进入 `辩证否定` 节点就只解释「破旧立新、扬弃改造」信号 → 辩证否定。不得复制粘贴一份「全帽子表」。
- 复合题（题面要求多种思维方法并列时）必须做逐点对应：`材料信号A → 帽子A → 小方法A`、`材料信号B → 帽子B → 小方法B`。每点必须 1-2 句，不能只列方法名。
- 选择题需说明：正确选项的材料信号映射、每个错项为什么诱人 + 为什么错（属于第十二补一所要求的「正确项理由 + 诱人错项理由 + 陷阱类型」）。
- 推理题需说明：识别出哪种判断/推理类型，为什么是这种类型而不是相邻类型；与边界（必修四 / 选必一 / 选必二 / 形式逻辑边界）发生混淆时一律展开。

## 五、`【答案落点】` 硬规则（最严）

- 必须是考生在答题纸上能直接落笔写出的句子，结构 `思维方法 / 术语 + 本题材料事实 + 因果 / 作用 / 结论`。
- 至少 1 句、不超过 5 句；句中包含本题真实的具体名词（人、物、机构、产业、场景、数据），不能只写抽象方法名。
- 多节点同题：每个节点要写一段卷面句，针对本节点的小方法把材料事实写到位。
- 选择题：写明正确选项的字母 + 选项内容引用 + 入选理由 + 高频错项理由 + 陷阱类型；不得仅以「先圈材料动作再看选项」结尾。
- 推理题：写明本题真实结论或补全的推理，例如 `这是充分条件假言推理的否定后件式，结构有效，结论成立`；不得仅以「先翻译逻辑结构、套规则」结尾。
- 答案落点不得复制 `【为什么能想到】` 段落最后一句；二者必须存在「教学解释 → 卷面作答」的递进。

### 五·禁止短语集（forbidden phrases）

任何 `【答案落点】`、`【设问】`、`【材料触发点】` 中只要出现以下任一短语，整条直接 fail：

- `本题要求结合材料说明其体现的思维方法`
- `本题要求依据材料判断正确选项`
- `本题要求识别或运用`
- `卷面要把`
- `先写`
- `要写`
- `本题需要`
- `设问要求`
- `采分点`
- `细则要求`
- `落到`
- `v7漏了`
- `关键词最稳`
- `因此做选择题时先圈材料动作，再看选项是否把方法、主体、关系或作用说准。`
- `先翻译逻辑结构，再套本题型最小规则，最后判断能否必然推出。`
- `按……五步写`、`三步落笔`（「N 步走」式制作指令）
- `参考答案`、`答案写`、`评标`、`细则要求`、`可从……角度作答`、`PASS`、`yes`、`A-formal`、`B-choice-signal` 等审计字段。

## 六、多节点挂载硬规则

- 同一道题挂载到多个框架节点时，必须为每个节点写出独立四要件，但共享同一份原题设问。
- 节点专属四要件由 `【为什么能想到】` 第一句声明，例如：`本节点关注分析与综合视角：材料的多主体协同动作是……`。
- 不允许把「科学思维 + 辩证思维 + 创新思维」整张表复制到每一个节点；每个节点只解释自己负责的那一支。
- 如果一道复合题在某节点其实只是擦边出现，必须做出取舍：
  - 该节点价值高 → 写独立四要件；
  - 该节点价值低 → 不挂载，仅在「同类题索引」里出现。

## 七、证据等级与未确认题源

- 学生稿条目仅允许使用 `A-formal`、`A-support` 来源；`B-choice-signal`、`C-boundary`、`missing` 一律不得直接出现在学生稿里。
- 选择题如答案表无法确认或与细则冲突，整条 `BLOCKED_NEEDS_SOURCE`；不得用题面信号反推答案。
- 推理题如题号、题面或答案表存在 OCR 残缺，整条 `BLOCKED_NEEDS_SOURCE`；不得用通用收尾掩盖。
- `2026石景山期末` 已由用户明确排除，全模块不得纳入。

## 八、学生稿与审计稿分离

- 学生稿不得出现 `path`、`line id`、`file id`、`OCR`、`debug`、`/Users/...`、`C:\...`、`yes`、`pass`、`A-formal` 等审计字段。
- 学生稿不得出现「高频陷阱」「分析步骤①②③」「补充说明」等审计版栏目；选择题/推理题展开陷阱时使用学生友好语句，例如 `易错项是 A`、`常见错法`、`陷阱类型：思维抽象与思维具体颠倒`。
- 学生稿不得出现 `【材料与设问】` 这种合并字段；材料与设问必须各自独立。

## 九、自检清单（每条上线前对照打勾）

| # | 检查项 | 通过标准 |
|---|---|---|
| 1 | `【材料触发点】` 是否含具体材料信号 | 至少一个原材料引文短语，并指明主体/动作/关系/趋势之一 |
| 2 | `【设问】` 是否原题逐字 | 含真实动作词、子问号；无禁止模板 |
| 3 | `【为什么能想到】` 是否节点专属 | 第一句声明节点视角；解释为什么是该小方法 |
| 4 | `【答案落点】` 是否卷面句 | 包含方法术语 + 本题材料事实 + 因果/作用/结论；不含禁止短语 |
| 5 | 多节点 | 每个节点四要件独立、不复制粘贴 |
| 6 | 复合题 | 逐点对应 `材料信号 → 帽子 → 小方法 → 答案句` |
| 7 | 选择题 | 含正项理由 + 错项分析 + 陷阱类型 |
| 8 | 推理题 | 含真实判断/推理类型 + 真实结论 |
| 9 | 证据等级 | A-formal 或 A-support；其他状态走 `BLOCKED_NEEDS_SOURCE` |
| 10 | 学生稿干净度 | 无审计字段、路径、ID、英文管线词 |

任意一项不通过 → `HARD_FAIL_FOUR_ELEMENT_GATE`，回炉。
