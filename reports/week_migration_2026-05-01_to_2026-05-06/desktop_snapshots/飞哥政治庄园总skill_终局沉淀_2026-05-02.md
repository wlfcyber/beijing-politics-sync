## 本次沉淀进总 skill 的核心规则

### 1. 先读小本本

- 每次重开、重跑、生成 Word、交给 Claude Code 或接手别人输出前，先找并读对应模块的小本本。
- 小本本优先级高于旧成品和模型记忆；新要求只能追加，不能覆盖旧要求。
- 必修四、选必一、选必二已有本地小本本，路径变化时先在北京高考政治工作区搜索。

### 2. 旧线作废就真作废

- 用户说“从0开始”“全部作废”“不继承旧结论”，就从原始三年模拟题、答案、细则、讲评等来源重新做。
- 旧文件最多用于定位、对照、审计，不得当证据，不得继承结论。
- 如果用户后续明确允许“当前本轮从0产物可用于收尾返修”，只能使用本轮产物，不能偷用旧线。

### 3. 证据边界不能混

- 评分细则、评标、阅卷总结、正式评分规则优先。
- 普通参考答案不能冒充细则；除非用户明确确认可用。
- 缺证据就说缺证据，不硬塞，不用“等角度”强行扩展。
- 2026石景山期末按用户复核结果全模块排除，除非以后用户提供新的可用细则。

### 4. 任何格式源都要想办法处理

- PDF、Word、PPT、图片、扫描件、漫画、表格都不能因为单个工具缺失就放弃。
- 缓存优先但不是缓存唯一。缓存丢失选项、漫画、表格、页码、版式或细则结构时，必须回源渲染、OCR、视觉读取或用 Office 工具处理。

### 5. 学生版和审计版彻底分离

- 学生文档是给学生看的讲义，不是日志、审计报告、逐题流水账。
- 学生版不得出现路径、line id、file id、OCR/debug、模型状态、`pass/filled/yes/correct_option_chain` 等字段。
- `评标`、`参考答案`、`细则要求`、`可从……角度作答` 等只能在审计文件里，不能冒充答案落点或材料触发。

### 6. 必修四哲学文档的硬结构

- 只做哲学时，文化线不能混进来。
- 正文按飞哥原理方法论框架组织，不按套卷顺序流水账排列。
- 每条主观题按学生思维顺序写：`材料触发点 -> 设问 -> 为什么能想到 -> 答案落点`。
- `设问` 只放真正任务句，不能把整段材料塞进去。
- `为什么能想到` 必须讲材料信号到原理的知识逻辑。
- `答案落点` 必须是学生能写进卷面的自然答案句或方向，不写元话术。

### 7. 选择题、漫画题、引号和术语

- 选择题若作为学生复习材料，四个选项要统一显示。
- 选项建议逐项换行，页面上尽量保持同组选项不被撕裂。
- 漫画题尽量嵌入真实漫画图；每次重建 Word 后都要检查图片是否还在。
- 中文材料引用统一使用 `“”`，不要混用英文单引号或弯单引号。
- 学生不自然或容易误解的词，比如用户指出过的“配伍”，要换成自然表述。

### 8. Word 才是真交付

- `.docx` 不是脚本生成成功就算完成。
- 最终 Word 要有页码、可点击目录、页码引用、干净目录样式、稳定标题层级。
- 标题可以适度蓝色美化，但目录正文项不能被整片染蓝。
- Microsoft Word 可用时，要用 Word 打开、保存并验收。
- 可行时导出 PDF 或渲染页面，检查目录、分页、选项、漫画、标签颜色、水印等。

### 9. “不要动内容”的正确理解

- 用户说“不要动内容”，指不要改正文知识性内容、原理判断、答案逻辑。
- 仍然要修格式问题：目录颜色、选项缺失、设问标签颜色、图片丢失、分页、水印、页码、可点击目录。
- 格式修订后尽量做段落文本 hash 或数量对比，证明知识内容未变。

### 10. 水印规则

- 用户要水印时，正文部分加低干扰水印，逐步按用户反馈调深。
- 每次加深后要检查不遮挡正文、不破坏页码目录、不删图片、不改内容。
- 对“不可去除”要诚实：可编辑 DOCX 里不存在绝对不可删除；若要更强防删，分发扁平化 PDF 更可靠。

## 本次我被纠正后必须记住的反思

- 不能拿另一个“看起来更高级”的版本重建，导致用户满意版的内容、目录或图片被改坏。用户满意后，后续都应在满意版上做窄修。
- 不能把 Word 美化理解成大面积上色。目录项全蓝就是失败，必须视觉自查。
- 不能只看 XML 觉得图片还在，要渲染或导出 PDF 看漫画是否真的显示。
- 不能用过宽规则去删“重复题号”或改选择题格式；格式脚本必须限定题型和段落模式。
- 不能把“不要动内容”误解成“不修格式”。审稿人的职责就是在不改知识内容的前提下修正格式。
- 不能把 `python-docx` 成功当作 Word 成功。Word 打开保存、目录域、页码、PDF 渲染都属于最终验收。
- 不能因为缓存里没有四个选项就默认题目本来不需要选项；要回源补齐。
- 不能让学生版混进审计语、细则语、内部状态语。学生版必须像真正课堂讲义。
- 不能让 Claude Code 自己跑偏或停在半成品；监督者要按进度、日志、覆盖表、最终文档持续叫醒和纠偏。
