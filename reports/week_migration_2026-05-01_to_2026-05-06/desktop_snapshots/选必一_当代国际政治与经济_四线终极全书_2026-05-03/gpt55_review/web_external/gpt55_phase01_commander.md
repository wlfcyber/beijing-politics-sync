## 2. 下一阶段第一优先级

**第一优先级：把 11 个高优先题从候选源状态推进到 evidence locked 状态。**

不要急着扩大全书条目数量。下一阶段先把每一道优先题做成可审计的证据卡，至少包括：

1. 原题定位状态：是否找到本地题面源，是否存在图片、表格、PPT、PDF、讲评稿。
2. 评分来源边界：官方评分、阅卷细则、讲评参考、教师整理、普通参考答案要分层标注。
3. 材料触发：只写可由本地材料支撑的触发词和触发逻辑。
4. 可入学生版术语：必须剔除后台语、debug 语、reference-only 术语。
5. 六桶归属：主桶、交叉桶、禁止误挂桶。
6. 证据等级：confirmed、probable、needs visual check、reference only、blocked。
7. 合并建议：可合并、不可合并、只能同题内合并、只能作为替代表述。

阶段目标不是产出漂亮文档，阶段目标是建立 **证据能追、边界能守、合并能解释、学生不被误导** 的底座。

## 3. Codex 生产线任务

Codex 作为本地总控和本地证据裁判，建议下一阶段按下面顺序推进。

### 3.1 Codex A 内部 worker

先只做 11 个高优先题，不扩展到全量 1372 源。

每题输出一张 evidence card，字段建议固定为：

| 字段       | 要求                                             |
| -------- | ---------------------------------------------- |
| 题号       | 统一到 年份、区、考试、题号、小问                              |
| 源文件状态    | found、multiple candidates、needs visual、blocked |
| 题面来源     | 只给本地可追踪索引，不进入学生版                               |
| 评分来源类型   | 官方评分、阅卷细则、讲评参考、教师整理、普通参考                       |
| 本题核心设问   | 只保留设问限定，不混材料概述                                 |
| 材料触发     | 关键词、句群、图表信息、图片信息分开                             |
| 可入学生版术语  | 必须经过六桶和禁词过滤                                    |
| 不可入学生版内容 | 路径、日志、模型聊天、后台语、reference-only                  |
| 争议点      | 证据不足、视觉待核、来源边界不清、桶位争议                          |
| 建议证据等级   | worker 初判，最终由 Codex 裁判                         |

### 3.2 Codex A regulator

监管者优先查三件事：

第一，是否把参考答案伪装成评分细则。
第二，是否把材料概述写成完整设问。
第三，是否把中国外交术语错挂到政治多极化桶。

监管者每题给出 PASS、FIX、BLOCK 三档。BLOCK 题不得进入合并池。

### 3.3 Codex A patcher

补丁者不要急着润色答案，先做风险补丁。

重点补丁对象：

| 风险             | 补丁方向                        |
| -------------- | --------------------------- |
| 2025海淀期中Q16(2) | 区分知识产权合规风险、贸易摩擦、国际组织机制      |
| 2025海淀期中Q21(2) | 必须视觉复核图片或表格细则，禁止 cache-only |
| 2026西城期末Q20    | 旧线视觉 blocker 必须复判           |
| 经济全球化方向        | 保留开放、包容、普惠、平衡、共赢完整表达        |
| 时代主题           | 不拆成碎片化多点，优先整句表达和平与发展仍是时代主题  |
| 人类命运共同体        | 主归中国外交或中国桶，按题目可交叉政治多极化      |
| 新型国际关系         | 标注中国外交属性和国际关系方向，避免单独泛化      |

### 3.4 Codex A automation detector

自动化检测者建议跑四类扫描：

1. 学生版禁入词扫描：路径、screen、debug、audit、progress、模型聊天、reference-only、2026石景山期末、后台阅卷语。
2. 六桶冲突扫描：同一术语是否被多个主桶收录且无交叉说明。
3. 证据等级扫描：confirmed 术语是否缺少本地来源定位。
4. cache-only 扫描：凡是非文本材料相关条目，若只来自缓存摘要，直接降级为 needs visual check。

### 3.5 Codex A decision maker

决策者本轮只做三个裁决：

第一，11 题哪些可以进入首批合并池。
第二，哪些术语只能保留为本题样例，不能全书迁移。
第三，哪些内容必须等 ClaudeCode B 独立线结果再合并。

## 5. Claude Opus 与 GPT 内容审稿检查点

### 5.1 Claude Opus 教学顾问检查点

Claude Opus 只适合做教学表达和学生迁移审稿，不承担本地证据裁判。

建议让 Opus 看脱敏后的 evidence cards 和学生版候选稿，重点检查：

1. 学生能不能从材料触发到术语。
2. 术语有没有堆砌感。
3. 答案句是否像高三学生能写出的高分卷面答案。
4. 是否存在过度官方报告腔。
5. 是否把同类术语合并过度，导致学生无法区分。
6. 是否缺少禁用条件，导致学生乱套。

Opus 结果只作为教学建议，由 Codex 结合本地证据裁决是否采纳。

### 5.2 GPT 内容审稿检查点

GPT 侧重点放在结构与风险，不碰本地证据真假裁判。

建议 GPT 每轮审稿固定查七项：

1. **设问纯度**：完整设问栏是否混入材料概述、教师转述、答案方向。
2. **证据边界**：评分细则、参考答案、讲评、教师整理是否被混称。
3. **六桶归属**：中国外交术语是否错挂政治多极化，联合国术语是否泛化到所有国际组织。
4. **迁移安全**：术语有没有适用条件和禁用条件。
5. **答案成型**：是否只有术语卡，没有整题答案结构。
6. **学生语言**：是否出现后台语、审稿语、debug 语、模型语。
7. **合并风险**：经济全球化方向、时代主题、新型国际关系、人类命运共同体是否被误合并。

## 6. Governor 必查项

Governor round2 不能只看文件是否生成，必须做实质性 veto。

建议必查项如下：

### 6.1 源证据必查

1. 11 个优先题是否都有 source ledger 条目。
2. 每题是否标注源类型和证据等级。
3. 非文本材料是否完成视觉复核或明确降级。
4. 旧索引是否只用于定位，没有直接用于定论。
5. needs_source_recheck 是否逐条处理，未处理不得转 confirmed。

### 6.2 评分边界必查

1. 是否把普通参考答案写成评分细则。
2. 是否把讲评 PPT 的答案层当成官方评分。
3. 是否标出必答点、替代点、任意点、参考表达之间的差异。
4. 是否避免在学生版出现后台阅卷表述。

### 6.3 六桶必查

重点查四个高危区域：

| 高危术语    | Governor 检查口径         |
| ------- | --------------------- |
| 人类命运共同体 | 主归中国外交或中国桶，可按题意交叉全球治理 |
| 正确义利观   | 主归中国外交，不能泛化为政治多极化普通点  |
| 新型国际关系  | 标注合作共赢与中国外交主张属性       |
| 共商共建共享  | 区分全球治理观、开放合作机制、具体合作方式 |

### 6.4 合并规则必查

同类项合并必须满足三条件：

1. 教材语义同源或评分可替代。
2. 材料触发高度一致。
3. 学生迁移不会被误导。

只要有一个条件不满足，就只能做交叉提示，不能硬合并。

### 6.5 学生版禁入必查

学生版不得出现本地路径、模型聊天、debug、audit、progress、reference-only、后台阅卷语、2026石景山期末、以及本轮尚未证实的来源标签。

## 7. Confucius/学生迁移验证点

Confucius 验证目标应从好不好看改成学生能不能迁移。

建议设置五个小测。

### 7.1 盲审设问测

给学生只看题干设问，不看答案，让学生判断：

1. 题型是什么。
2. 主体是谁。
3. 知识范围是什么。
4. 应答角度是原因、意义、措施、评析还是关系处理。
5. 哪些内容属于材料信息，哪些属于设问限定。

只要学生分不清设问和材料概述，该条目要返修。

### 7.2 材料触发测

给学生材料关键词，让学生选术语。

例如：

| 材料关键词             | 学生应能触发                 |
| ----------------- | ---------------------- |
| 逆全球化、治理赤字、有效治理期待  | 时代主题、全球治理方向            |
| 全球南方、农业增产、减本、育才   | 共同利益、南南合作、普惠平衡共赢       |
| 自贸协定、免签、人才流商品流资金流 | 高水平开放、贸易投资便利化、两个市场两种资源 |
| 联合国宪章、主权平等、国际法治   | 联合国框架和国际法治             |
| 中国方案、公共产品、大国担当    | 中国智慧中国方案、人类命运共同体       |

### 7.3 禁用条件测

每个高频术语至少补一句什么时候不要用。

例如：

| 术语        | 禁用提醒                       |
| --------- | -------------------------- |
| 贸易摩擦      | 只有一般海外商标纠纷时，不要直接写成贸易摩擦     |
| 联合国宪章     | 没有联合国、宪章、国际法治、主权平等等材料信号时慎用 |
| 经济全球化正确方向 | 不能只写普惠、平衡、共赢而丢掉开放、包容       |
| 人类命运共同体   | 不能当成所有国际关系题的万能结尾           |
| 国际竞争实质    | 没有科技、经济实力、综合国力、创新能力时慎用     |

### 7.4 整题成文测

学生不能只背术语卡。每道优先题至少要有一版整题结构：

1. 题目问什么。
2. 材料分几层。
3. 每层对应什么教材术语。
4. 答题卡上写几段。
5. 哪些句子是主干，哪些句子是替换。

### 7.5 人味与卷面测

学生版答案句应满足：

1. 像高三学生可写进答题卡。
2. 每点 40 到 70 字为宜。
3. 有教材术语，有材料对应，有结论。
4. 不写宏大空话。
5. 不出现后台制作痕迹。

## 8. 最大风险和具体防错动作

| 最大风险                      | 具体表现                  | 防错动作                                   |
| ------------------------- | --------------------- | -------------------------------------- |
| cache-first 变成 cache-only | PPT、PDF、图片、表格只靠缓存摘要定论 | 非文本来源统一设 needs visual check，完成视觉复核后再升级 |
| 参考答案冒充评分细则                | 学生以为所有表述都是正式评分依据      | source ledger 强制标注官方评分、讲评参考、教师整理、普通参考  |
| 完整设问混入材料概述                | 学生误以为答案线索来自设问本身       | 拆成原始设问、材料摘要、审题结论三栏                     |
| 经济全球化方向被压缩                | 只剩普惠、平衡、共赢，丢掉开放、包容    | 统一标准表达：开放、包容、普惠、平衡、共赢                  |
| 时代主题碎片化                   | 和平与发展被拆成多个散点          | 优先用完整判断句，避免重复拆点                        |
| 中国外交术语错桶                  | 人类命运共同体、正确义利观挂到政治多极化  | 主归中国桶或中国外交，政治多极化只作交叉                   |
| 新型国际关系泛化                  | 所有合作题都套合作共赢           | 增加触发条件：国际关系主体、合作机制、共同发展、非零和            |
| 贸易摩擦触发过宽                  | 海外知识产权纠纷直接写贸易摩擦       | 先写合规和知识产权风险，出现贸易限制或规则争端再上升             |
| 旧索引污染                     | 旧线不确定结论被当成事实          | 旧索引只定位，不定论，所有优先题维持 recheck 到本地复核完成     |
| 学生版混入后台语                  | 出现路径、日志、模型语、审稿语       | 自动扫描加 Governor 人审双门槛                   |
