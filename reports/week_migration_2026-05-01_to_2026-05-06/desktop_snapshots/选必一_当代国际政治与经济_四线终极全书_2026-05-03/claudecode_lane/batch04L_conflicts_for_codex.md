## Conflict Items

### C1 — 整批 _guarded 的 evidence_tier 分级
本批 Q20 doc 给「学科用语清单 + 4 级等级表 + 3 段示例」但**无逐点细则**，与 04K 房山 P0 逐点 docx 是不同源类。

- 建议 fusion schema 显式区分：
  - `P0_scoring_docx`（逐点细则，房山 04K 类）
  - `P0_scoring_docx_guarded`（含官方学科用语清单 + 等级表 + 示例段，无逐点切分；本批 04L 类）
  - `P1_pptx_only`（仅 PPTX 参考答案；丰台早期类）
- 不建议把 04L 当作 04K 同 tier 直接落分；至少在 fusion 元数据中保留 _guarded 后缀。

### C2 — ATOM-02「人类命运共同体」是否纳入「价值意蕴」族跨题枢纽
石景山 Q20 角度清单内含"人类命运共同体"，与延庆 2026 Q19(2)「价值意蕴」三件套（人类命运共同体 / 共商共建共享全球治理观 / 新型国际关系）共族。

- 建议 fusion 跨题登记：在「人类命运共同体」族下，石景山 Q20 04L 作为 _guarded 支证，延庆 Q19(2) 作为 P0 主证（如延庆是 P0 逐点）。
- 不建议把石景山 Q20 当作主证锚定，避免 _guarded 源越级。

### C3 — Q19 必修三「共建共治共享社会治理格局」 vs Q20 选必一「共商共建共享」撞形
两题在同一份 doc 中前后相邻，字面高度相近但模块完全不同。

- 建议 fusion 在术语对齐表中显式分两 entry：
  - `必修三 / 社会治理 / 共建共治共享社会治理格局`（不进选必一矩阵）
  - `选必一 / 全球治理 / 共商共建共享(全球治理观)`（进选必一矩阵；本批 ATOM-01）
- 不建议把两个串合并到同一 cluster；会污染选必一术语库。

### C4 — Q20 题面 hard-cap「任选两个」与赋分公式「关键词×4 = 8」叠加
ATOM-06 + ATOM-11 两条元数据共同约束学生稿生成。

- 建议 fusion 落字段：
  - `scoring_formula = "4 per keyword × max(2 keywords) = cap 8"`
  - `question_hard_caps = ["pick exactly 2 of {共同, 开放, 包容}"]`
  - `applies_to = [ATOM-01..05]`
- 学生稿生成必须显式绑定关键词到学科用语：例如「共同 + 共商共建共享 + 共同维护/共同推进/共享繁荣 + 维护共同利益落点」 → 该关键词 4 分；写 3 个关键词或不绑定不加分。

### C5 — 示例段修辞配料 vs 学科用语原子的边界

### C6 — 「人类命运共同体 / 和平发展合作共赢」二者在示例段对位强弱的差异
