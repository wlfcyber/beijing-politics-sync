# Decision Maker Findings

Run: 选必一《当代国际政治与经济》四线终极全书 2026-05-03

Role: Codex production lane A / 决策者

## 下一瓶颈

当前瓶颈不是正式条目生产，而是从 Phase 1C 转入 Phase 2 的可审计源定位：五角色已经在总账中启动，但劳动者、补丁者、ClaudeCode 还没有拿到同一批高权重套卷顺序和同一套证据边界。下一 60 分钟必须先把首批高权重套卷定位清楚，确保后续条目只从评分细则、评标、阅卷细则、阅卷总结、用户确认评分材料进入主链。

## 下一 60 分钟优先级

1. 0-10 分钟：劳动者先建立首批高权重套卷的源定位清单，只记录套卷、题号、评分材料类型、可读位置、是否需要回原图/OCR/渲染页，不写正式术语条目。
2. 10-30 分钟：按“海淀 -> 西城 -> 东城 -> 朝阳”处理首批源定位；硬样本 2026 通州期末 Q20 作为校验插队项同步定位，但不挤掉海西东朝顺序。
3. 30-45 分钟：补丁者对已定位题目只做风险标记：一题多点、总说/分说、同核心误拆、选必一/必修二边界、普通参考答案冒充细则风险。
4. 45-60 分钟：ClaudeCode 生产线 B 做独立源定位挑战，输出与 Codex A 不一致的候选缺口；决策者下一轮只看冲突、缺口和阻塞，不看泛泛进度。

## 高权重套卷顺序

首轮顺序必须体现海西东朝优先：

1. 海淀：2025 海淀期中 Q16(2)、2025 海淀期中 Q21(2)、2025 海淀期末 Q22；2026 海淀期末已确认没考选必一，只作为排除记录。
2. 西城：2026 西城期末 Q20。
3. 东城：2024 东城一模 Q16、2024 东城一模 Q20。
4. 朝阳：2026 朝阳期中 Q17、2026 朝阳一模 Q20。
5. 硬样本插队校验：2026 通州期末 Q20 六点细则必须完整定位，用于检验后续“六点不漏、人类命运共同体不误归类”的硬规则。
6. 第二批候选：顺义、丰台、昌平、通州其他题、房山、大兴、密云、延庆、平谷等，在首批源定位闭合后再排。
7. 永久排除：2026 石景山期末全模块排除，除非用户明确提供新的可用评分细则。

## 给劳动者的任务

- 只做源定位和题号定位，不写正式条目。
- 对每个首批题目记录：套卷、题号、题型/设问简记、评分材料类型、位置线索、是否可读、是否需要回原卷/PPT/OCR/渲染页、是否可进入后续条目生产。
- 证据不足时写 blocked/reference-only，不得用普通参考答案补成细则。
- 遇到混合模块题，先标出选必一相关点与必修二/政治与法治/其他模块边界，不做强行归并。

## 给补丁者的任务

- 对劳动者首批定位结果做风险扫描，不重做源条目。
- 优先查四类风险：总说/分说漏收、一题多答题点漏收、同核心误拆/误合并、框架桶错位。
- 硬盯样本：2026 通州期末 Q20 六点、2026 朝阳期中 Q17 总说/分说、2024 东城一模 Q16/Q20、2025 海淀期中 Q16(2)/Q21(2)。
- 输出给下一轮决策者的应是“哪题必须补读哪一页/哪种评分材料”，不是学生版措辞。

## 给 ClaudeCode 的任务

- 作为生产线 B 独立挑战 Codex A 的首批源定位，不继承 Codex A 的判断结论。
- 按同一顺序“海淀 -> 西城 -> 东城 -> 朝阳 -> 通州硬样本”查找评分细则、评标、阅卷细则、阅卷总结或用户确认评分材料。
- 输出只需三类：confirmed_source、blocked_no_scoring_source、conflict_with_codex_a。
- 任何普通参考答案只能标 reference-only；不得作为主链细则证据提交。

## 不应阻塞事项

- GPT-5.5 Pro、Claude Opus、Word/PDF 渲染、最终学生化措辞不应阻塞首批源定位。
- 频次统计、六桶最终合并、表述积累表不应早于首批源定位闭合。
- 非海西东朝的普通套卷不应阻塞首小时推进。
- 旧终稿、五题样例、v8-v12 复查结论只能帮助定位和查漏，不能直接成为新主链证据。
- 2026 石景山期末排除不是缺口；2026 海淀期末没考选必一也不是缺口，只需保留排除状态。
