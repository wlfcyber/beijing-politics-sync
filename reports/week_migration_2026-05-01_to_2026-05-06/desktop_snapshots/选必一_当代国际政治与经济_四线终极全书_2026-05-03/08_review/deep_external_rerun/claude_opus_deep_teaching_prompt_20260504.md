## 你的角色边界

你不是本地证据裁判，不能新增评分细则，不能凭模型常识改采分点。所有“术语真假、分值、是否必答、同核心是否真的同核心”最终由 Codex 根据本地证据裁定。

你的任务是教学成品化：判断零基础或薄弱学生读完后，是否能从材料信号迁移到框架落点，再写出自然的答题句。

## 重点检查清单

1. 学生学习路径：
   - 每题是否能看出“设问问什么 -> 材料触发什么关系 -> 为什么对应这个答题点 -> 考场怎么写”。
   - 是否存在“术语很多，但学生不知道何时用”的问题。
2. 六桶结构：
   - 六桶索引是否能作为考前框架，而不是一堆题号列表。
   - 六桶术语积累复盘是否能帮助学生背一个核心点、会多种写法。
3. 同类项合并后的表达积累：
   - 是否需要把“答题点自身积累”拆成更适合学生记忆的结构。
   - 是否有同义变体堆叠太长、题目专属句混进通用积累的问题。
4. 答案句质量：
   - 是否像真实考场卷面。
   - 是否有口号化、空泛化、语序不自然、太像制作说明的句子。
   - 是否需要把部分长句改成“两句式/三段式”。
5. 慎用提醒：
   - 是否具体到“为什么慎用、什么时候不能用、什么时候可以作为表述积累”。
   - 是否只是模板化重复。
6. 学生误学风险：
   - 是否把开放题训练点写得像固定采分点。
   - 是否把跨模块内容混入选必一主链。
   - 是否把“全球治理倡议 / 人类命运共同体 / 中国智慧中国方案 / 新型国际关系”讲混。
7. 成品文档观感：
   - 是否需要在每个大桶前加一句“使用口诀/判断规则”。
   - 是否需要给高频核心点做“最稳写法 / 可替换写法 / 不能乱用”。

## 输出格式

请输出两部分。

第一部分：问题表，至少努力找 10 个高价值问题。

字段：

`issue_id | severity | location | problem | why_it_matters_for_student | teaching_rewrite_suggestion | local_evidence_check_needed`

severity 只用：

- `must_fix_teaching`：不改会让学生学错、不会迁移或写错卷面。
- `should_fix_transfer`：会明显提升迁移和背诵效率。
- `style_or_readability`：表达和版式优化。
- `needs_local_evidence_before_change`：你觉得可能要改，但必须 Codex 本地核验证据。

第二部分：请给一个“最值得改的成品化方案”，包括：

1. 六桶索引如何改得更像考前框架。
2. 答题点自身积累如何拆分。
3. 每题闭环是否要增加或压缩字段。
4. 哪 5 个高频核心点最需要做“最稳写法 / 可替换写法 / 禁用边界”。

最后给总体 verdict：

- `PASS`：教学表达已足够成品化。
- `PASS_AFTER_FIX`：主体可用，但建议补丁后回归。
- `NEEDS_TEACHING_REWRITE`：需要系统性教学重写。

请尖锐一点，按高三学生能不能真正用来答题来审。
