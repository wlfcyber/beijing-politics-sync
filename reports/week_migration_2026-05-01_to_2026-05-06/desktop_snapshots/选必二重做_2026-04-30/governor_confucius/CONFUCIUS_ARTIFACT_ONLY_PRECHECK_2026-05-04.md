# Confucius Artifact-only 学会性初验

日期：2026-05-04

角色边界：本文件只从已生成 artifact 判断“零基础学生能不能学会”。未重新查源文件，未裁定任何题目答案或细则真伪。它是内部预验，不是 Confucius 闭环，不是 Final Governor PASS。

## 总判断

零基础学生现在“能看到一条路”，但还不能直接靠现有两个 final 文档学完。

可学的部分已经出现：一核把法律题收束为“具体法律关系中的规则适用”；三问让学生知道答案要写“判什么、凭什么、有什么意义”；四步让学生按“定主体关系 -> 找争点事实 -> 对规则条件 -> 按题型落结果”走；五域把八域压成学生能背的生活关系入口。

仍卡住的部分也很明确：合集太像后台原料库，审计标签、评分细则、参考答案、非法律模块答案和 OCR 提示太多；选择题待锁和细则待锁尚未完全隔离；命题路径还需要变成学生看得懂的题型动作，而不是每题都读成“事实 -> 规则 -> 结果”。

## 零基础学生能学会什么

1. 能初步学会先找“谁和谁发生什么法律关系”，不再把法律题写成泛泛法治口号。
2. 能初步学会用三问组织答案：先判结果，再给依据，最后写意义。
3. 能初步学会价值后置：意义题先回本案处理，再上升到诚信、公平、家庭伦理、劳动保护、创新秩序等价值。
4. 能初步按五域识别题目入口：合同消费、侵权人格、婚姻家庭与继承财产、劳动职业、知识产权与竞争。

## 学生会卡在哪里

| 卡点 | 现象 | 风险 |
|---|---|---|
| K1 不会定法律关系 | 只写“这是法律问题”“要依法维权”，说不出谁对谁、关于什么、凭什么 | 一核仍停留在口号层 |
| K2 不会对规则条件 | 直接写结论合法/违法，不逐条件贴材料 | 细则采分点拿不到 |
| K3 选择题待锁误读 | 看到候选答案就当官方答案背 | 后续一旦答案变动，学生记错 |
| K4 `rubric_match_pending` 误读 | 待细则题被当成正式高频样本 | 主干证据不稳 |
| K5 八域和五域混淆 | 学生一会儿背五域，一会儿又看到八域题库标签 | 记忆负担反弹 |
| K6 审计语言污染 | 学生看到“证据状态、匹配方式、OCR、评分细则、参考答案、可从……角度作答” | 不知道哪些是要背的，哪些只是后台证据 |
| K7 混合题未拆 | 法律小问、哲学/逻辑/经济/文化小问混在同一题 | 学生误以为所有角度都属于选必二法律 |
| K8 命题路径未分型 | 每题都像“事实 -> 规则 -> 结果” | 遇到表格题、评析题、程序指导题时不会迁移 |

## 必须修的地方

1. 先做一个干净学生版，不从合集直接拷贝。学生版只留“材料信号 -> 法律关系 -> 可用规则 -> 答题动作 -> 易错点 -> 来源例题”，不要出现审计词。
2. 把五域固定为学生前台，八域只留给教师/后台索引；学生页面不要同时让学生背八域。
3. 给一核加口令：“谁 vs 谁，关于什么，凭什么”。每道题先填这三个空。
4. 给第三步“对规则条件”加小表：规则条件、材料事实、能否对应、答题句。
5. 给第四步按题型分流：选择题排干扰，案例题落责任，表格题对栏目，评析题拆观点，程序题走节点。
6. 9 道 `answer_pending_official` 不进学生正式错项；只能放在教师待核验区。
7. 26 个 `rubric_match_pending` 不进“正式高频必背例题”；最多放入开放题/待核验题池。
8. 混合/特殊题先拆法律小问；未拆前不能用作学生主干训练样本。

## 预验后的 Confucius 裁决

当前学生可学性：`partial_learnable`

可进入下一步：教师/后台继续融合候选框架，制作干净学生版样章。

不可进入下一步：直接把 `选必二法律题全量处理合集_2026-05-03.md` 当学生最终资料，或宣布 Confucius 闭环。

最小修复样章建议：先选 3 题做学生版验证。

- 1 道正式细则已锁的主观案例题：验证“四步 + 规则条件表”。
- 1 道正式答案已锁的选择题：验证“错项不对在哪里、为什么不对”。
- 1 道程序/评析/表格题：验证命题路径分型是否真能迁移。
