# can_do_unseen_subjective_questions

**ALMOST**

已经从“旧题检索式学习”升级到“可迁移框架”的门槛以上。理由是：框架版已经给出“争点三种锁定法”，包括设问反推、双方主张、要件模糊；也给出主观题三段法，即规则依据、材料事实对应、结果落点。情境版也把主观题处理成“材料故事、争点、细则、细则解决争点的方式、作答逻辑”的结构。这个设计能让零基础学生在陌生题里先找争点，再套规则，不再只背旧题。 

# can_stably_full_score

**ALMOST**

主观题得分链已经基本成型，但还存在两处会影响稳定满分的结构风险：第一，五域里“创新竞争与公共救济”把知识产权、不正当竞争、调解、诉讼、公益保护放进同一个入口，学生容易混用模板；第二，框架版局部仍有情境节点错挂，例如“纠纷解决与程序救济”下面出现“继承、遗嘱、遗赠扶养协议与赡养义务”，这会破坏零基础学生的域判断。

# must_fix_before_word_delivery

| issue_id | must_fix                                                                                                                                             |
| -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| MF01     | **修正错挂节点：纠纷解决与程序救济下不应出现“继承、遗嘱、遗赠扶养协议与赡养义务”。** 该节点应回到“财产、相邻、继承与家庭”或“家庭继承”内部。否则零基础学生会把继承纠纷当作程序题模板来写。                                                   |
| MF02     | **修正错挂节点：人格权与侵权责任下不应直接挂“消费者权益、虚假宣传、格式条款与平台规则”。** 若该节点是跨域案例，应标明“主域：交易关系与消费者保护；副域：人格权或侵权”。否则学生会把消费欺诈、格式条款写成侵权主模板。                                      |
| MF03     | **给第五域“创新竞争与公共救济”加硬分流说明。** 现在五域决策树把作品、商标、商业秘密、数据、竞争秩序、调解、诉讼、公益保护放进同一域。交付前必须补一句：本域内部先分“创新竞争题”和“程序公益题”，二者不得共用同一主观题模板。否则学生会把知识产权题写成程序救济，把公益诉讼题写成知识产权保护。 |
| MF04     | **将“特殊侵权”卡改名为“特殊侵权识别卡”，并加一句“不背归责体系，只查免责减责”。** 当前卡片已写无过错、过错推定、安全保障义务、免责减责，方向可用；但若标题不降级，学生容易背成法考式侵权分类。                                                 |

# final_advice_to_codex

**结论：CONDITIONAL PASS，可进入 Word 前最后小修。**

执行顺序：

1. 先修错挂节点：继承节点从“纠纷解决与程序救济”移回家庭继承；消费者节点从“人格权与侵权责任”移回交易保护。
2. 再修第五域：保留五域上限，但第五域内部必须写成两条互斥入口：创新竞争线、程序公益线。
3. 再修特殊侵权标题：改成“特殊侵权识别卡”，加“不背归责体系，只查免责减责和举证边界”。
4. 最后微调自动句式：把“责任、效力、归属、程序或价值”这种通用后台话，替换成每题自己的具体落点。

完成这四项后，可以转 **PASS**。当前版本已经具备陌生主观题迁移能力，但第五域合并和局部节点错挂仍会让零基础学生在少数题上误套。
