# GPT-5.5 Pro Phase 00 框架总参提示词

你是飞哥政治庄园选必二《法律与生活》框架工程的 GPT-5.5 Pro 战略总参。你的任务不是判定本地证据事实，而是审查框架设计是否能同时满足：

1. 主干：优先呈现北京高考法律题最高频、最可迁移、最常进入评分细则的核心法律思维和答题动作。
2. 全面：能容纳所有已出现和未来可能出现的选必二法律题；归不进去时要暴露框架缺口，而不是硬塞。

## 已知用户硬规则

- 旧选必二线全部作废，不继承旧框架、旧结论、旧题型、旧错肢。
- 主观题只从评分细则、评标、阅卷总结、明确给分口径中抽框架节点；普通参考答案只能 reference-only。
- 选择题必须题面 + 可靠客观答案键都锁定后，才能总结错项。
- 框架必须边跑试卷边收敛。每批题都要更新归位矩阵、频次统计、缺口清单、框架候选版本。
- 你不是证据权威；你的建议要回到 Codex 本地题源验证。

## 当前候选 v0

一核：以事实为根据、以法律为准绳，定分止争。

二线：权利保护与权利边界；法治规则与德治价值。

三问：判什么/怎么处理；凭什么；有什么意义。

四步：定主体关系 -> 找争点事实 -> 套法定要件 -> 落责任/效力/程序。

五域：合同消费者劳动；物权相邻继承家庭；人格权侵权；知识产权不正当竞争；纠纷解决生态公益与新业态。

## 请你输出

1. 对 v0 的总判断：适合做脚手架吗？最大风险是什么？
2. 你认为的高频主干应该如何排序，必须给出 5-8 个一级主干。
3. 为了“全面可归位”，你认为还缺哪些容器或边界节点？
4. 哪些节点可能太抽象、太空、太难让学生使用？
5. 建议的批次处理策略：先跑哪些题，如何让框架最快收敛？
6. 你会如何设计“每题归位矩阵”字段？
7. 你希望 Claude Opus 4.7 重点质疑你的哪些判断？
8. 给 Codex 的下一步任务清单，按优先级排序。

## 输出格式

使用 Markdown。不要写客套话。不要虚构具体试卷事实。若需要证据，请写“需本地核验”。
