# Claude Opus 4.7 Adaptive Thinking Web Framework Council Report

## 0. Mode Confirmation

本报告基于用户在 Claude 网页端新建的独立对话生成，模式为 Claude Opus 4.7 Adaptive Thinking。本轮未继承其他项目、对话或线程上下文，仅以本 prompt 提供的 Batch01 证据包为输入。本报告输出的所有判断仅为框架建议，不对题面事实和评分细则作权威裁定，最终需 Codex 回到本地题面与评分细则裁决。

## 1. Learnability Diagnosis

V0 框架的可学性问题主要在层级冗余与语言风格两端。

层级方面，V0 把"一核二线三问四步五域"并列推给学生，结构性记忆节点至少 15 个（1+2+3+4+5），而其中**真正驱动答题动作的只有"四步"**。"一核"是元原则，学生背得下但用不上；"二线"是抽象二元论（权利保护/边界、法治/德治），不能直接转译为答题动作；"三问"与"四步"在问题识别层有部分重叠；"五域"是内容分类，与答题结构不在同一层。把它们并列展示给学生，会让学生在考场上面对一道题时不知道该调用哪一层。

语言方面，"套法定要件""落责任/效力/程序"带有法考语感。高中政治学生未受过请求权基础训练，模仿这种语言容易写出"看起来像法律推理但落不到要件"的答案。从 Batch01 三道题的 rubric_rewarded_action 来看，奖励的是"事实匹配 + 关系归纳 + 意义分层"这种**可操作动作**，不是法考式的要件论证。

Codex 候选框架的可学性改进有限但方向正确：把命题路径降为教师后台、把答题动作压缩到四步、把价值题压缩到一规，是把"教师研究框架"和"学生使用框架"分离的正确动作。但"对规则/条件"这一步措辞仍偏法考，且未明确说明 V0 中"三问/二线/一核"是否被吸收或废弃——这是学生最容易困惑的地方。

## 2. Student-Usable Trunk

从频次快照看，三道题命中率 3/3、formal_scoring 命中 2/3 的节点只有四个：法律关系判断、事实→要件匹配、责任/效力/程序落点、价值收束锚定具体领域。这四个节点几乎完全对应 Codex 的"答题四步 + 价值一规"，**这是当前证据可支持的学生主干假设**。

建议学生面前只放三件东西：

第一是**四步答题动作**：定关系（这是谁对谁、什么类型的法律关系）→ 抓争议（争点事实是什么）→ 对规则（对应哪条法律规定/构成什么要件）→ 落处理（责任、效力、程序、结果落在哪里）。

第二是**价值一规**：遇到"有什么意义/体现什么价值"类设问时，必须先回到本案的法律关系与处理结果，再上升到制度、秩序、价值，禁止跳过案件直接喊价值口号。B01-Q3 的 rubric_rewarded_action（法律确认 + 个案公平 + 司法理念 + 传统美德）完全符合这个一规，是该规则的强证据点（formal_or_scoring_source）。

第三是**五域知识索引**：五域不是主干，是知识抓手，用来在答题前快速定位"这道题考的是哪个教材模块"，从而调出对应的法律关系类型库（劳动关系要件、继承顺序、侵权构成等）。

警告：N=3 的样本下"3/3 命中"统计意义薄弱，主干假设需要更多 batch 验证才能从假设升格为定论。

## 3. Teacher Backend Framework

Codex 提出的"命题目标 → 载体选择 → 设问路径 → 细则奖励动作"作为**教师后台**是合适的，但建议显式增加两个守卫：

一是 **evidence_level 守卫**：教师后台的每一条命题路径推论都要标注证据等级（formal_or_scoring_source / reference_answer / hypothesis），reference_answer 级别的推论不进入框架升级决策，只能挂开放容器。这一条是本 prompt 明确要求的证据边界，必须在教师后台显式化。

二是 **rubric 反推守卫**：教师后台的"奖励动作"必须从评分细则原文反推，不能从 reference 答案反推；reference 答案与评分细则的差距，本身就是命题路径研究的对象。

教师后台还应保留 V0 中的"一核二线三问"作为分类工具——它们对教师做命题分类、出题预测、错题归因仍然有用，只是不进入学生主干。

## 4. Proposition Path For Students

学生不需要做命题人反推。学生在考场上只需要做三件识别动作：**载体识别**（这是哪类案例：劳动/继承/侵权/新技术……）→ **问点识别**（要我判什么、凭什么、还是有什么意义）→ **收束识别**（最后是落到处理结果还是落到价值意义）。

这三个识别动作完成后，进入答题四步即可。把"命题目标反推"塞给学生，等于在四步之前加一个抽象层，违反了"考场上不让学生多套一层"的原则。

V0 的"三问"应当吸收进学生侧的**问点识别**这一识别动作中，不再作为独立的框架层级。"判什么/怎么处理 / 凭什么 / 有什么意义"就是问点识别的三个分支模板，与四步是上下游而非并列关系。

## 5. Open Containers And Knowledge Anchors

证据包暴露了至少两个需要保留的开放容器。

**新技术开放容器**：B01-Q2（虚拟数字人著作权与不正当竞争）属 reference_answer 级证据，不能升格为 formal scoring source，因此**不能驱动主干扩张**，但应明确挂在"新技术 / 知识产权与不正当竞争"开放容器内，等待后续 batch 出现 formal_or_scoring_source 级证据后再决定是否升格为独立主干分支或维持容器位。当前状态：placeholder，待证据。

**新业态开放容器**：B01-Q1 已是 formal 级证据，但只有 1 条，单点不能确定"新就业形态劳动关系"是否要在五域中独立分支，还是继续归入"合同消费者劳动"。建议保留为可分裂节点，等 batch 累积。

**知识锚点保留**：五域不是主干但必须保留为知识索引，因为它对应教材章节边界，承担"考前知识盘点"和"考场快速定位"两个功能。删掉五域会让学生失去知识抓手，主干虽然清晰但没有内容支撑。

## 6. Risks Of Abstraction Or Misuse

| node | risk | fix |
|---|---|---|
| 二线（权利保护/边界、法治/德治） | 抽象二元论，学生难以转化为答题动作；易写成口号 | 降为教师后台主题词，不进学生主干；学生侧由"价值一规"承担其功能 |
| 套法定要件 | 法考语言，学生模仿后写出形似神不似的要件论证 | 改为"对应法律规定 / 对应构成条件"，与高中教材语言对齐 |
| 价值收束 | 学生易跳过案件直接喊"体现社会主义核心价值观" | 强制价值一规：先本案法律关系与处理结果，再上升 |
| 五域作为主干 | 与答题动作不同层，并列展示造成调用混乱 | 降为知识索引，定位"考哪个模块"，不是"怎么答" |
| 三问独立成层 | 与四步的"定关系/抓争议"重叠，记忆冗余 | 吸收进"问点识别"作为四步前的识别动作 |
| 命题路径作为学生内容 | 让学生反推命题人意图 = 多套一层抽象 | 命题路径只在教师后台，学生侧只做载体/问点/收束识别 |
| reference_answer 升格 | 把非细则证据当主干依据，会让框架建立在沙土上 | evidence_level 守卫：reference_only 只挂开放容器，不进主干 |
| N=3 即定主干 | 样本量过小，频次统计意义弱 | 主干假设需 batch 累积到一定阈值后才能从假设升格为定论 |

## 7. Proposed Student-Facing Structure

建议学生面前只呈现以下三件东西，全部都能在 30 秒内复述：

**主干一：四步答题**——定关系 → 抓争议 → 对规则 → 落处理。这是任何一道法律生活主观题都要走的动作流。

**主干二：价值一规**——遇到"意义 / 价值 / 体现什么"类设问时，先案件后价值：先回到本案的法律关系与处理结果（一句到两句），再上升到制度、秩序、价值（一到三层）。禁止跳过案件直接上价值。

**索引：五域知识抓手**——五域不是答题结构，是知识地图。考前用它做模块盘点，考场上用它做"这题考哪个模块"的快速定位，定位完成后调用对应模块的法律关系类型库（劳动关系要件、继承顺序、侵权构成、知识产权类型、合同效力等），然后回到主干一的四步。

教师后台保留但不教给学生：命题目标 → 载体 → 设问路径 → 奖励动作；以及 V0 的"一核二线三问"作为教师分类工具。

学生记忆负担：4（步）+ 1（规）+ 5（域索引）= 10 个节点，但其中只有 4+1=5 是结构性动作，其余是知识地图，远低于 V0 的 15 节点全并列。

## 8. Evidence/Placement/Proposition-Path Risks

**证据风险**：B01-Q2 为 reference_answer 级，按本 prompt 明示边界不能升格为 formal scoring source。任何"新技术 / 数字人 / AI 著作权"主干扩张提议都不能援引 B01-Q2 作为唯一证据。当前状态：开放容器待证据。

**样本量风险**：N=3 不足以确认四节点（法律关系判断、事实→要件匹配、责任落点、价值收束）的"3/3 命中"是否具有统计意义。下一批次需重点验证：四节点是否仍 3/3、五域是否出现新的 formal 命中、是否出现四步装不下的题型。

**评分细则文本风险**：证据包列出 rubric_rewarded_action 摘要，但未提供评分细则原文。所有"评分细则奖励 X 动作"的判断需 Codex 回到本地评分细则核验。本报告所有"formal_or_scoring_source 支持 X"的措辞均为基于摘要的推论，需本地核验。

**命题路径假设风险**：表中 question_path 的归纳（"事实→要件→裁决理由→意义"等）也是基于摘要，未与原始设问文本逐字比对。命题路径是否真的呈现这种线性结构，需 Codex 在题面中核验。

## 9. Version Upgrade Criteria

V0 → V1 的升级条件，建议至少满足以下四条：

第一，formal_or_scoring_source 级证据累计达到一定阈值（建议 ≥ 10 条）支持"四步答题主干"在多种题型中都被评分细则奖励，而不仅是当前 Batch01 的两条。

第二，至少出现一条 formal_or_scoring_source 级证据进入"新技术 / 知识产权与不正当竞争"开放容器，使该容器从假设位升格为主干分支或被合并。在此之前 B01-Q2 类题型只能挂容器。

第三，五域命中率统计完成（建议覆盖跨年份、跨区、跨阶段至少 30+ 题），确认五域作为知识索引而非答题主干的合理性。如果某一域命中率畸高且 rubric 行为模式独立，需考虑是否提升为独立主干分支。

第四，学生试用反馈数据：考场上学生是否真的能用四步+一规独立产出符合评分细则的答案，而不是退回到 V0 的"先想用哪一线哪一问"的多层调用。这条是最终判据。

辅助升级触发：出现明确装不进四步的题型（程序题、调解题、立法目的题、生态公益型公益诉讼题等），需要明确判断是扩四步、还是开新容器、还是细化"对规则"步内部分支。

## 10. Questions For GPT-5.5 Pro

第一，问点识别（V0 三问）是否要从主干中显式拆出作为四步之前的"零步"？还是隐含在"载体→问点→收束"识别动作里就够了？过度显式化会增加学生记忆负担，过度隐含可能让学生混淆问点导致答题方向偏。

第二，价值一规如何防御"喊口号"型答案？是否需要在一规内部细化价值层级（个案公平 → 制度功能 → 秩序价值 → 核心价值观），并要求学生至少触及两层？还是保持一规简洁、由教师在批改时把控？

第三，五域与新技术开放容器的边界如何处理？例如生态环境、公益诉讼、新业态劳动既属于五域中的"纠纷解决生态公益与新业态"分支，又可能触发新技术 / 新业态开放容器，重叠时如何判定主属？

第四，选必二《法律与生活》与必修三《政治与法治》的法律部分如何在五域索引层做切分？两者教材内容有重叠（如基本权利、法治原则），框架是否需要标注每一域的"教材出处"以避免学生跨教材调用混乱？

第五，是否要把"命题人 / 区域偏好"作为教师后台的元数据维度（如朝阳偏新业态、海淀偏新技术与价值收束）？如果要，需要多少个 batch 才能稳定提取偏好特征而不是过拟合？

第六，evidence_level 守卫是否需要建立 formal / reference 之间的中间档（例如"半正式：教师内部使用的评阅参考"）？当前二分法可能过严，把一些可用证据排除在外。

第七（提示给 Codex 的核验请求）：本报告所有基于"rubric_rewarded_action 摘要"做出的归纳，需要回到本地评分细则原文逐条核验，特别是 B01-Q1 的"三个从属性材料匹配"和 B01-Q3 的"养老司法理念"是否在评分细则原文中以这种形式出现，还是研究者归纳。这一核验直接决定主干假设是否成立。
