## 2. 我反对或保留的建议

我反对现在就“删三问”。三问“判什么、凭什么、有什么意义”虽然与四步有重叠，但它更像阅卷视角的压缩检查表。直接删除会损失一种防漏机制。建议不让学生背两套，但在教师版保留为验收规则：

- 四步是学生答题动作。
- 三问是教师检查答案是否完整的后台规则。

我也反对把“五域分类本身”完全降到开放容器。五域不应是学生背诵主干，但它可能仍是题源归档、样本覆盖和课程编排的主轴。若完全容器化，后续容易出现主干动作很顺，但题域覆盖失衡的问题。

## 3. 需要本地题源/细则进一步验证的地方

这些不能凭当前 N=3 定论：

1. 四步链能否覆盖纯程序题：如仲裁、诉讼、举证、调解、管辖、诉讼权利。
2. 四步链能否覆盖纯效力题：合同有效、无效、可撤销、效力待定。
3. 四步链能否覆盖比较题：如不同纠纷解决方式、不同权利保护路径比较。
4. “载体—问点—收束”是否适用于选择题、小题、材料辨析题。
5. “价值收束必须先锚定具体法律关系”是否适用于纯意义题。
6. 新技术 IP 是否只是热点容器，还是已经有稳定 rubric 动作。

## 4. 对 `主干高频层` 的影响

我建议主干高频层采用 Claude 的四步链，但做一个小修正：

```text
定关系 → 抓争点 → 对规则/要件 → 给结论/责任/程序
```

其中第 3 步不要只叫“套要件”，因为程序题、效力题、比较题未必都是“要件”。改成“对规则/要件”更稳。

主干高频层不应纳入五域名称，也不应纳入口号式价值。但主干旁边必须保留一个“价值收束规则”：

```text
先说本案法律关系和判断，再说制度、秩序、价值。
```

## 5. 对 `开放容器层` 的影响

开放容器层应分级，不应一锅放：

- A类容器：已有 formal 样板，但暂未跨题域高频，如新就业形态劳动关系、遗赠扶养。
- B类容器：reference_only 但命题真实度高，如 AI/虚拟数字人/IP/不正当竞争。
- C类容器：仅用于题域索引，如合同、侵权、继承、劳动、知识产权、纠纷解决。

这样既不污染主干，又不会把重要新题型教学放弃。

## 6. 更能解释命题路径的地方

Claude 的“载体—问点—收束”有价值，但我建议改为教师版四段：

```text
命题目标 → 载体选择 → 设问路径 → 细则奖励动作
```

学生版不用背这四段，只转化为：

```text
材料给了什么关系？
题目到底问判什么？
答案最后要落到责任、效力、程序，还是意义？
```

对 Codex 来说，真正有用的是“细则奖励动作”。框架不是看知识点名称，而是看细则奖励学生做了什么：匹配要件、归纳关系、说明责任、分层意义、比较路径等。

## 7. 法律职业考试化 / 必修三化风险

Claude 的报告总体避开了法律职业考试化，但“套要件”这个词有潜在风险。若后续展开过细，容易滑向法考式构成要件、例外规则、判例细节。高中政治应保留“材料事实对应教材规则”的粒度。

也要防止必修三化。比如“法治与德治”“公平正义”“社会治理”不能脱离本案法律关系单独写。选必二的价值表达应是：

```text
具体权利义务/责任判断 → 纠纷解决或权利保护 → 秩序/公平/诚信/法治价值
```

而不是直接写“全面依法治国意义重大”。
