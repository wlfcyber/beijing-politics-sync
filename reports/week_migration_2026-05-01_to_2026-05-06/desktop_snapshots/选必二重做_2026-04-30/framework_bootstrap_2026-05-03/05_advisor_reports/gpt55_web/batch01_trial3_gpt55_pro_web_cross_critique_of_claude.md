# GPT-5.5 Pro Cross-Critique of Claude Opus 4.7 Adaptive Report

以下批判基于所附 Claude Opus 4.7 Adaptive Raw Report，且只依据当前 Batch01 Evidence Pack 与该报告本身，不引入外部题面事实。凡涉及具体评分细则原文、题面原文、区域偏好、学生使用效果，均标注为“需本地核验”。

## 1. 我同意 Claude 的哪些框架变化，为什么

### 1.1 同意“教师后台”和“学生前台”分离

Claude 把“命题目标 → 载体选择 → 设问路径 → 细则奖励动作”放到教师后台，把“定关系 → 抓争议 → 对规则 → 落处理”放到学生前台，这个方向应当接受。

理由：

1. Batch01 三题的高频动作确实集中在法律关系判断、事实与规则匹配、结果落点、价值收束。
2. 教师需要研究命题路径，但学生考场上不宜先反推命题目标。
3. 分层后能避免 V0 的“一核二线三问四步五域”全部并列展示，降低调用混乱。

但要补一句：学生仍需要做最低限度的“问点识别”，例如判断设问要求“判什么”“凭什么”“有什么意义”。不能把学生端压缩到只剩机械四步。

### 1.2 同意“四步答题动作”作为暂定学生主干

Claude 主张学生侧保留四步：

> 定关系 → 抓争议 → 对规则 → 落处理。

这个可以进入 Codex 融合方案，但只能标为“Batch01 暂定主干”，不能标为稳定 V1 主干。

理由：

1. B01-Q1、B01-Q3 两道 formal_or_scoring_source 均支持“法律关系判断 + 事实匹配 + 结果落点”。
2. B01-Q2 虽然也是 3/3 命中之一，但证据等级是 reference_answer，不能推高主干证据强度。
3. 当前更稳妥的表述是：四步是“主观案例题高适配动作流”，是否适配程序题、比较题、选择题，需本地核验。

### 1.3 同意“价值一规”

Claude 的价值一规值得接受：

> 遇到意义、价值、体现类设问，先回到本案法律关系与处理结果，再上升到制度、秩序、价值。

理由：

1. B01-Q3 的证据包显示，评分动作包含“法律确认 + 个案公平 + 养老司法理念 + 传统美德/核心价值观”。
2. 这说明价值表达不能脱离遗赠扶养协议效力、扶养履行、遗产归属这些具体法律问题。
3. 该规则能防止学生把选必二答成必修三宏观法治题。

需要修正的是，Claude 提到价值可上升“一到三层”，这个层级数量不能提前固化。是否需要“个案公平 → 制度功能 → 秩序价值 → 核心价值观”的层级链，需本地核验评分细则原文。

### 1.4 同意“五域降为知识索引”

Claude 将五域定位为知识索引，这一点应接受。

理由：

1. 五域适合帮助学生判断题目属于劳动、继承、侵权、知识产权、纠纷解决等哪个教材模块。
2. 五域本身不能替代答题动作。
3. Batch01 中，劳动、知识产权、继承扶养都能归位，但评分奖励动作主要仍是关系、事实、规则、结果、价值。

但不能删除五域。若只剩四步，学生会知道“怎么推”，却未必知道“从哪个知识仓库调规则”。

### 1.5 同意 evidence_level 守卫和 rubric 反推守卫

这是 Claude 报告最应保留的部分之一。

应接受两条守卫：

1. reference_answer 只能进入开放容器或候选节点，不能推动主干。
2. 评分奖励动作必须回到 formal_or_scoring_source，不能从 reference answer 直接反推。

理由：B01-Q2 正好是硬案例。它可以提示“新技术知识产权/不正当竞争”容器，但不能证明该节点已是高频主干。

### 1.6 同意“新技术开放容器”暂不升格

Claude 将 B01-Q2 放入“新技术/知识产权与不正当竞争”开放容器，且明确不能推动主干，这一点正确。

理由：

1. B01-Q2 的 evidence_level 是 reference_answer。
2. “虚拟数字人、AI、新技术、创新秩序”等关键词不能凭一题参考答案升格。
3. 后续需要 formal_or_scoring_source 补强。

## 2. 我反对或降级 Claude 的哪些变化，为什么

### 2.1 降级：“V0 真正驱动答题动作的只有四步”

Claude 对 V0 的批评有价值，但“只有四步驱动答题动作”说得过强。

应改为：

> 四步是学生端核心动作，但一核、二线、三问、五域仍分别承担原则、价值、问点识别、知识归位功能。

理由：

1. “一核”可以作为法律题总原则，尤其服务于“以事实为根据、以法律为准绳”的表达。
2. “三问”可转化为问点识别，不能完全消失。
3. “二线”可降为教师后台和价值收束资源，但不能直接废弃。
4. “五域”是知识索引，不能当主干，也不能删掉。

### 2.2 降级：“学生面前只放三件东西，30 秒内复述”

该建议方向上减负，但存在过度压缩风险。

风险：

1. 纯程序题可能需要“选路径 → 说理由 → 明后果”。
2. 比较题可能需要“比较对象 → 比较标准 → 分别适用 → 选择理由”。
3. 选择题可能需要“定关系 → 排除错位规则 → 核对法律后果”。
4. 效力题可能需要单独突出“成立、有效、履行、归属”的链条。

修正建议：

> 学生主界面可以是“四步 + 价值一规 + 五域索引”，但教师训练时必须配套题型触发卡，至少包括程序题、效力题、比较题、选择题。

### 2.3 反对：过度弱化“要件/条件匹配”

Claude 担心“套法定要件”有法考语感，这个担心成立。但如果完全回避“要件”或“条件”，会损害 B01-Q1 的核心得分动作。

B01-Q1 的证据包明确出现“三个从属性材料匹配 + 劳动关系归纳”。这本质上就是事实与条件匹配。高中政治可以不用法考化语言，但不能取消这一动作。

建议改写为：

> 对教材规则、法律条件、构成要素、程序要求。

这样既保留推理精度，又避免把选必二训练成法考。

### 2.4 降级：“学生不需要做命题人反推”

这句话需要修正。

学生确实不需要反推“命题目标、区域偏好、载体选择逻辑”，但学生必须识别设问路径。否则无法判断答案重心是“处理结果”“理由说明”还是“意义价值”。

建议改为：

> 命题路径留在教师后台；学生只保留低成本问点识别：问我判什么、凭什么、怎么处理、有什么意义。

### 2.5 降级：“新业态开放容器”

Claude 把 B01-Q1 推出“新业态开放容器”，这个建议可以暂存，但不能升格。

理由：

1. B01-Q1 的 formal 证据支持“新就业形态劳动关系”，但只有 1 条。
2. 它更稳的位置仍是“合同消费者劳动 / 劳动关系认定 / 新就业形态载体”。
3. 是否需要单列“新业态开放容器”，需看后续是否出现平台用工、直播带货、算法管理、灵活就业等 formal 题面与评分细则，需本地核验。

### 2.6 降级：V1 升级阈值中的具体数字

Claude 建议 formal_or_scoring_source ≥ 10、跨年份跨区跨阶段 30+ 题、学生反馈等条件。方向合理，但具体数字属于方法设定，当前证据包不能证明其必要性。

建议暂缓固定阈值。可以保留原则：

1. formal 证据优先。
2. 多题型覆盖。
3. 多领域复现。
4. 学生作答效果验证。

具体阈值由 Codex 根据本地样本规模决定，需本地核验。

## 3. 哪些建议会增加学生负担、过度抽象、过度压缩或导致漏题

### 3.1 “载体识别 → 问点识别 → 收束识别 → 四步”可能形成双层流程

Claude 想给学生减负，但如果呈现不当，学生会先背三个识别动作，再背四步，实际变成七个动作。

修正：

> 把“载体、问点、收束”压成零步：先看问什么。之后进入四步。

学生端可写成：

> 零步看问法；四步写答案；意义题加价值一规。

### 3.2 “五域 + 法律关系类型库”可能膨胀

Claude 提到五域可调出“劳动关系要件、继承顺序、侵权构成、知识产权类型、合同效力”等。这个方向有用，但如果每个域都扩成大表，会回到记忆负担过重。

修正：

每个域只保留三类最小卡片：

1. 常见主体关系。
2. 常见争点。
3. 常见结果落点。

例如继承扶养域：

> 主体：扶养人与被扶养人、法定继承人、遗赠扶养协议相对方。
> 争点：协议效力、扶养履行、遗产归属。
> 结果：确认效力、确定归属、价值收束。

### 3.3 “价值一到三层”可能诱导套话

如果学生机械写三层价值，可能出现空泛套话。当前证据只支持“价值必须锚定本案”，不支持固定三层结构。

修正：

> 有几层价值，由设问和评分细则决定。学生至少要做到“本案处理结果 + 对应制度或社会价值”。

### 3.4 “30 秒复述”可能过度压缩

30 秒复述适合记忆口令，不适合完整答题训练。尤其是程序题、效力题、比较题、选择题，可能需要额外分支。

修正：

> 口令可以短，训练不能短。Codex 应在本地题库中测试四步对特殊题型的覆盖力。

### 3.5 “二线降为教师后台”可能导致价值题弱化

如果二线完全不进入学生视野，学生遇到权利边界、法治与德治、创新与秩序等题时，可能缺少价值语言资源。

修正：

> 二线不作为独立主干，但可进入“价值词库”。学生不背二线结构，但知道价值收束可以从权利保护、权利边界、法治秩序、德治价值中选取。

### 3.6 “落处理”表述偏窄

“落处理”简洁，但容易漏掉归属、救济路径、裁判理由、证据事实等落点。

建议扩为：

> 落责任/效力/归属/救济路径/裁判理由。

学生口令仍可叫“落处理”，教师框架内要列全。

> 教师后台研究“命题目标 → 载体 → 设问 → 评分动作”，并加 evidence_level 守卫；学生前台使用“零步看问法 + 四步答题 + 价值一规 + 五域索引”。四步暂定为“定关系 → 抓争点 → 对规则/条件/程序要求 → 落责任/效力/归属/救济路径/裁判理由”。B01-Q2 新技术题只进开放容器，不能推主干。纯程序题、效力题、比较题、选择题必须等待 formal_or_scoring_source 本地核验。
