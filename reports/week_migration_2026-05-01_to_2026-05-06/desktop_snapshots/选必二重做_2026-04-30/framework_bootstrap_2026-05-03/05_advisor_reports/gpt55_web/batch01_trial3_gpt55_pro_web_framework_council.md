# GPT-5.5 Pro Web Framework Council Report

## 0. Mode Confirmation

本报告基于用户可见的 GPT-5.5 Pro 网页端新对话生成。

本轮只依据用户提供的 Batch01 Evidence Pack 作框架委员会意见，不引入其他项目、对话、线程、文件或题面结论。所有具体试卷事实与评分细则细节，均需等待 Codex 回到本地题面和评分细则裁决。

## 1. Diagnosis

核心判断：Codex Candidate 的方向基本成立，可以作为 V0.1 候选骨架，但不能直接升格为稳定框架。现有证据只有三道主观题，其中 formal_or_scoring_source 为 2 道，reference_answer 为 1 道。它足以支持“法律关系判断、事实与要件匹配、责任或效力落点、价值收束锚定具体领域”成为暂定主干，但不足以支持对所有题型、所有模块、所有命题路径的覆盖性结论。

V0 的优势是有清晰的法律思维骨架：

1. “一核”能统摄主观题裁判逻辑。
2. “四步”能解释 B01-Q1 和 B01-Q3 的 formal 评分动作。
3. “五域”能容纳劳动、知识产权、继承扶养等材料场景。
4. “三问”能解释主观题常见设问，即判什么、凭什么、有什么意义。

V0 的主要缺口是：

1. **五域偏知识目录，命题路径不足。**
   B01-Q1、B01-Q3 显示，评分奖励动作并非只奖励学生知道“劳动”或“继承”，更奖励学生完成“主体关系识别 → 争点事实提取 → 规则条件匹配 → 处理结果或价值收束”。

2. **价值线需要降噪。**
   B01-Q3 支持德治价值、传统美德、核心价值观等表达，但这些表达必须锚定遗赠扶养协议、履行扶养义务、遗产归属等具体法律关系。若直接上升到宏观法治或道德倡议，会弱化主观题得分动作。

3. **reference answer 不能推动主干。**
   B01-Q2 可以提示“新技术知识产权、不正当竞争、创新秩序”是开放容器，但它没有 formal_scoring_count 支撑，不能据此把“AI 数字人、新技术 IP”提升为高频主干。

4. **题型覆盖不足。**
   当前证据包集中在主观题。纯程序题、纯效力题、比较题、选择题、图表题、案例评析题、开放建议题等是否适配，需本地核验。

5. **框架容易走两个极端。**
   一端是把选必二做成法考式要件细目背诵，另一端是把法律题做成必修三宏观法治价值题。Batch01 反映出的更稳妥方向是：用教材法理和生活案例完成“事实，规则，处理，价值”的有限推理。

## 2. High-Frequency Trunk

建议暂定高频主干为“五个动作”，证据强度按 Batch01 当前数据排序：

### 2.1 定关系：法律关系判断

证据：B01-Q1、B01-Q2、B01-Q3 均涉及法律关系判断；formal_or_scoring_source 中有 B01-Q1、B01-Q3 两道支持。

应升为主干。它是学生进入题目的第一动作。没有关系判断，后续的权利义务、责任、效力、程序、价值都容易漂浮。

可表达为：

> 先确定谁与谁之间发生何种法律关系，再判断争议属于权利保护、效力确认、责任承担、程序救济或价值说明中的哪一类。

### 2.2 抓争点：从材料事实中提取评分事实

证据：三题均体现“事实 → 要件匹配”。B01-Q1 尤其明确，评分动作奖励“三个从属性材料匹配 + 劳动关系归纳”。B01-Q3 奖励协议效力、扶养履行、遗产归属的对应分析。

应升为主干。此节点要突出“材料事实不是背景故事，材料事实是得分条件”。

### 2.3 对规则：规则、条件、要件、制度功能匹配

证据：B01-Q1 的劳动关系认定，B01-Q3 的遗赠扶养协议效力与履行，均说明评分细则更偏向“材料事实对应法定条件或制度规则”。

建议把 V0 的“套法定要件”扩展为：

> 对规则/条件/要件/程序要求。

原因：选必二不是所有题都以严格“构成要件”出现。程序题可能考调解、仲裁、诉讼选择；效力题可能考合同效力、协议效力、遗嘱或遗赠扶养安排；比较题可能考不同救济路径或不同法律关系。用“规则/条件/要件/程序要求”更稳。

### 2.4 落结果：责任、效力、归属、程序、裁判理由

证据：B01-Q1 落到劳动仲裁支持经济补偿；B01-Q3 落到确认协议有效并判遗产归居委会。B01-Q2 虽为 reference_answer，也提示新技术场景中会落到消除影响、赔偿损失等结果，但不能作为 formal 证据推动主干。

建议将 V0 的“落责任/效力/程序”改为：

> 落责任/效力/归属/救济路径/裁判结果。

这样能覆盖 B01-Q3 的遗产归属，也为后续物权、继承、合同效力、侵权赔偿、程序救济留位置。

### 2.5 收价值：价值必须锚定具体法律关系和处理结果

证据：三题都有“意义”或“价值收束”节点；formal_or_scoring_source 中 B01-Q1、B01-Q3 都支持“价值不能空泛”。B01-Q3 尤其说明，公平正义、养老司法理念、传统美德、核心价值观要以遗赠扶养协议的确认和履行为锚。

建议作为主干中的“价值一规”：

> 问意义时，先说本案法律关系和处理结果，再上升到制度、秩序、价值。

这条规则可以防止学生把选必二答成必修三宏观法治题。

## 3. Proposition Path Diagnosis

### 3.1 B01-Q1：平台经济劳动关系题

命题路径：

> 平台经济典型载体 → 合作协议外观与实际用工事实冲突 → 三个从属性材料匹配 → 事实劳动关系认定 → 经济补偿与治理意义。

框架启示：

1. 高频主干应突出“外观名称不能替代法律关系实质判断”。
2. 材料中的事实细节要被转化为规则条件。
3. “意义”不能脱离劳动关系确认与劳动者权益保护。

可归位节点：

> 合同消费者劳动 / 劳动关系认定 / 法律关系实质判断 / 事实与要件匹配 / 处理结果与治理意义。

### 3.2 B01-Q2：虚拟数字人、新技术知识产权与不正当竞争题

命题路径：

> 新技术现象 → 权利保护 → 市场秩序 → 创新与规范平衡。

框架启示：

1. 它提示新技术场景可能成为命题载体。
2. 它可以放入“知识产权不正当竞争”的开放容器。
3. 因 evidence_level 为 reference_answer，不能升格为 formal scoring source。
4. 不能据此断言“AI 数字人”已经是高频主干，需本地核验 formal 题面和评分细则。

可归位节点：

> 知识产权不正当竞争 / 新技术开放容器 / 权利保护与市场秩序。

状态：

> reference_only_cannot_promote。

### 3.3 B01-Q3：遗赠扶养协议与德治价值题

命题路径：

> 养老与继承生活案例 → 协议效力与履行争议 → 裁判确认遗产归属 → 个案公平与社会价值。

框架启示：

1. 继承扶养题不是单纯背继承顺序，更考协议效力、履行事实、遗产归属。
2. 法治价值较强，但必须从具体法律关系推出。
3. “法治规则与德治价值”可保留，但要放在处理结果之后。

可归位节点：

> 物权相邻继承家庭 / 继承扶养 / 效力确认 / 履行事实 / 归属落点 / 价值收束。

## 4. Open Containers

以下节点应保留为开放容器，暂不提升为主干，等待本地 formal 证据补强。

### 4.1 新技术知识产权与不正当竞争

来源：B01-Q2。
状态：reference_only。
用途：容纳虚拟数字人、AI 生成内容、短视频使用、商业宣传、平台传播、市场秩序等题材。

应预留位置：合同效力、遗嘱效力、遗赠扶养协议效力、婚姻家庭身份关系效力、格式条款效力等。

### 4.4 比较题与路径选择题

当前 Batch01 无直接证据。
应预留位置：调解与诉讼比较、仲裁与诉讼比较、侵权责任与合同责任比较、法定继承与遗嘱继承比较、劳动关系与劳务关系比较。

### 4.5 选择题与客观题动作

当前证据包为主观题。
应预留位置：概念辨析、主体关系识别、权利类型识别、程序路径判断、法律后果判断、价值判断纠偏。
需本地核验：选择题是否也遵循“关系 → 争点 → 规则 → 后果”的微型路径。

### 4.6 人格权侵权、消费者合同、物权相邻、生态公益与新业态

V0 已有覆盖索引，但 Batch01 仅直接覆盖劳动、知识产权、继承扶养。
需本地核验：这些领域是否在正式评分细则中出现，以及奖励动作是否仍可归入五个主干动作。

## 6. Hard Placement Or Proposition-Path Risks

### 6.1 Codex Candidate 是否过早

结论：可以作为 V0.1 候选，但升级为稳定框架过早。

原因：

1. Batch01 只有三道主观题。
2. formal_or_scoring_source 只有 B01-Q1、B01-Q3。
3. B01-Q2 是 reference_answer，只能作为开放容器证据。
4. 暂无纯程序题、纯效力题、比较题、选择题样本。
5. 暂无学生真实作答错误样本，不能判断框架是否真正提升得分。

建议：保留 Codex Candidate 的主干结构，但所有新增节点加证据等级标签。

### 6.2 是否漏掉纯程序题

风险：中高。

V0 的“五域”含“纠纷解决生态公益与新业态”，但“四步”更偏实体权利和责任落点。若出现程序题，学生可能不知道如何处理“调解、仲裁、诉讼、举证、管辖、执行”等路径选择。

修正建议：

> 在“对规则/条件/要件”中加入“程序要求”，在“落结果”中加入“救济路径”。

需本地核验：纯程序题正式评分细则。

### 6.3 是否漏掉纯效力题

风险：中。

B01-Q3 已涉及协议效力，但还不能代表所有效力题。合同效力、格式条款、遗嘱效力、婚姻家庭身份效力等可能要求更细的判断路径。

修正建议：

> 将“落责任/效力/程序”扩展为“责任、效力、归属、救济路径、裁判结果”。

### 6.4 是否漏掉比较题

风险：中高。

当前主干适合单一案例裁判题。若设问要求比较不同制度、不同救济方式、不同法律关系，框架需要增加“比较维度”。

建议新增比较题动作：

> 比较对象 → 比较标准 → 分别适用条件 → 选择或评价 → 价值收束。

需本地核验：是否有正式比较题评分细则。

### 6.5 是否漏掉选择题

风险：中。

选择题通常不要求完整表达，但它仍可能隐含“关系，争点，规则，后果”的压缩路径。框架应给客观题保留微型版。

建议新增选择题动作：

> 先判关系，再排除错位规则，最后核对法律后果。

需本地核验：选择题错因统计和高频干扰项。

### 6.6 是否会误做成法考

风险：中。

如果把“套法定要件”无限细化，学生可能背诵超教材概念，反而偏离高中政治评分语言。

控制规则：

1. 优先使用教材表述和评分细则表述。
2. 不引入超范围专业法条细目，除非题面材料或评分细则明确要求。
3. 要件匹配服务于答题得分，不追求法学完整性。
4. 框架节点写成答题动作，避免写成法考知识清单。

### 6.7 是否会误做成必修三宏观法治

风险：中高。

B01-Q3 的价值色彩强，但其得分支点仍是遗赠扶养协议效力、扶养履行、遗产归属。若学生只写公平正义、弘扬美德、法治社会，会缺少法律关系和处理结果。

控制规则：

> 凡价值题，先回到本案法律关系和处理结果，再上升到制度、秩序、价值。

## 7. Proposed Next Framework

建议将 V0 升级为 **V0.1 证据分层版**。该版本不追求一次性完备，重点是把“主干动作、知识归位、命题路径、证据等级”分开。

### 7.1 一核

> 以事实为根据、以法律为准绳，通过具体法律关系中的规则适用实现定分止争。

这一核保留 V0 表述，但强调“具体法律关系”，防止空泛价值化。

### 7.2 双视角

#### 教师后台

> 命题目标 → 载体选择 → 设问路径 → 评分奖励动作。

用途：反推为什么选这个案例、问什么、奖励什么。

#### 学生前台

> 看载体 → 定问点 → 抓事实 → 对规则 → 落处理 → 收价值。

用途：将命题路径转化为答题路径。

### 7.3 五个主干动作

1. **定关系**
   谁与谁之间发生什么法律关系。

2. **抓争点**
   材料中哪些事实决定权利、义务、效力、责任、归属或程序路径。

3. **对规则**
   将事实对应教材规则、法律条件、制度要求、程序要求。

4. **落结果**
   得出责任、效力、归属、救济路径、裁判理由或处理方案。

5. **收价值**
   先说本案处理的制度意义，再上升到权利保护、市场秩序、公平正义、家庭伦理、劳动保护、创新秩序等价值。

### 7.4 四类结果落点

为避免“责任”中心过窄，建议单列四类结果：

1. **责任落点**：赔偿、补偿、停止侵害、消除影响等。
2. **效力落点**：合同、协议、遗嘱、遗赠扶养安排、格式条款等是否有效。
3. **归属落点**：财产、遗产、权利成果、收益、权益归谁。
4. **程序落点**：调解、仲裁、诉讼、举证、执行、救济路径。

### 7.5 五域作为覆盖索引

五域保留，但定位为“归位索引”，不作唯一主干。

1. 合同消费者劳动。
2. 物权相邻继承家庭。
3. 人格权侵权。
4. 知识产权不正当竞争。
5. 纠纷解决生态公益与新业态。

### 7.6 证据等级闸门

每个节点必须标注证据等级：

1. **A 级：formal_or_scoring_source 多题支持**
   可进入主干。

2. **B 级：formal_or_scoring_source 单题支持**
   可进入重点节点，但需继续核验。

3. **C 级：reference_answer 支持**
   只能进入开放容器或候选节点。

4. **D 级：框架推断**
   必须标注“需本地核验”。

## 8. Version Upgrade Criteria

建议用以下标准决定 V0.1 是否升级为 V1。

### 8.1 主干升级标准

某节点进入稳定主干，需要同时满足：

1. 至少 3 道 formal_or_scoring_source 支持。
2. 至少覆盖 2 个以上法律领域。
3. 能解释评分细则奖励动作。
4. 能转化为学生可执行答题动作。
5. 不依赖单一热点载体。

当前符合度较高的候选：

1. 法律关系判断。
2. 事实与规则匹配。
3. 落处理结果。
4. 价值锚定具体法律关系。

但仍需更多 formal 样本确认。

### 8.2 开放容器升级标准

开放容器升级为重点节点，需要满足：

1. 至少 2 道 formal_or_scoring_source 支持。
2. 评分细则中有明确奖励动作。
3. 不是只在参考答案中出现。
4. 能跨年份或跨区县复现。

当前“新技术知识产权/不正当竞争”尚不满足。

### 8.3 框架覆盖标准

V1 至少应能自然归位以下题型：

1. 主观案例分析题。
2. 选择题。
3. 比较题。
4. 程序路径题。
5. 效力判断题。
6. 价值意义题。
7. 开放建议题。
8. 新技术或新业态题。

当前 Batch01 只能较强支持主观案例分析题、效力兼价值题、劳动关系认定题。

### 8.4 防偏标准

框架升级时必须通过两个反向测试：

1. **防法考化测试**
   学生是否会被引向超教材法条细节。若会，则节点表述要降到高中政治评分语言。

2. **防宏观法治化测试**
   学生是否会跳过法律关系和具体处理，直接写价值口号。若会，则必须加强“价值后置、锚定本案”的规则。

### 8.5 证据使用标准

1. reference answer 不得推动主干。
2. formal scoring source 优先。
3. 没有题面和评分细则时，只能写“需本地核验”。
4. 框架推断必须与证据节点分开标注。
5. 单一热点题不能直接推导高频趋势。

## 9. Questions For Claude Opus 4.7 Adaptive Thinking

1. Codex Candidate 中“教师后台：命题目标 → 载体选择 → 设问路径 → 细则奖励动作”是否能覆盖选择题，还是只适合主观题？

2. “学生看题：载体 → 问点 → 收束”是否过于简略？是否需要改成“载体 → 法律关系 → 争点 → 规则 → 结果 → 价值”？

3. 当前把“五域”降为覆盖索引是否合理？有没有证据显示某些知识域本身就是高频主干？

4. B01-Q1 的“三个从属性材料匹配”应归为“劳动关系专属节点”，还是归为“事实与要件匹配”的通用主干？

5. B01-Q3 的“传统美德/核心价值观”应归为“继承扶养专属价值”，还是归为“价值收束锚定具体法律关系”的通用规则？

6. B01-Q2 虽然是 reference_answer，但它的“新技术 + 权利保护 + 市场秩序”路径是否值得作为开放容器优先追踪？

12. 如何避免学生把“法律关系判断”写成空泛标签？是否需要给每个五域配一组“常见主体关系对”供训练？

14. 继承扶养题中的“个案公平、养老司法理念、传统美德”是否会在评分细则中并列奖励，需本地核验其层级关系。

15. 新就业形态劳动关系是否应作为年度热点容器追踪，还是只归入“劳动关系认定”的普通子节点？
