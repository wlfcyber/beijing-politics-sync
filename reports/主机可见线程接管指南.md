# 主机可见线程接管指南

用途：把本机已经整理好的北京高考政治督工状态，迁移到主机上，并尽量让线程以你能看到、能手动操作的形式打开。

## 结论

- GitHub 已经保存了可迁移的工作状态：核心 Markdown、进度、governor 记录、current-state 和 artifacts 副本。
- 不建议直接复制本机 `~/.codex/state_5.sqlite` 到主机；那里是本机 Codex 状态库，容易把本机私有会话状态一起覆盖过去。
- 主机上要用可见线程时，优先用 `codex resume`，不要用 `codex exec resume`。
- `codex exec resume` 是后台非交互执行，适合自动督工，但你不容易看到和接管。
- `codex resume --include-non-interactive <thread_id>` 会尝试把旧线程打开成交互式 TUI；打开后就在主机本地历史里可见。
- 如果遇到 `The model gpt-5.5 does not exist or you do not have access to it`，说明主机 `~/.codex/config.toml` 里写了不可用模型；先把默认模型改成可用模型，或用本指南脚本的 `CODEX_MODEL=...` 覆盖。

## 主机第一步

在主机终端进入同步仓库：

```bash
cd ~/GaokaoPolitics/beijing-politics-sync
git pull --ff-only
```

如果主机路径不同，先进入实际 clone 目录。

## 辅助脚本

仓库里提供了一个单线程打开脚本：

```bash
./scripts/open_visible_thread.sh supervisor
```

脚本默认使用 `gpt-5.4` 避开不可用的 `gpt-5.5` 配置；如果你的账号需要用别的模型，可以这样覆盖：

```bash
CODEX_MODEL=gpt-5.2 ./scripts/open_visible_thread.sh supervisor
```

可选参数：

```bash
./scripts/open_visible_thread.sh supervisor
./scripts/open_visible_thread.sh philosophy-forward
./scripts/open_visible_thread.sh philosophy-reverse
./scripts/open_visible_thread.sh culture
./scripts/open_visible_thread.sh elective1
./scripts/open_visible_thread.sh elective2
./scripts/open_visible_thread.sh elective3
```

这个脚本一次只打开一个交互式线程。要同时看多个线程，就在多个终端窗口里分别运行。

## 如果旧线程打不开

如果主机报找不到 thread/session，说明这个 thread_id 没有同步到主机本地 Codex 状态库。此时不要硬拷贝 sqlite，直接新建一个可见接管线程即可：

```bash
codex -m gpt-5.4 -C ~/GaokaoPolitics/beijing-politics-sync "你是北京高考政治项目的可见督工接管线程。先 git pull --ff-only，然后阅读 skills/beijing-gaokao-politics-rubric/references/current-state.md、reports/continuous_jobs/哲学必修四_三线闭环穷尽满分课/PROGRESS.md、reports/governor_board.md 和 reports/主机可见线程接管指南.md。保持暂停，不要启动后台 exec；等待用户下一步。"
```

新线程虽然不是旧 thread_id，但会读取同一套 GitHub 状态，后续工作不会丢。
