# Claude Code v4 后续工作硬规则

更新时间：2026-04-27

## 用户确认的改正

1. `trigger_logic` 必须写知识触发链，不写程序性归档理由。
   - 要解释：材料里哪一句、哪一个词、哪一种关系，为什么能触发该原理/方法论。
   - 不要写：因为细则哪里提到了，所以可以放在这里。
   - “细则明确提到”属于准确性检验，放进 `rubric_excerpt` 或审计证据，不作为学生版触发逻辑。

2. 来源路径、行号、文件号、slide 号、entry id 只保留在检查性文件。
   - 学生终稿不呈现 `/Users/...`、`L24`、`F04`、`slide3`、`S015_F01` 等信息。
   - Claude Code 的 JSON 中，`question_excerpt`、`rubric_excerpt`、`answer_key_source`、`evidence_excerpt` 可以保留行号用于审计。
   - `material_signal`、`trigger_logic`、`correct_trigger`、`wrong_option_notes`、`boundary_note` 必须写成干净中文。

3. 最终文档必须让学生读得轻松。
   - 终稿按用户原框架节点组织，不按题目堆列表。
   - 终稿字段采用：知识点、材料触发点、为什么能想到这个原理、材料与设问、细则对应、必要边界。
   - 证据路径、阻塞项、未核 OCR、原文缺失、反推内容，进入审计证据版，不进入学生版正文。

4. 设问必须抄全，这是刚性原则。
   - 主观题进入终稿时，完整保留设问，不能只写题号或半截设问。
   - 材料不要求全文照搬，只摘与本条原理触发直接相关的重点句。
   - 同一题拆成多个原理时，各条都保留完整设问，材料触发重点按原理分别摘。
   - 选择题尽量保留完整题干和选项，至少不能漏掉正确项和被分析的错项。

5. 原理方法论必须回应设问，这是刚性原则。
   - `trigger_logic` 必须写成“材料触发点 -> 原理方法论 -> 回答设问”的完整链条。
   - 设问问“为什么要/何以/为何”，就要写出：因为材料体现某原理，所以能推出该原因或必要性。
   - 设问问“如何/怎样/怎么做”，就要写出：主体应按该方法论采取什么行动，从而回应做法。
   - 设问问“说明/体现”，就要写出：材料中的动作、关系、过程分别对应原理中的哪些要点。
   - 不能只说“触发某原理”，最后必须落回本题设问的答案方向。

6. “回应设问”必须写成答案落点，不能写成空泛要求。
   - 禁止写：“要回应设问”“要让原理服务题目”“把原理转化为做法”“说明材料体现该原理”这类元话术。
   - 必须写出：这个原理在本题答案中到底答哪一句、解释哪一个原因、推出哪一个做法、论证哪一个关系。
   - 建议格式：`答案落点：……`
   - 合格示例：材料中我国发布人工智能规划、确定基本原则和“三步走”目标，说明我国不是抽象地喊发展人工智能，而是根据人工智能发展的现实水平、趋势和阶段任务作出分阶段部署；所以回答“如何运用科学思维”时，要写“从我国人工智能发展实际出发，客观判断趋势，提前部署、分段规划，并在实践中检验和完善政策”。
   - 不合格示例：回到设问，要让“一切从实际出发”服务于题目要求，而不是停留在识别知识点。

## 后续运行要求

- 从 S018 继续逐套回源审计。
- 不复用旧 v2/v3 的正文判断。
- 每条主观题入框架必须有题干/材料摘录和细则/评标/阅卷摘录。
- 主观题 `question_excerpt` 必须含完整设问，材料只摘触发重点。
- `trigger_logic` 必须让原理方法论回应设问，并写成可直接进入答案的具体“答案落点”。
- 选择题必须有题面和可靠答案键。
- 纯选必三、经济、政治、法律、文化内容不能硬挂必修四哲学。
- 若材料原文缺失、OCR 不足、需要反推，记录 followup 或审计证据，不进入学生终稿。
