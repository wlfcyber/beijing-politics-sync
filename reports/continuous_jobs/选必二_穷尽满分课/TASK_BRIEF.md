## 研究边界

- 只研究 `选必二《法律与生活》`
- 要区分：
  - `选必二正式法律题`
  - `法律情境但主考点不在选必二`
  - `只有题面或答案 key 的低证据题`
- 不能把其他模块混入选必二母题链

## 核心方法

- 主观题：
  - 以 `细则`、`阅卷总结`、`评标细则`、`讲评中明确给分口径` 为核心证据
  - 建立 `材料 -> 给分点 -> 触发逻辑 -> 可复用表达` 链条
- 选择题：
  - 以 `试卷 + 可靠答案 key` 锁定正确答案
  - 提炼高频错项、易错表述、稳定知识边界
- 框架更新：
  - 所有稳定答题点必须回填到用户已有框架，而不是另起一套体系
  - 同时压出适合课堂讲授的“零基础 -> 能拿分 -> 能迁移”的表达顺序

## 完成标准

- 已对本机所有可访问的 `选必二《法律与生活》` 题源完成穷尽式盘点
- 每一道题都至少被归入以下之一：

- 主框架中已经覆盖所有高频母题、稳定答题链、意义收束词、细则共词和常见失分点
- 输出结果不能只是研究者笔记，还必须能直接转化成“给零基础学生上课”的内容
- 至少有一份文档明确做到：

  - `这类题怎么拆`
  - `这类题先写什么后写什么`
  - `哪些词反复出现`
  - `遇到新题怎么快速套母题`
- 只有当开发计划文档中的全部步骤都在进度文档中以完成状态出现时，任务才允许停止
