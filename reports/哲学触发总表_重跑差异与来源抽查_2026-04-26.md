# 必修四哲学触发总表：重跑差异与来源抽查

生成时间：2026-04-26

## 一、重跑版与旧终极版差异

本报告把“旧终极版”分两层处理：

- Git 正本 Markdown：`artifacts/必修四哲学材料-知识触发总框架_持续更新版_v2.md`
- 旧 Word：`artifacts/必修四哲学材料-知识触发总框架_持续更新版_v2.docx`

### 0. 更正结论：ClaudeCode 正文不是有效独立重跑

结论：ClaudeCode Markdown 正文与 Git 正本完全一致，应判定为“母版复制/转档完成”，不能判定为“独立重跑后验证完成”。

- Git 正本 Markdown：1838 行。
- ClaudeCode Markdown：1838 行。
- 二者 SHA256 完全一致：`5cd317ae263fbb51d68039898ae0ed2127e63f4c699610185ce7591357ee835e`。
- `cmp=0`，即字节级一致。
- ClaudeCode `PROGRESS.md` 明写“母版已复制”。
- ClaudeCode `DECISION_LOG.md` 明写“以母版为起点复制到 outputs 作为本轮草稿，在此基础上补证、纠错、增边界说明”。
- 但最终 Markdown 没有形成任何补证、纠错或新增边界说明的字节级差异。

因此，本轮 ClaudeCode 不能作为第二套独立正文意见使用。它的价值只在于：确认母版被安全复制、转成 Word，并补了一部分控制层记录；不能作为“另一个模型重跑后给出的不同内容”来与 Codex 版本融合。

### 1. ClaudeCode Markdown vs Git 正本 Markdown

结论：无正文差异，且这不是“高度一致”，而是“同一正文”。

如果“之前的终极版”指 Git 正本 Markdown，则 ClaudeCode 产物没有新增、删减或改写任何条目。后续不能把它当成独立重跑结果引用。

### 2. ClaudeCode 完整重跑 Word vs 旧 Word

结论：Word 形态有明显差异，但差异来自“新 Word 由当前 Git 正本转档”，不是来自 ClaudeCode 独立改写正文。

- 旧 Word：约 109 KB；抽取文本约 64254 字符。
- ClaudeCode 完整重跑 Word：约 149 KB；抽取文本约 128487 字符；渲染 74 页。
- Cowork 控制版 Word：约 63 KB；抽取文本约 43652 字符；渲染 30 页。

旧 Word 尾部停在“2026海淀期中第22题评分细则链”附近，未包含当前 Git 正本后续追加的大批套卷级三线闭环进展。ClaudeCode 完整重跑 Word 则包含当前 Git 正本全量 1838 行内容。

### 3. Cowork 重跑版 vs Git 正本

结论：Cowork 版不是完整主表，只能当控制附录/验收账。

- Cowork Markdown：896 行。
- 它新增了 Cowork 头部、控制文件清单、覆盖矩阵与证据边界、角色降级、cache-first 记录。
- 它删去了 Git 正本中 `## 更新日志`、`## 后续更新规则`、`## 接管记录与迁入备查`、大量“本轮进展（套卷级三线闭环）”附录。
- 所以完整主交付应使用 ClaudeCode 1838 行版；Cowork 版只作监督账。

## 二、来源抽查原则

本轮只按硬证据验：

- 主观题：必须在细则、评标、阅卷总结、讲评给分口径中出现该原理名或明确等价表达。
- 选择题：必须有可靠答案表，且正确项本身明确包含该原理或等价判断；不能把普通材料感想硬升格为原理方法论。
- 若来源只列“可从某角度作答”而没有逐句材料对应，标为“可用但需注明材料链为整理者解释”。

## 四、当前风险判断

本轮抽查没有发现上述样本存在“答案里没有该原理，却被强行加进材料—原理链”的硬错误。

但发现三个必须长期守住的风险点：

1. Cowork 896 行版不能当完整主表，否则会漏掉后半部套卷级进展。
2. 选择题正确项触发只能标“选择题正确项支持”，不能写成“阅卷细则明确给出”。
3. 像 2026西城期末第21题这类评标 PPT 只给“可从……角度作答”，材料到原理的细链是整理者解释；可以用，但必须如实标明证据层级。

## 五、建议的下一步

对全表做一次“来源强度分级”：

- A 级：细则/评标/阅卷总结明写原理及材料点。
- B 级：细则/评标只列角度，材料链由整理者补齐。
- C 级：选择题正确项触发，有可靠答案表。
- D 级：reference-only，仅作方向，不升格为细则。

这样以后人工抽查时，可以直接优先审 B/C/D 级，避免把模型解释误当成官方给分点。

## 六、Codex 与 Claude 的融合口径

当前不能把 ClaudeCode 正文与 Codex 正文做“内容融合”，因为 ClaudeCode 正文并没有提供独立内容。可融合的是三层：

1. 正文底稿：继续以 Git 正本 `artifacts/必修四哲学材料-知识触发总框架_持续更新版_v2.md` 为唯一底稿。
2. 控制层：吸收 Claude/Cowork 已落地的 `SOURCE_LEDGER.csv`、`COVERAGE_MATRIX.csv`、`GOVERNOR_CHECKLIST.md`、角色报告等，用来检查覆盖面和流程门。
3. 证据层：新增“来源强度分级”和“逐条抽证状态”。只有被再次核过的条目，才能从“原有正文”升级为“融合确认版”。

融合后的验收标准应改为：

- 每条不是只看有没有来源名，而是要能回到原文证明“该原理/方法论确实出现在答案、细则、评标或正确项中”。
- 若来源只给角度清单，正文必须标注“材料链为整理者解释”。
- 若来源来自选择题，必须标注“选择题正确项触发”，不能混成主观题细则。
- Claude 后续若要继续参与，只允许输出“差异清单/纠错清单/证据评级清单”，不再允许直接复制母版后报告重跑完成。
