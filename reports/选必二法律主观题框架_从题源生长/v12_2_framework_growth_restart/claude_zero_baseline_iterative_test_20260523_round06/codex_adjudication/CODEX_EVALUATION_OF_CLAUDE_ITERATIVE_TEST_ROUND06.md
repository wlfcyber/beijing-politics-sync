# Codex Evaluation of Claude Zero-Baseline Iterative Test Round06

时间：2026-05-23 22:25 +08:00

## 结论

Round06 是真实 Claude Opus 4.7 Adaptive 网页输出。Claude 只收到 v13.6 框架和题面压缩信息，未收到隐藏答案钥匙。

v13.6 解决了合同责任并列和双链基本配比，但 Claude 仍明确要求 v13.7。Codex 复核认为这不是主轴缺陷，而是 B1 表格题与双链题的最后一段行内精度问题：

- 表格列名“部分可见/示例可见”时仍要靠学生猜。
- 双链题“略偏主诉求链”的量化不够清楚。
- 表格行内出现“启示/价值/意义”列时，是否要从裁判要点升级到价值收束，规则不够硬。

因此 Round06 判定为 `not_final_requires_v13_7_precision_patch`。

## v13.7 必要补丁

1. B1 列名口令映射表：把“了解案件、分析事实、印证法理、参考示例”等常见设问固定映射成可写列名和内容形状。
2. 双链题量化规则：无分值但有主诉求/副链时按 6:4；没有主副时按 5:5；主链比副链多写一个责任方式或边界句。
3. 表格“启示/价值/意义”列触发规则：列名出现启示/意义/作用/价值时，最后一格必须写“规则保护什么、约束什么、形成什么秩序”，否则只写裁判要点。
