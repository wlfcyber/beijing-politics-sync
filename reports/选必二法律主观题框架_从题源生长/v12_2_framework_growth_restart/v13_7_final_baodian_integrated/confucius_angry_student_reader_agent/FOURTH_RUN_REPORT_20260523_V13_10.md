## 1. Verdict

结论：v13.10 已经基本像一个能交给学生的框架了，不再只是分类表。

我不高兴的是，它还是太长，且正文出现乱码/编码问题时会直接打断学习。但从结构上看，它终于有了三件学生真正需要的东西：

1. 先定 A 轴生活冲突，不让 A1、A10 抢戏。
2. 再定 B 轴设问动作，不把所有题都写成价值题。
3. 混合题按“先实体、后程序；先主链、后副链；先责任、后意义”排序。

这不是完美学生讲义，但已经不是“看完会贴标签、不会写答案”的假框架。剩下主要问题属于篇幅、记忆负担、个别法律细节、题源细节限制，不是框架结构断裂。

## 3. One-Page Student Memory Map

我读完后能留下来的地图如下：

1. 先问“谁和谁，为了什么生活利益冲突”。这决定 A 轴。
2. A1 管行为能力、代理、效力、时效，但除非题目直接问效力，不许抢主入口。
3. A2 管人格权：隐私、个人信息、名誉、肖像、生命健康。
4. A3 管物权、共有部分、相邻关系、采光通行安全、业主共同决定。
5. A4 管合同：约定、交付、质量、迟延、退费、违约金。
6. A4+A6 管瑕疵商品/服务造成损害：先合同违约，后侵权/损害赔偿。
7. A5 管知识产权、商业秘密、数据抓取、不正当竞争；劳动离职只是背景。
8. A6 管侵权：损害、过错/违法、因果、责任。
9. A7 管婚姻家庭继承。
10. A8 管劳动关系：录用、试用期、解除、竞业、社保。
11. A9 管消费者和经营者：未成年人消费、虚假宣传、格式条款、预扣款、安全保障。
12. A10 管证据、调解、仲裁、诉讼、维权路径。除非题目问路径，否则不抢实体题。

B 轴：

1. B1 表格/裁判要点：一行一链，事实 -> 规则 -> 裁判要点。
2. B2 判决理由/责任：法律关系 -> 规则 -> 事实 -> 结论 -> 责任。
3. B3 诉求能否支持：先结论，再逐项写权利基础、要件、事实、责任方式。
4. B4 评析观点：合理处 -> 片面处 -> 正确法律立场。
5. B5 意义价值：保护什么、约束什么、形成什么秩序。
6. B6 维权路径：证据 -> 协商/调解/仲裁/诉讼 -> 请求。
7. B7 法律问题识别/填空：短语，不写长判决。

混合题排序：先实体后程序，先主入口后副入口，先必得分后价值，题干顺序优先。

## 5. Where I Got Stuck

1. 编码/乱码是真问题。作为学生我能猜出大部分意思，但这会降低低摩擦程度。若正式交付仍是这种显示效果，直接扣交付分。
2. v13.10 仍然厚。框架结构对了，但学生不会背整章，必须配“一页 A/B 入口卡 + 三类混合题卡”。
3. B1 表格题仍依赖原表头。v13.10 给了默认三列/四列，已经能救场，但真实题目列名不同，学生仍要服从题目。
4. B3 的“支持/部分支持/不支持”有尺度，但没有足够多的“典型结论句”。比如未成年人打赏到底全退、部分退、合理分担，框架不该乱猜，但学生会焦虑。
5. A9 和 A4 在“商业购买/消费购买”边界上还可再加一句：为商业经营购买时，消费者保护可能不当然适用，先按合同。
