# ACCEPT

## 1. 接受「完成表格 / 一格一链 / 一案一原则」

### 判断

接受，且应进入核心主干。

Claude 的这个入口证据很强。它抓住了北京选必二法律主观题里非常重要的一类命题方式：

```text
完成下表
参照示例
完成裁判理由
多个案例分别处理
```

v11.1 也已经把“裁判理由 / 补表 / 完成理由”列为强分诊入口，并特别强调只有出现“完成裁判理由/补表/处理结果已给定”时才能进入，评析题不能误入。

### 支撑题

可以支撑这个入口的题包括：

```text
CC0077
CC0084
CC0137
CC0157
CC0180
CC0189
CC0213 / CC0214
CC0289
CC0325
```

其中 `CC0137` 已在 v11.1 中被修正为 AI 著作权格 + 信用卡合同格，明确是一格一链，不能只写 AI 或创新价值。

### 处理建议

保留，但名称建议压缩成飞哥式：

```text
表格/裁判理由题：一格一链
```

不要写成“入口①”这种 AI 编号腔。

---

## 2. 接受「诉求 / 请求能否支持 + 理由」

### 判断

接受，作为核心入口之一。

这类题学生必须先表态，再写法律依据、材料事实和责任结果。Claude 这一点抓得对。

### 支撑题

强支撑包括：

```text
CC0143：消费者合同欺诈，法院支持诉讼请求
CC0305：隐私权链 + 消费者欺诈链，诉求支持
CC0373：性别就业歧视 / 竞业限制诉求
```

`CC0143` 的材料核心是机票代销平台隐蔽搭售，消费者无法清楚知悉或有效拒绝，争点是合同成立后是否可撤销及三倍赔偿，这明确不是平台用工，而是消费者欺诈。

### 处理建议

保留，但不要写成“能支持/不能支持双链并行”固定模板。不是每道诉求题都有“部分支持、部分不支持”。更稳的写法是：

```text
诉求题：先判，再证。
支持什么，为什么支持；不支持什么，为什么不支持。
```

---

## 3. 接受「评析 / 认识 / 谈谈」中的“一主体一链”

### 判断

部分接受。
“评析行为”这一块必须进入主干。

`CC0238` 是最强证据：真实设问是“评析乙公司和张某的行为”，v11.1 已经从错误的“裁判理由”修正为“评析题，一主体一链”。

### 支撑题

```text
CC0238：乙公司一链，张某一链
CC0200：未成年人、平台、监护人、小刘本人，多方过错
CC0332：校园欺凌，惩教并行
```

### 处理建议

接受“评析行为 = 一主体一链”。

但 Claude 把“认识 / 谈谈 / 评析”放在一起太宽。应拆成：

```text
评析行为：一主体一链
认识制度/规则：规则是什么、解决什么问题、产生什么价值
```

否则“评析乙公司和张某”与“谈竞业限制制度的原因”会被混成一个入口。

---

## 4. 接受「法律建议 / 维权准备和路径」

### 判断

接受，但只作为二级主干，不作为最核心 6 主干之一。

这类题确实存在，比如无人机维权准备、调解题、权利保护路径题。Claude 的“定关系 → 选路径 → 备证据”是对的。

### 支撑题

```text
CC0244：无人机法律责任及依据
CC0245：维权准备和路径
CC0125：调解方案和理由
CC0289：任选权利保护路径
```

`CC0245` 的题链明确是“维权准备和路径”，要求选择调解、仲裁或诉讼，并准备合同订单、邮件转账、无人机故障、伤害和损失等证据。

### 处理建议

保留为：

```text
问怎么解决 / 如何维权：路径 + 理由 + 证据
```

不要单独扩大成“所有程序救济题”的万能入口。

# MODIFY

## 1. 修改「判决 / 裁判理由」

### 问题

Claude 把“判决/裁判理由”写成：

```text
程序合法
实体定性
权利与限制平衡
```

这有两个问题：

第一，**程序合法不是多数题的高频得分主轴**。
很多题的核心是私法规则和材料事实，不是管辖、起诉资格或程序合法。

第二，它把三类题混在一起：

```text
裁判理由 / 补表
判决结果 / 责任分担
诉求能否支持
```

v11.1 已经专门把这些分开：有“问诉求能否支持/法院为什么支持”“问判决结果/责任分担”“问裁判理由/补表/完成理由”三个入口。

### 修改建议

改成三个更精准入口：

```text
1. 裁判理由/补表：一格一链
2. 诉求能否支持：先判，再证
3. 判决结果/责任分担：多方过错 + 公平原则
```

### 证据

`CC0200` 是判决结果/责任分担题，不是普通裁判理由题；v11.1 已修为“未成年人网络打赏效力、多方过错与公平原则责任分担”。

---

## 2. 修改「判决 / 典型案例意义」

### 问题

Claude 的“个体层面—行业层面—公共层面”三层意义，有一定教学价值，但容易把意义题写成固定三段套话。

你的框架要求不是“价值三层套”，而是：

```text
规则保护谁
规范谁
救济谁
维护什么秩序
```

v9 的先导也明确：意义题不能裸写价值，价值必须从法律规则和材料事实里长出来。

### 修改建议

并入 GPT Round 01 的主干：

```text
规则生价值
```

具体写法：

```text
规则 → 材料问题 → 价值结果
```

不要固定成“个体、行业、公共”三层。
有的题适合三层，有的题不适合。

### 证据

`CC0229` 和 `CC0283` 都是“一案一手段，一手段一价值”的多案例意义题，不是泛泛三层意义。v11.1 已把 `CC0229` 修为三案例法律手段链：权属确认、惩罚性赔偿、规制恶意诉讼。

---

## 3. 修改「司法 / 法律如何保护、守护、推动」

### 问题

Claude 的“手段—对象—效果”是有用的，但入口名太大：

```text
司法 / 法律如何保护（守护、推动）××
```

这个名字很危险，容易重新必修三化。

比如学生看到“法院如何保护知识产权”，如果不先写具体法律手段，很容易写成：

```text
坚持公正司法
推进全面依法治国
营造法治环境
```

### 修改建议

降级为“意义题内部写法”，不要作为独立入口。

改为：

```text
意义题的高级写法：法律手段 → 保护对象 → 价值结果
```

尤其用于：

```text
知识产权护航创新
民法典推动绿色发展
集体协商平衡公平效率
```

### 证据

`CC0283` 的题源非常典型：人民法院以“调、惩、辩”三种法律手段保护知识产权、护航新质生产力，不是泛泛写司法保护。

---

## 4. 修改「Teaching Sequence」

### 问题

Claude 的教学序列：

```text
识题型
标主体
找材料触发
选权利义务/法律链
写事实+法律+后果
检查避坑
```

方向对，但这不是框架主干，而是“上课流程”。

### 修改建议

保留为教师授课建议，不进入学生最短记忆版。

学生版应更短：

```text
先看设问
再翻材料
最后写规则 + 事实 + 结论
```

v9 的“三步六问”已经更贴近你的课堂风格：先定题型，再翻材料，最后写得分句。

# REJECT

## 1. 拒绝「法律规则解释 / 识别法律问题 / 调解理由 / 减责理由」作为一个主干入口

### 原因

这个入口太杂。

Claude 把这些东西放在一起：

```text
法律规则解释
识别法律问题
调解理由
减责理由
```

它们在题源中不是同一种入口：

```text
CC0002：好意同乘减责
CC0125：调解理由
CC0092：识别公共区域电池柜法律问题
CC0181：竞业限制规则解释
CC0223：多类纠纷解决
```

这些题的共同点只是“要先说规则再用到本案”，但这几乎等于所有法律题的共同点。作为独立主干会泛化。

### 处理建议

拆掉，不保留“入口⑧”。

分别归入：

```text
好意同乘减责 → 责任分担 / 侵权减责
调解理由 → 路径 + 理由 + 材料
识别法律问题 → 事实转法词
竞业限制解释 → 规则生价值 / 劳动权益平衡
```

---

## 2. 拒绝把“程序合法”作为判决理由题的固定第一步

### 原因

多数私法题不考程序合法。强行加这一层，会把学生带向必修三或诉讼程序泛化。

### 处理建议

改成：

```text
裁判理由题：处理结果 → 法律规则 → 材料事实 → 结论
```

只有材料或细则明确给程序、举证、管辖、调解、诉讼路径时，才写程序。

# Claude 遗漏的强支撑点

## 1. 遗漏「事实转法词」作为第一主干

Claude 虽然在 Teaching Sequence 里提到“材料触发”，但没有把它升格为主干。

这不符合题源特征。

大量题真正考的不是“入口类型”，而是学生能不能把材料里的生活话翻译成法律话：

```text
隐蔽搭售 → 欺诈 / 知情权 / 自主选择权
派单签到奖惩 → 三从属性
未经许可使用作品 → 著作权侵权
高额打赏 → 限制民事行为能力人 + 多方过错
抢注唤醒词 → 不正当竞争
```

这个动作是选必二主观题的中心。v9 先导也明确，法律题考的是把生活纠纷翻译成法律得分点。

### 处理

必须进入最终候选主干：

```text
事实转法词
```

---

## 2. 遗漏「规则—事实—结论链」作为总骨架

Claude 每个入口内部都有“事实 + 规则 + 后果”，但没有把它抽成全框架底层骨架。

这会导致框架像 8 个入口，而缺少统一作答核心。

### 处理

必须保留 GPT Round 01 的主干：

```text
规则—事实—结论链
```

它可以作为所有理由题、判决题、诉求题的共同底层。

---

## 3. 遗漏「反向筛查」作为学生框架的一等结构

Claude 每个入口都有易错点，但没有把反向筛查系统化。

你的旧框架里，“正向触发 / 反向筛查”是核心方法，不是点缀。旧框架风格 DNA 已明确：旧框架不只是告诉学生写什么，还告诉学生别写什么，且语言要有裁断感。

v9 的反向筛查页已经形成很好的基本盘：

```text
看到法院 ≠ 必修三公正司法
看到法治 ≠ 全面依法治国
看到企业 ≠ 经济生活
看到消费者 ≠ 只写诚信经营
看到平台 ≠ 新业态高质量发展
看到意义题 ≠ 裸写价值
看到案例 ≠ 法考
看到多个主体 ≠ 一句都不诚信
```

这些应进入候选框架，而不是散落在每个入口的易错点里。
