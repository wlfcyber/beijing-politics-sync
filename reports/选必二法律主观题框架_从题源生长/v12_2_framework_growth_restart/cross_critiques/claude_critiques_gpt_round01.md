> 本轮仅对 GPT Round 01 的框架结构、节点定位、证据引用做批判。所有判定回指证据卡或证据理由。本轮不输出终态信号。

## ACCEPT（接受并纳入 Claude 框架修订）

### A1. 「先定设问交付物」作为预备步骤
- **GPT 主干 1** 把"理由 / 评析 / 意义 / 路径 / 诉求判断"作为题型识别的总入口，比直接跳进答题脚手架更稳。
- 我 Round 01 的 TEACHING_SEQUENCE Stage A 已隐含这一步，但没有提升到主干。GPT 把它前置是对的。
- **证据**：CC0019、CC0025、CC0077、CC0125、CC0137、CC0200、CC0238 这一组中，每张卡的"当前主入口"字段都精确对应一个交付物动词；GPT 抓到了这条共性。
- **接受方式**：作为 Stage 0 / 预备步骤，但不替代题型分类——题型分类仍要在 Stage 1。

### A2. 「事实转法词」作为独立教学动作
- **GPT 主干 5** 把"派单签到 → 三从属性""隐蔽搭售 → 欺诈""唤醒词抢注 → 不正当竞争"显式化为独立技能。
- 我 Round 01 的 Stage C "找材料触发"是隐式的，GPT 显式化更易教。
- **证据**：CC0025、CC0143、CC0157、CC0206、CC0305 都是典型的"生活语→法律语"训练材料。
- **接受方式**：纳入 TEACHING_SEQUENCE 的 Stage C，并升格为可单独训练的技能模块（不升为主干题型）。

### A3. 「命题人路径」六条路径分析
- GPT 第 4 节六条命题人路径（法院判决载体 / 表格补链 / 新情境包装旧规则 / 多主体责任分配 / 意义设问逼规则 / 模块边界挖坑）是元层分析。
- 这一节不与我的主干冲突，可以作为**老师备课视角**的辅助层，帮老师识别哪类题在挖坑。
- **接受方式**：作为框架附录或老师手册的一部分。

### A4. 关于 6 张下一版回填卡的保守立场
- **GPT 红线 8**「不把 6 道 next backfill 当已闭合」与我 Round 01 EVIDENCE_GAPS 完全一致。
- **接受**：CC0276、CC0277、CC0251、CC0317、CC0318、CC0319 一致不进 Round 02 正文。

### A5. 「不把法院题写成必修三」红线
- GPT 红线 10-14 警示"看到法院 / 平台 / 消费者 / 新技术"就套必修三模板，这是 v10 时代的真实失败模式。
- **接受**：补入 Claude Round 01 OVER_ABSTRACTION_RISKS 节。

### A6. STUDENT_STARTERS 中的口语启动句
- "别被合作协议骗，先看派单、签到、奖惩、结算、业务组成"（CC0025、CC0143）
- "没有欺诈别乱写惩罚性赔偿"（CC0063、CC0143、CC0305）
- "先判权利对象，再判侵害方式，最后再写创新价值"（CC0131、CC0229、CC0283）
- **接受**：这三句直接吸收进 Claude TRANSFER_LANGUAGE。

## MODIFY（接受方向但需修改）

### M1. GPT 主干 3「一格一链」与主干 4「一主体一链」存在重叠
- **问题**：两条主干都用"不要糊成一锅"作核心警告。CC0084、CC0200、CC0238 同时被列入两个主干。
- **证据**：CC0084 既是表格题（三按钮），也是多主体题（经营者+自媒体+平台）——但学生只走一条路径，不需要走两次。
- **修改建议**：
  - 「一格一链」专管**形式触发**（题面有表格 / 多问 / 任选其一）。
  - 「一主体一链」并入「一格一链」的子规则——当题面无表格但材料里有多主体时，自我构造表格。
  - 不再两套主干并列。

### M2. GPT 主干 2「规则—事实—结论链」与主干 5「事实转法词」边界模糊
- **问题**：两条都讲"事实→法律"，但主干 2 走全链（事实+法+果），主干 5 只走半步（事实→法词）。学生看不出何时用哪条。
- **证据**：CC0143 同时被列入 2 和 5，但 CC0143 的答题骨架是完整三链，不只是"隐蔽搭售→欺诈"。
- **修改建议**：
  - 「事实转法词」降级为 Stage C 的训练技能（如 A2 所述）。
  - 「规则—事实—结论链」保留为主干，但需重命名为「答题三链：事实 + 规则 + 后果」并补充"必要时第四段写价值"。

### M3. 「规则生价值」覆盖 5 张证据卡，但收口过窄
- **问题**：GPT 把意义题 / 价值题 / 作用题 / 推动题 / 守护题全装进"规则生价值"一筐。
- **证据**：
  - CC0103 / CC0229 / CC0283 是「一案一手段一价值」结构，需要的是「调、惩、辩」三段，不只是"规则→价值"。
  - CC0019 / CC0025 是「典型意义」，要分三层（保护当事人 + 规范同类 + 维护秩序）。
  - CC0011 / CC0340 是「制度作用」，要落到具体物权 / 公益诉讼规则。
- **修改建议**：拆分为两条：
  - 主干 A：**判决/典型案例意义**（CC0019、CC0025、CC0045、CC0063、CC0103、CC0150 等）。
  - 主干 B：**如何保护/推动/守护**（CC0011、CC0195、CC0229、CC0283、CC0289、CC0340 等）。
- **理由**：意义题与"如何保护"题的句式不同——前者收三层（个体/行业/公共），后者收三段（手段+对象+效果）。GPT 把它们压扁。

### M4. CC0195（集体协商）被降为「容器 3 综合经济+法律题」
- **问题**：GPT 把 CC0195 仅当作综合题处理，但其证据卡有完整法律链：集体协商机制 + 协商核心议题（工资/加班/劳动定额）+ 平衡公平效率。
- **证据**：CC0195 的"现有答题骨架"字段明确指向「集体协商是依法表达劳动者诉求的机制 + 调薪/加班/工资支付/劳动定额协商 + 平衡公平与效率」三链结构，属于典型的「如何保护/推动」题型。
- **修改建议**：CC0195 应作为 M3 拆出的「如何保护/推动/守护」主干的核心证据卡，**不应仅在容器**。

### M5. CC0051（婚姻法变迁）被降为「容器 4 法律变迁/良法评析」
- **问题**：GPT 把它当容器，理由是"容易必修三化"。但 CC0051 的禁止误判明确警告"一句话判 1950 年错误"，并要求"先表态：片面 + 两个时代各放回各自语境"——这是典型评析题脚手架。
- **证据**：CC0051 + CC0238 + CC0254 + CC0332 + CC0061 + CC0340 一组卡均要求「定性 + 论证 + 落地」三段评析。
- **修改建议**：增设主干「认识 / 评析 / 谈谈题」，覆盖 CC0051、CC0061、CC0238、CC0254、CC0332、CC0340 等。GPT 把"评析"塞进"一主体一链"是错位——CC0061（督促履行）、CC0254（法治温度）、CC0051（婚姻法变迁）都没有"评析甲乙双方"结构。

### M6. CC0019 在主干 5 被简化为单链
- **问题**：GPT 写"拖欠货款、码上诚信 → 诚信原则"。
- **证据**：CC0019 的材料触发三链分别是：① 合同诚信（B 公司拖欠货款 → 法院判付清+违约金）；② 司法诚信（购物中心"码上诚信"公示信用 → 消费者放心消费）；③ 经营者诚信（依法诚信经营 → 保护消费者）。三链不能合并。
- **修改建议**：CC0019 在事实转法词训练中**必须分三链展开**，不得收口为"诚信原则"一条。

### M7. CC0011 在主干 6 被简化为「绿色原则」
- **问题**：GPT 仅写"民法典绿色原则和生态保护"。
- **证据**：CC0011 实际有三链：① 知识产权（污水分离技术专利） → 绿色技术应用；② 长期超标排污 → 民事公益诉讼 → 停止侵害+消除危险+赔偿；③ 法律责任落到修复和预防。其中第二链是公益诉讼制度独立要点。
- **修改建议**：CC0011 在主干 B（如何保护/推动/守护）下要呈现「三链并行」，不可只取绿色原则。

### M8. 「主干 3 一格一链」遗漏 CC0325（举证责任分配三场景）
- **问题**：GPT 主干 3 列了 CC0077/CC0084/CC0137/CC0157/CC0289，没有列 CC0325。
- **证据**：CC0325 是举证责任分配的「三案例参照」结构——公诉案件（检察院举证）/ 行政诉讼（行政机关举证）/ 民事侵权（原告举证+建筑物使用人证明无过错），典型的一格一链题。
- **修改建议**：补入 CC0325 与 CC0213-CC0214（参照示例）。

## REJECT（拒绝采纳）

### R1. 拒绝「6 个高频作答动作 = 全部主干」的收口
- **理由**：6 个动作（设问交付物 / 三链 / 一格一链 / 一主体一链 / 转法词 / 规则生价值）虽然漂亮，但**不能覆盖以下题型的独特脚手架**：
  - **法律建议 / 维权路径题**（CC0244、CC0245）：要"定关系 → 选路径 → 备证据 → 提诉求"，不是"规则—事实—结论"能套的。
  - **法律规则解释 / 识别法律问题题**（CC0092、CC0180、CC0181、CC0223、CC0125）：要"先讲规则定性，再说本案落入哪个构成要件，再说规则保护谁 / 限制谁"。
  - **判决意义题** vs **如何保护题**：句式不同（见 M3）。
- **证据**：CC0244 的"现有答题骨架"明确写"先确定双方是什么法律关系。/ 再看行为违反了什么义务、侵害了什么权利。/ 最后写应承担的法律责任和依据"——这与 GPT 主干 2 三链不同，是「定关系」前置。
- **结论**：6 主干无法关闭题型缺口；至少需要 8 主干（详见 CLAUDE_REVISED_FRAMEWORK_DELTA）。

### R2. 拒绝把 CC0195 放进容器
- **理由**：见 M4。证据卡有完整法律链，不应因"综合经济+法律"标签被边缘化。"综合题"不等于"开放容器"。
- **替代处置**：CC0195 升入主干 B（如何保护/推动/守护）。

### R3. 拒绝把 CC0051 放进容器
- **理由**：见 M5。CC0051 评析题脚手架明确，容器化会让学生不知评析题怎么写。

### R4. 拒绝「学生第一步」表述过度口语化的若干句
- 「这题不是让我背一章法律，而是让我交一个"理由/评析/意义/路径/判决结果"」——这句话学生在考场上想没问题，但**作为框架文本它没有给出具体题型映射**。如果学生记住"交付物有 5 种"但不知每种对应哪套写法，他识别完依然抓瞎。
- 「这题问价值，但价值要从规则里长出来」——这是反向警示（不要裸价值），不是正向脚手架（怎么写）。
- **理由**：启动句必须配对脚手架，单独的启动句对中等及以下学生帮助有限。
- **替代**：每条启动句必须紧跟「然后做什么」的具体动作清单。

### R5. 拒绝把 CC0143、CC0305 当作"消费者欺诈双链"的两个独立证据
- **GPT 在主干 2 + 主干 5 都列 CC0143**，但只用其中"隐蔽搭售→欺诈"一链。
- CC0305 同样如此：GPT 只取"包间监控→隐私"和"虚假宣传→欺诈"两链。
- **理由**：CC0305 的"现有答题骨架"要求**双链并行展开**（隐私权侵害链 + 消费者欺诈链），不是择一。GPT 抽取主干例证时把双链拆碎了。
- **替代**：保留双链整体性，作为"双轨题"独立训练样本。

### R6. 拒绝主干 1「设问交付物」作为**唯一总入口**
- 它可以作为 Stage 0 预备步骤（见 A1），但不能替代题型识别。
- **理由**：识别"我要交什么"只是第一刀；学生还需要第二刀（确定题型）和第三刀（套用脚手架）。GPT 只切了第一刀就声称"放在框架最前面"，等于让学生举着空篮子去捡题。
