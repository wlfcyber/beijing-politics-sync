## 1. METHOD_LEARNING_NOTES（旧框架为什么对零基础学生好用）

读了 `PRIOR_FRAMEWORK_DEEP_DNA_20260520.md`、`LEGAL_REWRITE_SPEC_AFTER_PRIOR_STUDY_20260520.md` 和 `rendered_prior_samples/`（经济习题课、必修二五四三二一、主体积累页、选必一、逻辑与思维）。旧框架真正可教的地方有六条：

**(1) 先导页先讲"这本书在考什么、你会死在哪里"，再进框架。**
经济习题课第一页（prior_sample_01）不写知识点，先手写一个 `A —→ B` 基本模型。零基础学生第一眼得到的不是目录，而是"这门题的本质动作"。学生最大的恐惧是"我不知道从哪下笔"，先导页先把恐惧点名。

**(2) 有一个"基本模型 + 衍生题型"的生成器。**
prior_sample_01 的灵魂：`A→B`，然后衍生出"针对 A——应如何""针对过程——是如何""针对 B——为什么""注意复合型/意义作用"。学生学的是**一个会生小孩的模型**，不是 27 个孤立例题。设问尾词是接口：尾词决定你走 A、过程还是 B。

**(3) "五四三二一"把全书压成一张能背的地图。**
必修二五四三二一（prior_sample_13-18）不是散点知识，是一张总图。压缩的目的不是漂亮，是让学生脑里有"全书就这么大"。

**(4) 主体积累页：谁出场，就写哪句。**
经济"应如何类考题-主体积累页"（prior_sample_19-20）按主体（国家/政府/生产/分配/劳动者）列**能直接抄上卷面的句子**，还带"飞哥想说"这种口语教练旁注。它不是词表，是"主体一出现，手就知道写什么"。

**(5) 语言是"学生脑内操作语言"，不是"教师审计语言"。**
旧框架说"看到……先想……""先别写……""这一题不是问……而是在问……"。它在掰学生的脑袋，不是在归档证据。

**(6) 排版顺序固定：先导→总模型→速通→核心动作→主干地图→主体积累→情境触发→逐题演练（材料一句话/争点/第一判断/翻译表/满分句/完整答案/错法改法）→错法改法→教师证据附录放最后。**
证据等级、题号、细则原子是后台，绝不污染学生前台。

**一句话**：旧框架好用，是因为它先给"一个会自我繁殖的模型"，再给"主体/情境两本可抄的账"，最后才逐题演练；而且它用教练口吻说话。V7 必须继承这套"生成器 + 两本账 + 教练语言"，不能继承旧框架的任何旧法律结论。

## 2. WHY_V6_9_STILL_NOT_ENOUGH（狠批 V6.9）

V6.9 比 V6.2 干净（材料翻译表已填实、主卡/辅卡黑话已清、表格模板已改"列名优先"），文本卫生扫描全 PASS。但用户的抱怨"学生读完仍不能稳定满分"是对的。原因不在排版，在骨架：

**(1) 前台主轴选错了——"九个战场"本质是教材目录。**
框架硬约束明确写：`Do not use textbook chapter names as the front-stage framework`。可 V6.9 的 27 核心题恰恰按"合同成立与履行 / 消费交易 / 劳动用工 / 实体责任 / 创新竞争 / 程序救济 / 价值收束……"九个战场排。这九个名字就是民法典分编+劳动法+反不正当竞争法的目录改了个名。考场上零基础学生看到陌生题，脑子里冒出来的不是"这属于劳动用工战场"，而是"这个设问问的是**原因**，我该怎么下第一笔"。V6.9 把"题目属于哪一章"放在第一层，把"我的笔要产出什么"放在第二层（三型答案骨架被压在第 74 行一个小表里）。主轴和考场启动顺序是反的。

**(2) 没有"生成器"，只有"分类表 + 27 个样例"。**
旧框架的命脉是 `A→B + 衍生题型` 这个会繁殖的模型。V6.9 有"30 秒判题门"6 行表和"三型答案骨架"3 行表——这是**分类法**，不是**生成器**，而且只占两节。文档 90% 的体量是 27 道题的逐题精解。结构重量完全压在样例上。学生背完 27 题，遇到第 28 题（陌生题）依然没有"自动生成答案"的引擎。这正是 V5.9 被否定的"背样章"病，V6.9 只是把样章做得更精致。

**(3) 删掉了"一二三四五"总图。**
文本卫生扫描显示 `一二三四五: 0`。旧框架 DNA 第 4 节明确说这种压缩"是为了让学生记住全书地图"。V6.9 为了"干净"把全书地图删了，学生失去了"法律主观题总共就这么大"的安全感。

**(4) "三大责任逻辑链"这个真正的引擎，证据里有，V6.9 没把它抬到前台。**
CC0244 的阅卷报告（`R_CC0244_2026_东城_期末_18_08`）白纸黑字给了老师自己用的生成器：`违约逻辑：合同成立+合同有效+存在违约行为→违约责任；侵权逻辑：权利受保护+侵权要件→侵权责任；不正当竞争逻辑：竞争关系+不正当竞争行为→不正当竞争`。这三条链能直接生成 BATCH_01/02/03 里大半道题的骨架。V6.9 把它打散进各个战场，没有作为一个独立的、最高层的引擎页。

**(5) 归责档位（过错/过错推定/无过错）散落各处，没有统一硬卡。**
CC0244、CC0180、CC0254、CC0325 反复考"无过错责任 / 过错推定 / 举证责任倒置"，而且 `R_CC0244_2026_东城_期末_18_05` 明确警告："说销售者适用无过错侵权责任都是错误"。这是高频致命点，V6.9 只在 CC0244、CC0180 各自题内提，没有一张"三档归责对照卡"。

**(6) 仍是"未重测 + 带病证据"状态。**
DOCX QA 自认只过了"结构 QA + 首屏缩略图"，没有全页视觉 QA，没有 PDF 导出，没有对 V6.9 的新裸题盲测。证据侧 CC0223 的题面仍带答案污染、设问仍被讲解语污染（见 BATCH_03 原文），CC0150 的细则原子里混进了整段"周边外交/人类命运共同体"国际政治答案。带病证据没清，框架就不能宣称满分闭环。

**结论**：V6.9 不是烂稿，是"对的零件、错的装配顺序"。它该被当作零件库，不该被当作 V7。

## 3. BATCH_01_FINDINGS（高频实体判断：合同、消费、劳动、一般责任，10 题）

题目：CC0002 / CC0025 / CC0054 / CC0063 / CC0143 / CC0150 / CC0181 / CC0200 / CC0373 / RECOVER_2026_朝阳_期末_18_1。

**batch_mechanisms（真正反复出现的得分机制）**
- 合同链采点：要约→承诺→合同成立→（成立≠生效）→有效→履行→违约。`R_CC0054_2024_石景山_一模_17_M17_01/02/03`、`R_CC0143_2025_朝阳_一模_19_03`（"合同成立不等于生效"独立 1 分）、`R_RECOVER_2026_朝阳_期末_18_1_01/02`。
- 劳动关系=三从属性，且每条必须结合材料：`R_CC0025_2024_朝阳_二模_17_09`明确"仅写三个从属性、没结合材料只给 1 分"。
- 权利有边界、不得滥用：`R_CC0063_2024_西城_二模_16_01`（首单 vs 加购，"不得超过正当界限"）。
- 多过错分摊+公平原则：`R_CC0200_2025_西城_二模_18_01`（平台/监护人/未成年人三方过错，民法典 157 条）。
- 欺诈→可撤销→消法三倍赔偿：`R_CC0143_2025_朝阳_一模_19_04/05/08`。
- 理由分+意义分分别计：CC0002 法理 5 分+意义 3 分；CC0025 理由 4 分+意义 5 分。
- 多诉讼请求/多诉求必须分开：`R_CC0143_2025_朝阳_一模_19_16/23`明确"没有区分几个诉讼请求"是失分主因；CC0373 两诉求分别判（`P01-P07`）。
- 术语精确硬扣分：`R_CC0143_..._19_21/22`（"消费者权益保护法"写成"保护法"不给分、"欺诈"写成"诈骗"不严谨）；`R_CC0150_2025_朝阳_二模_20_05`（"隐私权"写错字不给分）。

**student_start_moves（看到这类题先做什么）**
1. 数问号、数诉求、数案例、数主体——先确定要答几段。
2. 认设问尾词成品：问"理由/依据"→走责任链；问"意义"→走价值四层；问"判决结果/能否支持"→先表态。
3. 认法律关系：是合同（违约链）、是侵权（侵权链），还是劳动关系（三从属性）。
4. 把材料关键句逐条圈出，准备"译全、用原词"。

**material_to_legal_translation（材料信号→法律语言→得分句）**
- 报价/比选报价、内容具体确定 → 要约 → "甲公司报价内容具体确定，属于要约。"（`R_CC0054_..._M17_01`）
- 中标通知送达/出票/付款 → 承诺、合同成立 → "中标通知书送达，双方合同成立。"（`R_CC0054_..._M17_03`）
- 平台派单+奖惩+签到 → 人格从属性 → "依据制度对其奖惩，具有较强人格从属性。"（`R_CC0025_..._17_02`）
- 擅自搭售/不能知悉/无法拒绝 → 欺诈、违背真实意思 → 可撤销 + 三倍赔偿（`R_CC0143_..._19_08`，"3 个条件必须写全"）。
- 摄像头拍到"日常出行规律及其他动态信息" → 隐私权侵害（`R_CC0150_..._20_07`，必须用材料原词）。
- 离职次日入职同岗+技术属"不为公众所知悉" → 竞业限制义务、商业秘密（`R_CC0181_..._M17_04`）。

**full_score_sentence_rules（句子公式，不是口号）**
- 合同判断句：`经XX（要约）与XX（承诺），双方买卖合同成立；主体适格、意思表示真实、形式内容合法，合同有效。`
- 违约落点句：`XX未按约/未全面诚信履行义务，构成违约，应承担赔偿损失等违约责任。`
- 劳动关系句：`从人格/经济/组织从属性看，材料中XX……，故双方存在事实劳动关系。`（三句各带材料）
- 理由+意义切开写：理由段只写"事实+规则"，意义段才写价值，禁混。
- 多诉求：一个诉求一段，段首先标"关于第X项请求"。

**bad_framework_nodes_to_delete_or_demote**
- 删"九个战场"作为前台主轴——合同/消费/劳动是教材分编名。降级为后台检索标签。
- CC0150 的细则原子 `R_CC0150_..._20_12~24` 整段是"周边外交/人类命运共同体"国际政治答案，属跨模块污染，必须从法律证据里剔除、回源。
- "写前判断：判断法律关系/行为性质/责任承担或程序路径(回源细化)"这串通用占位语在 BATCH_01 几乎每个原子都出现，对学生零价值，删。

## 4. BATCH_02_FINDINGS（程序救济与表格补全：路径、证据、请求、表格格子，8 题）

题目：CC0125 / CC0244 / RECOVER_2025_海淀_二模_18 / RECOVER_2026_通州_一模_20 / RECOVER_2024_东城_二模_19_1 / RECOVER_2024_东城_二模_19_2 / RECOVER_2026_延庆_一模_18_1 / RECOVER_2026_门头沟_一模_18_1。

**batch_mechanisms**
- 这一批藏着全套框架引擎。`R_CC0244_2026_东城_期末_18_08` 直接给出老师的生成器：`违约逻辑：合同成立+合同有效+存在违约行为→违约责任；侵权逻辑：权利受保护+侵权要件→侵权责任；不正当竞争逻辑：竞争关系+不正当竞争行为→构成不正当竞争`。
- 归责三档硬区分。`R_CC0244_..._18_06`：无过错责任典型=生产者产品责任/环境污染/饲养动物；过错推定典型=个人信息处理者/动物园动物/林木折断/高空坠物；举证责任倒置=过错推定倒过错、污染环境倒因果。`R_CC0244_..._18_05` 警告："说销售者适用无过错侵权责任、生产者适用一般过错责任，都是错误"。**这条直接否定了 V6.2 那句"本题通篇不能漏无过错责任"的绝对化硬卡。**
- 表格逐格采点：`RECOVER_2025_海淀_二模_18` 九个原子=九格各 1 分；旁听收获按层数给分（三层 4 分/两层 3 分/一层 2 分，`R_..._18_09`）。
- 程序题三件套：路径+证据+请求。`R_CC0245_2026_东城_期末_18_2_01` 给出三角度（维权途径 2 分/依法收集证据并类型化 2 分/明确权利义务诉求合理 2 分），并警告"材料已协商未果，不能再写协商""不能直接上诉/行政诉讼""不能罗列证据、要归纳成证据类型"。
- 起诉状四件套：诉讼请求+事实+理由（+当事人）。`RECOVER_2024_东城_二模_19_1_R01/02/03`。
- 预防/建议题独立题型：`RECOVER_2024_东城_二模_19_2_R01/02` —— 问"为避免类似问题"必须答"合同内容应清晰明确"，答"协商一致"只 1 分，答"诚信履行/全面履行"不给分。这是一类极易答偏的题。
- 公益诉讼三视角：主体适格+多元解纷方式+法律效力（司法确认/强制执行力），`RECOVER_2026_门头沟_一模_18_1_R01/02/03`。
- 调解题视角：调解结果+调解理由（诚信/合同解除/证据意识），`R_CC0125_2025_延庆_一模_19_M17_01~04`。

**student_start_moves**
1. 先判这题问"实体"还是"程序"——问"怎么办/维权/证据/请求/起诉状/调解"就是程序题，不要用实体责任模板答。
2. 程序题先排序：证据→请求→路径→效力。
3. 看材料已经走到哪一步（已协商未果？已起诉？），排除已用尽的路径。
4. 表格题先照抄真实列名和示例行，再判每格功能。
5. 预防题先认定"问题根源在合同订立不清还是履行态度"，前者答订立内容。

**material_to_legal_translation**
- "协商未果" → 该路径已用尽 → 写调解/仲裁/诉讼，不再写协商（`R_CC0245_..._18_2_02`）。
- "已丢弃商品、无法举证" → 缺乏证据意识 → 调解理由（`R_CC0125_..._M17_04`）。
- 模拟法庭"开庭准备、确认诉讼权利义务" → 回避制度（`R_RECOVER_2025_海淀_二模_18_01`）。
- "和解协议经法院审查、出具民事调解书" → 司法确认、强制执行力（`RECOVER_2026_门头沟_一模_18_1_R03`）。
- "未约定新增仓储费" → 合同内容不清 → 预防措施写"合同内容应清晰明确"（`RECOVER_2024_东城_二模_19_2_R01`）。

**full_score_sentence_rules**
- 三大逻辑链作为骨架句（见上）。
- 程序题：`当事人应围绕XX（合同/付款/损害/因果）收集书证、物证、电子证据等证据；提出XX诉讼请求；通过调解/仲裁/诉讼路径维权。`
- 表格题：一格一句一功能，材料事实格写"事实+因果/非因果判断"。
- 归责档位句：`产品缺陷致害、向生产者主张产品责任的，生产者承担无过错责任；向销售者主张的，销售者承担一般过错侵权责任。`（两条并列，不可互换）

**bad_framework_nodes_to_delete_or_demote**
- 删 V6.2/早期那句"本题通篇不能漏无过错责任"的绝对硬卡——证据 `R_CC0244_..._18_05` 证明它会诱导学生把无过错错挂到销售者。改成"三档归责对照卡"。
- "表格补全战场"作为独立战场降级——表格只是"成品形态"，机制仍是实体/程序判断，应并入"成品=表格"分支。
- CC0244 题面/细则需回源：`R_CC0244_..._18_01` 把整段材料塞进"原子"，原子边界不清。

## 5. BATCH_03_FINDINGS（创新竞争、AI 边界、价值收束，9 题）

题目：CC0045 / CC0103 / CC0206 / CC0223 / CC0229 / CC0238 / RECOVER_2025_丰台_二模_19_2 / RECOVER_2026_房山_一模_17_1 / RECOVER_2026_西城_二模_18_1。

**batch_mechanisms**
- 意义/价值题四层结构：当事人权利层→行业/创新秩序层→司法/审判层→法治社会层。`R_CC0103_2025_丰台_一模_19_01` 把四层和分值写死：企业层（维护当事人合法权益+严惩侵权）2 分、创新层（规范竞争秩序/公平市场环境/鼓励创新）2 分、司法层（同类案件范例/提高效率/增强公信力）2 分、法治层（法治信仰+依法办事）1 点 1 分。
- 价值句必须有铺垫、绑案件：`R_CC0103_..._19_01` 明确"维护社会公平正义不得分""没有铺垫直接表达法治中国建设不得分"。
- 题级禁用词/必写词：CC0103 禁"优化营商环境""维护双方/公民/被告合法权益"；CC0206 必写"营商环境/法治化营商环境"（`R_CC0206_2025_西城_期末_19_01`）。同一个词在两题判分相反。
- 不正当竞争四种行为型：混淆、虚假/引人误解宣传、商业诋毁、侵犯商业秘密。`R_CC0206_..._19_01`（混淆+虚假宣传）、`R_CC0238_2026_东城_二模_19_M17_01`（商业诋毁）、`R_CC0131_2025_房山_一模_19_05`（窃取技术秘密）。
- 多案例/多主体分别落不同机制：`CC0229` 三案例分别=调解/惩罚性赔偿/规制恶意诉讼（`R_CC0229_..._18_03/05/07`）；`CC0238` 评析题分别评乙公司（商业诋毁+违约）和张某（违反劳动纪律）。
- AI 题三要件：①AI 不是民事主体→其承诺无意思表示、无法律效力；②看企业是否尽提示/注意义务、有无过错；③看有无实际损害+因果关系。`RECOVER_2026_房山_一模_17_1_R01/02/03`、`RECOVER_2026_西城_二模_18_1_R01/02/03`。
- 平衡型理解题：`RECOVER_2025_丰台_二模_19_2_R01` —— "划红线又留空间"答法律在技术创新与社会秩序之间的平衡作用。

**student_start_moves**
1. 设问问"意义/价值/典型案例"→走价值四层，每层先写本案处理再推价值。
2. 设问问"评析/谈认识"→逐主体、逐案例分别表态。
3. 出现"创新/商标/商业秘密/数据/混淆" → 先认保护对象、再抓侵害行为类型。
4. 出现 AI/算法/生成式 → 先卡死"AI 不是民事主体"，再走侵权三要件。
5. 写价值句前先问："这句价值有没有本案铺垫？没有就不写。"

**material_to_legal_translation**
- "近 40 名技术人员离职、用原单位技术信息申请专利" → 侵害技术秘密、不正当竞争（`R_CC0103_..._19_01`）。
- "2 倍惩罚性赔偿、判赔 6.4 亿" → 严惩侵权、提高侵权成本（`R_CC0103`、`R_CC0229_..._18_05`）。
- "抢先注册他人有一定影响标识+引人误解宣传" → 混淆+虚假宣传=不正当竞争（`R_CC0206_..._19_01`）。
- "AI 生成'内容有误赔偿 10 万'承诺" → AI 无民事主体资格、该承诺无法律效力（`RECOVER_2026_房山_一模_17_1_R02`）。
- "捏造事实暗示对手质量低下" → 商业诋毁（`R_CC0238_..._19_M17_01`）。

**full_score_sentence_rules**
- 意义题四层公式：`本案中法院通过XX处理（铺垫），保护了XX当事人合法权益／严惩了XX侵权；有利于规范XX竞争秩序、营造公平市场环境、鼓励创新；为同类案件提供范例、提高审判效率、增强司法公信力；引导公民依法办事、增强法治信仰。`
- AI 题公式：`AI 不具有民事主体资格，不能独立作出有效意思表示，其XX承诺不产生法律效力；XX公司已尽提示/注意义务、无过错；XX未能证明实际损害及因果关系，侵权不成立。`
- 价值句铁律：每句价值前必带"本案……"铺垫，禁裸写"公平正义/法治中国"。

**bad_framework_nodes_to_delete_or_demote**
- CC0223 证据带病：BATCH_03 原文里 CC0223 的"材料/题干"整段是答案"①有利于明确权利边界②有利于化解社会矛盾③有利于维护公平正义"，设问被污染成"运用法治知识，设问指向是阐释定分止争的价值……"。必须回源清洗，V7 不得直接拿 CC0223 当核心闭合题。
- "价值收束战场"作为独立战场降级——价值是所有题的收口动作，不是一类题。应改为贯穿层"价值四层卡"。

## 6. BATCH_04_FINDINGS（38 道非核心与开放容器：保分、边界、reference_only）

38 题，其中 4 道 reference_only（CC0040、CC0162、CC0311、CC0353）。当前类型分布：低频规则题、综合边界题、转出本书题、普通答案练习题（reference_only）、设问/题面待回源题。

**batch_mechanisms**
- 这 38 题不是"次等核心题"，是四个**用途不同的容器**：
  1. 低频规则容器：CC0011 绿色原则、CC0254 动物侵权+法德结合、CC0332 校园欺凌惩教、CC0340 物权编绿色发展、RECOVER_2024_东城_一模_19 诉讼时效。考点低频，能保三句分，但绝不能抬成高频母题。
  2. 综合边界容器：CC0276 涉外法治、CC0277 数字员工（法律+哲学）、CC0380 开源智能体（必修三+选必二）、RECOVER_2026_西城_二模_18_2/18_3。`R_CC0277_..._18_07~11` 显示这类题后半段是哲学（联系/发展/扬弃），法律只占前半——必须"法律层最低句 + 其他模块另开段"。
  3. 转出容器：CC0364 后半是《逻辑与思维》充分/必要条件假言推理（`R_CC0364_..._19_1_05/06`），法律框架不能硬答。
  4. reference_only 练笔容器：CC0040/CC0162/CC0311/CC0353，证据等级 `reference_only/teacher_reference_answer`，只能练表达，不支撑 formal 满分闭环。
- 即使非核心，机制仍来自主干三条链——CC0317/CC0318 格式条款（公平原则、限制主要权利）、CC0319 不正当竞争、CC0325 举证责任三型（行政诉讼/谁主张谁举证/过错推定倒置）、CC0277 商业诋毁+商业秘密+著作权。
- 表格题集中在此批：CC0077、CC0084 都是"完成表格"，且阅卷报告给了大量学生病象（主体不清、抄材料、不用法律词、民法原则乱写）。
- 无过错责任在此批仍是硬点：`R_CC0180_2025_海淀_期末_20_M17_06` "通篇没有无过错侵权责任不能得满分；把烈性犬写成过错推定，法理部分不给分"。

**student_start_moves**
1. 先判这题是不是纯《法律与生活》核心题；不是就进容器。
2. 综合题：先写法律层最低三句，其他模块另起段，不用私法模板硬塞。
3. 转出题：判断主任务是不是逻辑/哲学，是就转出，只在有独立法律小问时写法律。
4. 低频题：按它本来的低频考点写三句保底，不强行升母题。
5. reference_only 题：当表达练习，不当满分细则背。

**material_to_legal_translation**
- "数字员工生成虚假负面文章" → 商业诋毁（反不正当竞争法）+ 名誉权（`R_CC0277_..._18_01`）。
- "核心代码上传公共 AI 平台" → 商业秘密泄露风险（`R_CC0277_..._18_02`）。
- "格式条款限制合同期满后自由订立合同" → 违反公平原则、限制主要权利、条款无效（`R_CC0317_..._18_01`）。
- "建筑物窗户坠落致车损" → 过错推定、举证责任倒置（`R_CC0325_..._18_02`）。
- "赡养费长期未付、被时效抗辩" → 赡养费请求不适用诉讼时效（`RECOVER_2024_东城_一模_19_R03`）。

**full_score_sentence_rules**
- 容器题统一公式：`【法律层最低三句】+【若涉其他模块：另开段写】`。
- 低频规则题：`点明低频规则名 + 该规则的功能/边界 + 一句价值`，三句即保底。
- 综合边界题：`先写选必二法律层（权利风险+救济方式）`，再写"若原题要求必修三/哲学，另开模块段"。
- 格式条款题公式：`该条款由一方预先拟定、重复使用、未协商，属格式条款；提供方应遵循公平原则确定双方权利义务；该条款限制了XX主要权利，应认定无效。`

**bad_framework_nodes_to_delete_or_demote**
- 删"38 道非核心保分索引"这种平铺附录式表达——它把四类用途完全不同的题压成一张索引。改为四个**有名字、有纪律的容器**（低频/边界/转出/练笔）。
- reference_only 四题（CC0040/CC0162/CC0311/CC0353）不得进入任何"满分闭环"统计，教师后台必须显式锁定。
- "设问或题面需要教师再核对的题"这个标签下挂了 24 题——数量过大（占 61 道 formal 的近四成），说明证据底座 canonical 回源严重欠账，必须在 V7 前清。

## 7. V7_FRAMEWORK_PROPOSAL（V7 候选框架）

V7 不再以"九个战场"开局。V7 的前台主轴换成**学生的笔要产出什么**，背后挂一台**生成器**。

### 7.0 总模型（第一页，继承旧框架 A→B）

```
生活冲突 A  ——→  法律判断 B
中间链：主体 → 行为 → 争点 → 规则 → 材料事实 → 落点
```

### 7.1 主干高频层（前台，闭合训练区）

**层一｜设问成品分流器（前台第一主轴，取代九个战场）**
看设问尾词，决定笔要产出哪种"成品"。六种成品 + 一个边界判断：

| 成品 | 设问尾词信号 | 第一笔 | 证据锚点 |
|---|---|---|---|
| 判断型 | 能否支持、是否有效、是否构成、判决结果 | XX请求应/不应支持，因为…… | CC0200、CC0373、CC0251 |
| 理由依据型 | 理由、法律依据、为什么这样判 | 依据XX规则，本案中XX，所以…… | CC0054、CC0143、RECOVER_2026_朝阳_期末_18_1 |
| 意义价值型 | 意义、价值、作用、典型案例 | 本案通过XX处理，保护了……规范了…… | CC0103、CC0025、CC0131 |
| 评析型 | 评析、谈认识、辨析 | 逐主体/逐案例分别表态 | CC0238、CC0229、RECOVER_2026_房山_一模_17_1 |
| 维权建议型 | 怎么办、维权、需做哪些工作、起诉状 | 围绕XX收集证据，提出XX请求，走XX路径 | CC0245、CC0125、RECOVER_2024_东城_二模_19_1 |
| 表格补全型 | 完成下表、补全笔记、划线处 | 照抄真实列名→逐格判功能→一格一句 | RECOVER_2025_海淀_二模_18、RECOVER_2026_通州_一模_20 |
| （边界判断） | 含 AI/治理/逻辑/哲学 | 先判是不是纯法律核心题，不是就进容器 | CC0364、CC0277、CC0380 |

**层二｜三大责任逻辑链（生成器引擎，取自 R_CC0244_2026_东城_期末_18_08）**

```
违约链：  合同成立(要约+承诺) + 合同有效(主体适格+意思表示真实+形式内容合法) + 违约行为 → 违约责任
侵权链：  权利受法律保护 + 侵权要件(行为+损害+因果+过错/无过错) → 侵权责任
不正当竞争链：竞争关系 + 不正当竞争行为(混淆/虚假宣传/商业诋毁/侵犯商业秘密) → 不正当竞争责任
```
证据：CC0054、CC0244、RECOVER_2026_朝阳_期末_18_1（违约链）；CC0150、CC0200、CC0251、CC0325（侵权链）；CC0206、CC0238、CC0131、CC0319（不正当竞争链）。

**层三｜归责三档对照卡（高频致命点，单独成卡）**

| 档位 | 典型情形 | 举证 | 证据 |
|---|---|---|---|
| 过错责任 | 一般侵权、销售者产品责任 | 谁主张谁举证 | R_CC0244_..._18_05、R_CC0325_..._18_03 |
| 过错推定 | 个人信息处理者、动物园动物、林木折断、高空坠物、建筑物坠落 | 倒置（倒过错） | R_CC0244_..._18_06、R_CC0325_..._18_02 |
| 无过错责任 | 生产者产品责任、环境污染、饲养动物 | —— | R_CC0244_..._18_06、R_CC0180_..._20_M17_01/04 |

硬纪律：销售者≠无过错、生产者≠一般过错（R_CC0244_..._18_05）；通篇不出现"无过错责任"四字在产品/动物题难满分（R_CC0180_..._20_M17_06）。

**层四｜材料翻译三步（继承旧框架"情境触发"+ 升级为强制动作）**
材料信号 → 法律术语 → 得分句。铁律：①逐条译全（CC0143"3 个条件写全才得分"）；②用材料原词（CC0150"日常出行规律"原词才给分）；③法律名称精确（CC0143"消费者权益保护法"不能简写）。

**层五｜落点与价值四层（收口）**
落点贴成品：判断型落责任/效力，维权型落证据请求路径，意义型落价值四层。价值四层=当事人权利→行业秩序→司法审判→法治社会，每层必带本案铺垫（R_CC0103_..._19_01）。

**层六｜一二三四五总图（恢复 V6.9 删掉的全书地图）**
一件事=把生活冲突判成法律结果；两条线=事实规则线+价值收束线；三问=判什么/凭什么/怎么落分；四步=圈主体/抓行为/定争点/落结果；五种成品世界（即层一六成品归并）。

**层七｜27 高频题逐题演练**
保留 V6.9 逐题九栏，但每题题头不再标"战场"，改标"成品类型 + 用到的逻辑链 + 归责档"。每题每个九栏节点必须挂 question_id + rubric_atom_id。

### 7.2 开放容器层（前台，保分/边界区，四个有纪律的容器）

| 容器 | 收什么 | 纪律 | 证据 |
|---|---|---|---|
| 低频规则容器 | 绿色原则、诉讼时效、校园欺凌、动物侵权法德结合 | 保三句分，不升高频母题 | CC0011、CC0254、CC0332、CC0340、RECOVER_2024_东城_一模_19 |

| reference_only 练笔容器 | 普通参考答案题 | 只练表达，不作 formal 满分闭环，教师后台锁定 | CC0040、CC0162、CC0311、CC0353 |

### 7.3 教师后台（不进学生前台）
证据等级、rubric_atom_id、待回源清单、cross-module 污染清单全部放后台附录。

## 9. QUESTION_BY_QUESTION_STRESS_TEST_PLAN（逐题压测方案）
