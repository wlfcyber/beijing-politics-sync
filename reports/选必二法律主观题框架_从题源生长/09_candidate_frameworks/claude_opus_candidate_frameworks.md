# 1. 候选框架 A：**七锚直码型 / Seven-Anchor Direct-Code**

## A.1 框架总逻辑

完全以 7 条 codebook 各自独立成节点，不做再聚类、不做上层路由。学生面对一题，依次按节点的"触发信号"过一遍清单，命中哪条用哪条；都不命中就显式标"框架未覆盖"。该方案的口号是**"宁缺勿编"**。

## A.2 节点一览

### 节点 A-N1 · 表格题独立赋分单元

- **功能**：处理 ask_text 含"完成下表/补充裁判理由"且评分以"格"为单位的题型。
- **code_id**：`CODE_COWORK_001`
- **支持的 question_id**：CC0077, CC0084, CC0157, CC0180, CC0325, RECOVER_2026_通州_一模_20
- **支持的 rubric_atom_id**：R_CC0077_…_02 / _07，R_CC0084_…_07，R_CC0180_…_M17_06
- **关联 material_atom_id**：M_CC0077_…_01/02/08/09/15，M_CC0084_…_01/02/13，M_CC0157_…_01–04，M_CC0180_…_01–04，M_CC0325_…_02/03/05/06/08/09/10/12，M_RECOVER_2026_通州_一模_20_01–04
- **材料触发信号**：题干印有"示例"格 + 表格结构（机制 / 以案说法 / 典型意义 / 裁判理由列）
- **最小必要判断**：每格对应哪一具体小问？写错格类（把"以案说法"答成"典型意义"）即整格 0 分。
- **满分句生成**：按格独立 ——"机制格 = 调解 / 仲裁 / 诉讼 / 和解 之一"+"以案说法 = 主体+行为+定性+依据"+"典型意义 = 行为对象+功能/价值"
- **易错偏题风险**：CC0084_11 显式扣"双方权利"式套话；CC0180_06 显式扣"通篇无无过错侵权责任不能得满分"——即跨格抄模板+主体不清。
- **证据强度**：**强**（6 题 ID 锚定，rubric 显式扣分句已抓到）

### 节点 A-N2 · 意义三角度分点

- **功能**：处理 ask_text 含"现实意义/社会价值/典型意义"且≥3 分的题。
- **code_id**：`CODE_COWORK_002`
- **支持的 question_id**：CC0002, CC0251
- **支持的 rubric_atom_id**：R_CC0002_…_M17_04 / _05 / _06
- **关联 material_atom_id**：M_CC0002_…_01–04，M_CC0251_…_01–07
- **材料触发信号**：法院判决原句（"判令…赔偿/驳回/支持"），及案件主体（当事人/经营者/消费者）
- **最小必要判断**：三层主语必须不同——个人 / 司法或行业秩序 / 社会公平或核心价值观；不能三句都是"维护公平正义"。
- **满分句生成**："该判决/案例有利于……（个人合法权益）；有利于……（司法/市场/行业秩序）；有利于……（公平正义/社会主义核心价值观/法治建设）。"
- **易错偏题风险**：CC0251_04 显式扣"必修三话术堆叠（法治政府/法治国家/公序良俗一锅炖）"——必修三化高风险点。
- **证据强度**：**中**（仅 2 题 ID 锚定；但 rubric 明文扣分细则强）

### 节点 A-N3 · 法院判决锚句"以事实为依据，以法律为准绳"

- **功能**：处理设问对象是法院/仲裁/法官且分值≥5 分的题，强制踩"锚句 1 分"。
- **code_id**：`CODE_COWORK_003`
- **支持的 question_id**：CC0002, CC0119, CC0251
- **支持的 rubric_atom_id**：R_CC0002_…_M17_01，R_CC0119_…_M17_01，R_CC0251_…_01
- **关联 material_atom_id**：M_CC0119_…_04
- **材料触发信号**：材料出现"法院判令/裁定/支持/驳回"等结论性表述
- **最小必要判断**：是否司法/仲裁机关的裁决结论？若是，首句必须给出"以事实为依据、以法律为准绳"或等价"根据《民法典》相关规定"。
- **满分句生成**："人民法院/仲裁委以事实为根据、以法律为准绳作出判决，依据《××法》规定……"
- **易错偏题风险**：仅停留锚句、不接法律规则定性，会被卡 1 分上限（Claude 观察）。
- **证据强度**：**强**（3 题 ID 锚定 + 显式细则）

### 节点 A-N4 · 案例分析四步法（定性—确权—解释—责任）

- **功能**：处理"分析上述案例/分析判决结果并说明理由"且核心是侵权/合同纠纷的题。
- **code_id**：`CODE_COWORK_004` + `CODE_COWORK_006`（**两码同源合并为一节点**）
- **支持的 question_id**：CC0150（侵权侧）, CC0244, CC0317（合同侧）, RECOVER_2026_朝阳_期末_18_1
- **支持的 rubric_atom_id**：R_CC0150_…_05–08，R_CC0244_…_08，R_RECOVER_2026_朝阳_期末_18_1_01
- **关联 material_atom_id**：M_CC0150_…_02，M_CC0244_…_02，M_RECOVER_2026_朝阳_期末_18_1_01
- **材料触发信号**：拍摄角度、商品瑕疵、商品被损、用户协议条款——具体行为细节需要嵌入要件位
- **最小必要判断**：选侵权 / 违约 / 不正当竞争之一；CC0244_08 口令——"合同成立+有效+违约→违约责任"或"权利受保护+侵权要件齐备→侵权责任"或"竞争关系+不正当竞争行为→构成不正当竞争"。
- **满分句生成**："××侵犯了××权（定性），根据《民法典》××条（确权），××的行为构成××（解释要件并嵌入材料具体事实），应承担××责任（责任）。"
- **易错偏题风险**：CC0244_08 显式提醒"债权与物权不要混同，本题不涉及物权变动"——过度细化诉权/过错推定属法考化。
- **证据强度**：**强**（侵权/合同两路共 4 题 ID 锚定，且 CC0244_08 有口令式细则）

### 节点 A-N5 · 评析题"表态先行 + 双依据"

- **功能**：处理"是否有效/能否得到法院支持/是否构成××"型题。
- **code_id**：`CODE_COWORK_005`
- **支持的 question_id**：CC0305_2026_海淀_一模_18_3, CC0373_2026_顺义_一模_18
- **支持的 rubric_atom_id**：R_CC0305_…_P01，R_CC0373_…_P01 / _P04
- **关联 material_atom_id**：M_CC0373_…_P01
- **材料触发信号**：材料出现两方诉求或两方行为对立，题目要求选边

- **满分句生成**："（表态）应予支持/不予支持。（法律依据）根据《××》规定……（事实依据）本案中……（结论复述）所以……"
- **易错偏题风险**：CC0373 多主体题易过度延伸"夫妻共同财产侵权法理"——超出教材属法考化。
- **证据强度**：**中**（仅 2 题 ID 锚定；codebook 文本提及 CC0317、RECOVER_2026_房山_一模_17_1 但 supporting_q 未挂，按字段口径不计入）

### 节点 A-N6 · 维权题"程序—准备—实体"三层

- **功能**：处理"如何维权 / 应对建议 / 怎么做"且≥2 分的题。
- **code_id**：`CODE_COWORK_007`
- **支持的 question_id**：CC0245_2026_东城_期末_18_2
- **支持的 rubric_atom_id**：R_CC0245_…_01 / _02 / _04
- **关联 material_atom_id**：M_CC0245_…_01
- **材料触发信号**：材料是否出现"协商未果 / 双方拟通过诉讼 / 已起诉 / 收到律师函"等程序节点

- **证据强度**：**弱—中**（仅 1 题 ID 锚定 CC0245_2026_东城_期末_18_2；CC0289 仅在 codebook 风险栏被引用，不在 supporting_q）

## A.3 框架 A 性能盘点

- **能顺利解决（满分闭环）**：CC0002 / CC0077 / CC0084 / CC0119 / CC0150 / CC0157 / CC0180 / CC0244 / CC0245 / CC0251 / CC0305 / CC0317 / CC0325 / CC0373 / RECOVER_2026_朝阳_期末_18_1 / RECOVER_2026_通州_一模_20，共 **16 题**。

- **证据强节点**：A-N1（6 题）、A-N3（3 题+显式锚句）、A-N4（4 题+口令细则）。

# 2. 候选框架 B：**设问识别→码本路由→句式模板（三层分流）**

## B.1 框架总逻辑

承认 7 条 codebook 是"奖励机制库"，但要让学生在考场上能用，必须先有一层"设问识别"把题目分流到正确的码上。三层结构：

- **Layer 1 · 设问触发**：基于 `ask_atoms` 的 `ask_function_plain` 字段（综合/说明/论证/意义/评析/原因/建议/开放）+ ask_text 关键词进行识别。这一层**只做识别，不发放分数判断**——分数判断必须来自 Layer 2 的 codebook。
- **Layer 2 · 码本节点**：与框架 A 的 6 个节点完全相同（CODE_001 / 002 / 003 / 004+006 / 005 / 007），不增加任何节点。
- **Layer 3 · 句式生成 + 易错检测**：照 codebook 的 `full_score_sentence_pattern` 与 `risk` 字段执行。

> ⚠️ **关键边界**：Layer 1 的 ask_function 来自题层结构性数据，不是 cross-validation 观察，因此可作"识别工具"使用；**但 Layer 1 不允许独立赋分**，分数判断只能从 Layer 2 的 codebook 出发。

## B.2 节点结构

### Layer 1（识别层，6 个识别口）

| 识别口 | ask_function 触发 | 关键词触发 | 拟分流到 |
|---|---|---|---|
| B-L1-T1 表格识别 | 综合/说明 | "完成下表 / 补充裁判理由" + 表格结构 | A-N1 (CODE_001) |
| B-L1-T2 意义识别 | 意义 / 论证 | "现实意义 / 社会价值 / 典型意义" 且分值≥3 | A-N2 (CODE_002) |
| B-L1-T3 司法锚句识别 | 综合 / 论证 / 意义 | 设问对象是"法院判决 / 裁判理由 / 法官调解"且分值≥5 | A-N3 (CODE_003) |
| B-L1-T4 案例四步识别 | 综合 / 评析 / 说明 | "分析上述案例 / 分析判决结果并说明理由" | A-N4 (CODE_004+006) |
| B-L1-T5 评析表态识别 | 评析 | "是否有效 / 能否得到法院支持 / 是否构成" | A-N5 (CODE_005) |
| B-L1-T6 维权识别 | 建议 / 综合 / 开放 | "如何维权 / 提出措施 / 应对建议 / 怎么做" | A-N6 (CODE_007) |

### Layer 2（码本层）：直接复用框架 A 的 6 节点，每节点字段、ID、风险与框架 A 完全一致，不重抄。

### Layer 3（句式层）：每节点的 `full_score_sentence_pattern` 即为模板；学生按模板插入材料原句。

## B.3 三类"开放容器"（不是核心节点，仅作压测用 hook）

> 这三类容器**不发放评分判断**，只用于压测时记录"码本失效在哪"。它们对应 23 条 pending 观察中的若干，但不直接采用其措辞。

- **Container B-C1 · IP/反不正当竞争触发集群**：CC0103 / CC0131 / CC0189 / CC0229 / CC0283 / CC0311 等含"知识产权 / 惩罚性赔偿 / 不正当竞争"的题。对应 pending `CMP_COWORK_002 / 018 / 020`（STRONG_OBS_08 vs OBS_LAWLIFE_W_014_ip_innovation_cluster_keywords），**pending 待源核**，不能升格。
- **Container B-C2 · 起诉状/书写格式三段式**：CC0143 / CC0223 / CC0276 / CC0340 等可能涉及起诉状或维权文书的题。对应 pending `CMP_COWORK_013 / 015 / 016`，**pending 待源核**。
- **Container B-C3 · reference_only 题（4 题）**：CC0040 / CC0162 / CC0311 / CC0353。对应 pending `CMP_COWORK_004 / 014`。这 4 题**只能"形式上类比"框架节点，不能宣称满分闭环**。

## B.4 框架 B 性能盘点

- **能顺利解决**：与框架 A 完全相同的 16 题（核心覆盖来自码本，不来自识别层）。
- **能"半解决"（启动→无终点）**：Layer 1 能把 49 题识别归类，但 Layer 2 找不到对应码——学生进得了类型口，出不了满分句。这对压测有诊断价值：能定位"识别成功但码本失效"的题。
- **解决不好**：本质仍是 49 题（同 A）。reference_only 4 题永远只能做参考。
- **证据强节点**：与 A 相同（A-N1 / A-N3 / A-N4）。识别层本身无证据等级问题，因为不参与赋分。
- **证据弱节点**：与 A 相同。
- **法考化风险**：与 A 相同水平，识别层不引入新风险。
- **必修三化风险**：与 A 相同。识别层若过度依赖"现实意义"关键词，可能把综合题误推到 A-N2，从而触发必修三话术——这是 B 独有的次级风险。
- **考场可启动性**：**最高**——学生先识别设问类型，再走码本；这是唯一带"入口判断"的方案。

# 3. 候选框架 C：**码—题—证据三栏可视化（诊断型）**

## C.1 框架总逻辑

不是给学生用的，是给压测和后续 codebook 扩展用的诊断地图。结构：左栏 7 个 code_id，中栏 16 题已挂题，右栏 49 题未挂题（按 ask_function 二级分群），并对每个"未挂题"标注**当前最相近的 pending 观察**与**回源动作**。其口号是**"先把缺口画清楚，再决定下一步要不要长出第 8、第 9 条 codebook"**。

## C.2 主结构

### C.2.1 已挂码部分（与框架 A 节点完全相同，不复述）

# 5. 压测推荐

**推荐用于全题压测的候选框架：框架 B（设问识别→码本路由→句式模板）**。

理由如下：

1. **它是唯一能让 65 题全部进入压测流的方案**。框架 A 对 49 题直接"无抓手"，压测只能记录"未覆盖"；框架 C 不服务作答。框架 B 把 49 题先送进 Layer 1 的 6 个识别口，能产出"识别成功但 Layer 2 码失效"这种**最有诊断价值的失败模式**——这正是未来扩 codebook 的入料。
2. **它没有突破 7 条 codebook 的边界**。Layer 2 仍是 A 的 6 节点，识别层只用 ask_atoms 数据，不引入 cross-validation 观察以外的奖励机制。pending 观察被严格放进 B-C1 / B-C2 / B-C3 三个容器，**不升格**。
3. **它带"反必修三 / 反法考 / 反表格模板"三道风险刹车**，这些刹车来自 codebook 本身的 risk 字段，不是我添加的。
4. **它具备 16 题强证据闭环 + 49 题可观察分流**的双重压测价值：满分闭环可验证句式模板的稳定性；分流可验证识别口的精度与漏识率。
5. **reference_only 4 题被显式装进 B-C3 容器**，符合"reference_only 只能参考、不能闭环"的工程红线。

> 同时建议在压测启动时**显式纸面声明**：本框架仍是候选阶段；49 题的失败模式不算证伪 codebook，而是回源 / 扩本的输入。本响应不构成最终宝典 / 最终框架定稿。
