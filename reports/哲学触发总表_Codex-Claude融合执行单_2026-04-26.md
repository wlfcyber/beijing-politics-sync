# 必修四哲学触发总表：Codex-Claude 融合执行单

生成时间：2026-04-26

## 一、当前判定

ClaudeCode 本轮 Markdown 与 Git 正本 `artifacts/必修四哲学材料-知识触发总框架_持续更新版_v2.md` SHA256 完全一致，属于母版复制/转档，不属于独立正文重跑。

因此，不能把 ClaudeCode 输出当作第二份正文意见与 Codex 正本做内容合并。

## 二、融合原则

1. Codex Git 正本继续作为唯一正文底稿。
2. Claude/Cowork 的控制层可以吸收，包括 `SOURCE_LEDGER.csv`、`COVERAGE_MATRIX.csv`、`GOVERNOR_CHECKLIST.md`、角色报告、Word 渲染记录。
3. Claude 后续只允许输出差异与审计，不允许直接复制母版后宣称重跑完成。
4. 任何新增或改写都必须通过来源强度评级后再进入正本。

## 三、来源强度评级

| 等级 | 可入正文程度 | 判定标准 |
| --- | --- | --- |
| A | 可直接进入融合确认版 | 细则/评标/阅卷总结明写原理，并能对应材料点 |
| B | 可入正文，但必须标注边界 | 细则/评标只列角度，材料链由整理者解释 |
| C | 可入选择题触发区 | 选择题正确项明确触发，有可靠答案表 |
| D | 暂不进入确认版 | 普通参考答案、弱来源、无法证明该原理在答案中的内容 |

## 四、Claude 后续任务口径

请 Claude 只做以下三份输出：

1. `DELTA_AUDIT.md`
   - 只列出它认为应新增、删除、改写的条目。
   - 每条必须写：原条目位置、建议动作、理由、证据文件、题号。
   - 若没有差异，必须说明抽查范围，而不是复制母版。

2. `SOURCE_STRENGTH_AUDIT.csv`
   - 字段：条目ID、来源套卷、题号、原理/方法论、来源文件、原文摘录位置、证据等级A/B/C/D、风险说明、建议动作。
   - 不允许只写“已核验”，必须写能回到原文的位置。

3. `SUSPECT_ENTRIES.md`
   - 专门列“材料能联想到，但答案/细则没明写”的条目。
   - 这类条目默认不得进入融合确认版，除非后续人工确认。

## 五、融合验收门

只有同时满足下列条件，才允许把 Claude 审计结果并入正本：

1. 输出文件与母版不能字节级相同；若相同，必须判定为无效重跑。
2. 每条建议必须能回到可靠来源原文。
3. B/C/D 级不能伪装成 A 级。
4. 选择题正确项只能作为选择题触发证据。
5. `FINAL_ACCEPTANCE_REPORT.md` 不能只以 Word 生成、行数一致、diff 一致作为完成依据。

## 六、给 ClaudeCode 的纠偏提示

你不是要重新复制母版。你现在的任务是对 Codex Git 正本做独立审计，输出差异与证据评级。

严禁：

- 复制母版后报告重跑完成。
- 把普通参考答案当成细则。

- 把“材料能联想到某原理”写成“答案要求某原理”。

必须：

- 逐条回到本地缓存或原始试卷/细则。
- 对每条来源做 A/B/C/D 证据强度分级。
- 对存疑条目列入 `SUSPECT_ENTRIES.md`。
- 最终报告必须写清：抽查了多少条、A/B/C/D 各多少条、发现哪些疑似臆造链、哪些建议合入 Codex 正本。
