## V5.9 最大 12 个失败点

1. **首屏仍然先给动作卡，没先给判题门。**  
   学生看到“分关系、认产权、排维权、划风险、推价值”会觉得都对，却不知道先按设问分流，还是先按生活情境分流。失败类型：框架启动，不是证据问题。

2. **“一题只选一张主卡”是纪律，不是算法。**  
   V5.9 没告诉学生卡片冲突时按什么优先级裁决。比如 AI + 侵权 + 价值 + 治理同时出现，学生容易把卡叠满。失败类型：框架启动。

3. **27 个核心题像答案样章，不像训练路径。**  
   每题都有好答案，但学生从陌生题走到这个答案的中间桥不够：怎样圈材料、怎样判争点、怎样把材料翻译成法律句，仍靠悟性。失败类型：教学组织。

4. **“结论句 + 规则句 + 材料句 + 落点句”太总。**  
   它能防空话，但不能保证满分。不同题的“规则句”需要不同最小条件：合同要要约承诺，劳动要三从属性，消费者要合同成立 + 欺诈事实 + 赔偿请求，程序题要证据 + 请求 + 路径。失败类型：满分句生成。

5. **价值题还缺“价值从哪里长出来”的硬槽位。**  
   V5.9 反复提醒不要空写价值，但没有把价值拆成“本案法律处理 -> 保护对象 -> 规范对象 -> 社会效果”。学生仍会裸写公平正义、核心价值观。失败类型：满分句生成。

6. **38 题保分索引证据纪律清楚，但考场动作不够清楚。**  
   source-check、reference-only、boundary、low-frequency 的红线保护了证据，却没充分训练“只保分时到底写几句、先写哪句、何时停笔”。失败类型：教学组织，不是证据问题。

7. **source-check 题最危险：它把教师工作留给了学生。**  
   “正式课堂使用前教师必须回看原卷设问”是对的，但学生手里如果只有 V5.9，就会把最低三句当最终答案。失败类型：教学组织/题包口径，不是 65 题底座问题。

8. **多小问机制仍不闭合。**  
   V5.9 有“多小问先拆开”的硬规则，但 CC0244 盲测已暴露题面包 ask_text 漏第（2）问。学生能不能写全，取决于源卡是否带全小问。失败类型：框架启动 + 题包生成口径。

9. **表格题只有“一格一答”，缺格子功能判别表。**  
   学生不知道一个空到底要“机制、裁判理由、裁判结果、证据类型、意义、权利名称”中的哪一种，就会把整段法律话塞进去。失败类型：满分句生成。

10. **AI/数字治理边界仍过窄地靠提醒防错。**  
    V5.9 告诉学生不要写宏观治理，但没有做成三岔路：具体民事风险、综合治理边界、其他模块/逻辑题转出。失败类型：框架启动。

11. **8 题盲测不能证明稳定满分。**  
    盲测结论是 PASS_WITH_GUARDS，但样本只有 4 核心 + 4 非核心，且没有系统覆盖 24 个 source-check、全部表格题、全部 AI 边界题、全部价值题。失败类型：验证覆盖不足，不是证据问题。

12. **学生前台混入太多后台标签。**  
    27 core、38 guard、source-check、reference-only、boundary、transfer 对证据管理必要，但放在学生学习主线前台会让学生学标签而不是学答题动作。失败类型：教学组织。

## 明确不是证据问题的部分

- 不是“65 题底座不该保留”。V6 必须继续使用当前 65 题作为证据底座。
- 不是“38 道非核心要升核心”。相反，V6 必须更清楚地把 38 题放到低频、回源、参考、边界、转出区。
- 不是“reference_only 可以支撑核心”。reference_only 只能当练笔表达，不当核心节点证据。
- 不是“V5.9 没有红线”。V5.9 有红线，失败在红线之后没有足够学生化的启动算法。
- 不是“答案句不够长”。很多答案够长，问题是学生不知道如何从陌生材料生成这些句子。

## 6 个题型场景：学生会怎么写偏

### 场景 1：多小问责任 + 维权准备题

代表：`CC0244_2026_东城_期末_18`

学生偏法：只写第（1）问合同责任、侵权责任、赔偿责任；漏第（2）问证据准备、请求设计、维权路径。即使背了“排维权步骤”，如果题面包只露出第（1）问，学生不会自动补第（2）问。

V6 必修：每题先生成“小问清单”，再逐问选卡。首屏必须写：`先拆问号，再选主卡；每个问号至少一段。`

### 场景 2：表格/笔记补全题

代表：`CC0077_2025_东城_一模_19`、`2025 海淀二模 第18题`、`2026 通州一模 第20题`

学生偏法：看到表格题就写一段“法治维护权益、调解有温度、法院公正司法”，把三个格子的功能混成一个段落。阅卷人会按格扣分，因为每格要的可能是机制、理由、结果、证据或意义。

V6 必修：新增“格子功能判别表”：示例行看什么、空格问什么、答案限几字、能不能写价值。

### 场景 3：AI 风险/合规/综合边界题

代表：`CC0277_2026_房山_二模_18`、`CC0380_2026_顺义_二模_18_2`、`RECOVER_2026_西城_二模_18_2`

学生偏法：一看到 AI 就写技术伦理、数字中国、治理现代化；或者反过来把所有 AI 题都写成隐私权/著作权/商业秘密。前者跑出选必二，后者把综合边界题硬塞核心。

V6 必修：AI 题先三分流：`具体民事权利风险 -> 写权利/规则/措施`；`产业治理或国家治理 -> 只写法律层最低句并另开模块`；`逻辑推理题 -> 转出法律框架`。

### 场景 4：source-check 保分题

代表：`CC0019_2024_朝阳_一模_19`、`CC0077_2025_东城_一模_19`、`CC0137_2025_昌平_二模_20`

学生偏法：把“最低可写三句”当闭合满分答案，不看原设问到底问意义、原因、怎么办还是补表格。阅卷人会认为方向有一点，但任务没对上。

V6 必修：source-check 题不能给单一成稿语气，要改成“若问意义/若问原因/若问怎么办/若问补表格”的分叉模板，并标注“不能背成最终答案”。

### 场景 5：reference_only 练笔题

代表：`CC0040_2024_海淀_一模_19`、`CC0162_2025_海淀_一模_18`、`CC0311_2026_海淀_二模_18_2`、`CC0353_2026_西城_期末_17`

学生偏法：觉得这些题写法漂亮，就把虚拟数字人、格式合同、知识产权融资、不正当竞争全部当成核心模板背。之后遇到正式细则题，会拿普通参考答案句替代采分规则。

V6 必修：reference_only 独立成“练笔区”，每题拆成两栏：`可借表达` 和 `不可借结论/不可支撑核心`。

### 场景 6：低频规则/价值混合题

代表：`CC0011_2024_丰台_二模_17`、`CC0254_2026_丰台_二模_18`、`CC0340_2026_西城_一模_17`

学生偏法：看到绿色原则、法治德治、生态公共利益，就写环保意识、司法温度、核心价值观，漏掉具体规则：绿色原则如何进民法，动物侵权无过错责任是什么，物权编怎样划权利义务责任救济。

V6 必修：低频题只给“规则骨架 + 三句保分 + 不升核心”三件事，不放进核心样章叙事里。

## V6 必须重构的目录

# 选必二法律主观题满分训练宝典 V6：先会启动，再写满分

## 0. 使用边界先写清

- 当前证据底座：65 题。
- 学生主训：27 strict core。
- 非核心索引：38 题，只保分/边界/回源/参考/转出。
- reference_only：只练表达，不支撑核心。
- 任何新恢复题：必须重新锁题面和正式细则。

## 1. 第一屏：30 秒判题门

先不背六张卡，先过三道门：

1. 设问门：问结论、理由、意义、怎么办、补表格，还是边界判断？
2. 情境门：消费、劳动、合同、侵权、知识产权、程序救济、AI 风险、公共利益，还是其他模块？
3. 证据门：27 核心可满分训练；低频/source-check/reference-only/boundary/transfer 只能按标签保分或转出。

## 2. 法律题基本模型

`生活冲突 -> 核心争点 -> 法律翻译 -> 规则条件 -> 材料事实 -> 责任/程序/价值`

这一章必须先于动作卡。学生先学“法律题到底在干什么”，再学卡片。

## 3. 六类考场产品

1. 判决理由/责任链
2. 表格/笔记补全
3. 程序救济/起诉状/证据
4. 知识产权/不正当竞争/创新保护
5. AI 风险/合规/边界
6. 意义/价值/典型案例

每类给“阅卷人到底按什么给分”的最小单元。

## 4. 满分句工厂

按句子零件训练，不先给整篇答案：

- 表态句
- 争点句
- 规则条件句
- 材料嵌入句
- 责任/程序落点句
- 价值收束句

每个句型后面都给错法改法。

## 5. 主体法律动作积累页

按主体列句：消费者、劳动者、经营者、平台、权利人、法院、公益组织、AI 使用者/经营者。每个主体给“出现时能写的法律动作”，而不是只列教材名词。

## 6. 27 核心逐题训练

每题固定六栏：

1. 陌生题第一眼怎么启动
2. 争点一句话
3. 阅卷踩分单元
4. 满分句零件
5. 考场成稿
6. 错法改法

## 7. 38 非核心保分索引

放在核心训练后面，不抢首屏。每题固定：

- 标签：low-frequency / source-check / reference-only / boundary / transfer / crossref
- 可写最低三句
- 不能写成什么
- 能借什么，不能借什么
- 是否需要教师回源

## 8. 六大陷阱专项练

- 多小问漏问
- 表格格子错位
- AI 宏观化
- 价值空泛化
- source-check 当满分
- reference_only 误升核心

## 9. 教师证据附录

把 rubric atom trace、65 题状态、70->65 去向、source-check 待回源、reference_only 锁定依据放在后台。学生前台只保留必要红线。

## V6 首屏框架草案

```
选必二法律题 30 秒启动

第一步：先看设问，不先背知识。
- 问是否支持/是否有效/是否构成：先表态，再写规则和材料。
- 问理由/依据：一条理由 = 一条规则 + 一个材料事实 + 一个责任结果。
- 问意义/价值：本案怎么处理 -> 保护谁 -> 规范谁 -> 产生什么社会效果。
- 问怎么办/维权：证据 -> 请求 -> 路径 -> 效力。
- 问表格/补全：一格只答一个功能，先看示例行。
- 问边界/AI/治理：先判是不是纯法律题，不是就只写法律层最低句。

第二步：把材料翻译成法律话。
主体是谁，行为是什么，争点是什么，规则条件是什么，最后落到责任、程序或价值。

第三步：看证据标签。
27 核心题可以练闭合满分。
低频题只保分，不升母题。
source-check 题先按问法保分，正式使用前回源。
reference_only 只借表达，不支撑核心。
boundary/transfer 题先写法律层，再另开其他模块。

第四步：每段都按这条线写。
表态 -> 争点 -> 规则 -> 材料 -> 落点。
```

## 最关键裁定

V5.9 的失败不在于“没有证据纪律”，而在于“证据纪律还没有完全转化成学生脑内启动程序”。V6 不应扩核心、抢救 reference_only 或推翻 65 题底座；V6 应把前台改成：先判题门、再基本模型、再句子工厂、再 27 核心训练、最后才是 38 非核心索引和教师证据附录。
