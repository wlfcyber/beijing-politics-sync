# METHOD_LEARNING_NOTES

## 1. 我从旧强框架里学到的核心方法

旧强框架强在 **先教学生怎么想，再给学生写什么**。它的本质不是把知识点排漂亮，而是把陌生题压成学生能启动的动作。

最重要的教学顺序应是：

```text
先导页 → 基本模型 → 读题动作 → 主体积累 → 材料翻译 → 满分句生成 → 错法改法 → 证据附录
```

法律稿不能一开始就摆节点、证据等级、题号、句库。学生第一眼要知道：

```text
这题争什么？
我第一句写什么？
材料哪句话能变成法律词？
最后落责任、程序、效力，还是价值？
```

旧强框架的精髓可以概括成 8 条。

---

## 2. 方法一：先导页要先解释“这类题到底考什么”

法律主观题不是考学生背多少民法术语，而是考：

```text
把生活冲突翻译成法律判断。
```

学生必须先知道自己在做什么：

```text
生活事实 A → 法律判断 B
```

中间的训练链应是：

```text
主体 → 行为 → 争点 → 规则 → 材料事实 → 结果
```

这条线比“合同、侵权、劳动、知识产权”这种目录式框架更重要。

证据例子：

| question_id            | rubric_atom_id                                                                                                  | 说明                                                  |
| ---------------------- | --------------------------------------------------------------------------------------------------------------- | --------------------------------------------------- |
| `CC0143_2025_朝阳_一模_19` | `R_CC0143_2025_朝阳_一模_19_02` 至 `R_CC0143_2025_朝阳_一模_19_10`                                                       | 不是让学生泛写消费者维权，而是把购票付款、隐蔽搭售、意思表示受影响、欺诈、三倍赔偿逐步翻译成法律判断。 |
| `CC0244_2026_东城_期末_18` | `R_CC0244_2026_东城_期末_18_03`、`R_CC0244_2026_东城_期末_18_05`、`R_CC0244_2026_东城_期末_18_07`、`R_CC0244_2026_东城_期末_18_08` | 无人机题不是“维权建议”四个字能解决，而是要把合同责任、侵权或产品责任、证据准备、请求设计分层处理。  |
| `CC0002_2024_丰台_一模_17` | `R_CC0002_2024_丰台_一模_17_M17_01` 至 `R_CC0002_2024_丰台_一模_17_M17_06`                                               | 好意同乘题要求把无偿搭载、无故意或重大过失、未系安全带、垫付费用翻译成减责依据和现实意义。       |

## 3. 方法二：节点必须是动作，不能是教材目录

旧强框架好用，是因为它告诉学生：

```text
看到……先做……
```

法律稿也应这样：

```text
看到是否支持 → 先表态
看到表格 → 一格一答
看到付款出票 → 先判合同成立
看到派单签到奖惩 → 判三从属性
看到作品商标商业秘密 → 先认保护对象
看到怎么办维权 → 证据、请求、路径
看到意义价值 → 本案处理、保护谁、规范谁
```

这比直接写“合同编、侵权责任、劳动法、知识产权”更能救学生。

证据例子：

| question_id             | rubric_atom_id                                              | 应训练的动作                                      |
| ----------------------- | ----------------------------------------------------------- | ------------------------------------------- |
| `RECOVER_2026_通州_一模_20` | `R_RECOVER_2026_通州_一模_20_01` 至 `R_RECOVER_2026_通州_一模_20_06` | 表格题要先看列名和示例行，再逐格判断责任、法律依据、材料事实、意义。          |
| `RECOVER_2025_海淀_二模_18` | `R_RECOVER_2025_海淀_二模_18_01` 至 `R_RECOVER_2025_海淀_二模_18_09` | 笔记补全题要按空格功能写程序、证据、商标侵权、抗辩、原则、收获，不能写成知识产权作文。 |
| `CC0373_2026_顺义_一模_18`  | `R_CC0373_2026_顺义_一模_18_P01` 至 `R_CC0373_2026_顺义_一模_18_P03` | 多主体诉求题要分别表态，不能把闫某和张某的诉求混成一段。                |

## 4. 方法三：每题必须有“争点一句话”

旧法律好稿最值得学的是：

```text
材料一句话 → 核心争点 → 细则如何解决争点 → 作答如何解决争点
```

学生不是先问“这题属于哪个节点”，而是先问：

```text
这件事到底争什么？
哪个事实决定输赢？
谁的请求支持，谁的请求不支持？
```

证据例子：

| question_id               | rubric_atom_id                                                | 争点应如何压缩                                     |
| ------------------------- | ------------------------------------------------------------- | ------------------------------------------- |
| `CC0143_2025_朝阳_一模_19`    | `R_CC0143_2025_朝阳_一模_19_02` 至 `R_CC0143_2025_朝阳_一模_19_10`     | 争点是“付款出票后合同成立，但隐蔽搭售是否构成欺诈并触发消费者三倍赔偿”。       |
| `CC0025_2024_朝阳_二模_17`    | `R_CC0025_2024_朝阳_二模_17_02` 至 `R_CC0025_2024_朝阳_二模_17_17`     | 争点是“名义合作关系能不能遮蔽事实劳动关系，三从属性是否成立”。            |

## 5. 方法四：主体积累页应先于逐题演练

旧框架的“主体积累页”很强。法律稿必须让学生形成条件反射：

| 主体       | 学生第一反应              | 可写方向                           |
| -------- | ------------------- | ------------------------------ |
| 法院、仲裁委   | 支持或不支持请求，事实加规则      | 以事实为根据、以法律为准绳，结合行为性质和法律后果作出处理。 |
| 经营者      | 诚信、真实告知、不得欺诈        | 保护消费者知情权、自主选择权、公平交易权。          |
| 消费者      | 依法、诚信、有证据地维权        | 不能滥用权利，尤其惩罚性赔偿题要写权利边界。         |
| 平台或用工方   | 管理、控制、报酬、业务组成       | 判断人格从属性、经济从属性、组织从属性。           |
| AI 或平台企业 | 主体资格、提示义务、审核义务、责任边界 | 不能一看到 AI 就写数字治理作文。             |
| 当事人      | 证据、请求、路径            | 维权题必须写证据链和诉讼请求。                |

证据例子：

| question_id            | rubric_atom_id                                                                                                  | 主体动作                                 |
| ---------------------- | --------------------------------------------------------------------------------------------------------------- | ------------------------------------ |
| `CC0025_2024_朝阳_二模_17` | `R_CC0025_2024_朝阳_二模_17_02` 至 `R_CC0025_2024_朝阳_二模_17_17`                                                       | 平台出现，学生要先想三从属性。                      |
| `CC0063_2024_西城_二模_16` | `R_CC0063_2024_西城_二模_16_01`                                                                                     | 消费者和经营者同时出现，既要保护消费者，也要写权利不得滥用。       |
| `CC0244_2026_东城_期末_18` | `R_CC0244_2026_东城_期末_18_03`、`R_CC0244_2026_东城_期末_18_05`、`R_CC0244_2026_东城_期末_18_07`、`R_CC0244_2026_东城_期末_18_08` | 当事人维权题必须写邮件、付款、收货、鉴定、医疗票据、损失证明和请求设计。 |

## 6. 方法五：材料翻译表必须是真翻译，不是把材料拆句贴进去

旧强框架最重要的技术是：

```text
材料事实 → 学科语言 → 得分句
```

法律版也必须这样。比如：

| 材料信号           | 法律语言                     | 可写得分句                                   |
| -------------- | ------------------------ | --------------------------------------- |
| 付款、出票          | 合同成立、要约承诺完成              | 王某付款出票后，双方购票合同成立。                       |
| 擅自搭售、费用不清、无法拒绝 | 欺诈、知情权和自主选择权受损           | 平台隐蔽搭售影响真实意思表示，构成欺诈。                    |
| 派单、签到、奖惩、结算    | 人格从属性、经济从属性、组织从属性        | 不能只看合作协议名称，要按三从属性判断事实劳动关系。              |
| 包厢监控、聊天记录、公开发布 | 隐私权、个人信息、停止侵害            | 未经同意公开私人活动信息，侵害隐私权。                     |
| 名称图标近似、引人误认    | 混淆、不正当竞争                 | 擅自使用有一定影响的标识，易使消费者混淆。                   |
| AI 承诺、AI 幻觉    | AI 不具民事主体资格、平台提示义务、损害和因果 | AI 自动输出不构成独立意思表示，责任要看平台是否有过错及用户是否有实际损害。 |

证据例子：

| question_id            | rubric_atom_id                                            | 翻译重点                                |
| ---------------------- | --------------------------------------------------------- | ----------------------------------- |
| `CC0143_2025_朝阳_一模_19` | `R_CC0143_2025_朝阳_一模_19_02` 至 `R_CC0143_2025_朝阳_一模_19_10` | 付款出票翻译成合同成立；隐蔽搭售翻译成欺诈；诉求翻译成撤销和三倍赔偿。 |
| `CC0206_2025_西城_期末_19` | `R_CC0206_2025_西城_期末_19_01`                               | 名称、图标、宣传混淆翻译成不正当竞争和法治化营商环境。         |
| `CC0229_2026_东城_一模_18` | `R_CC0229_2026_东城_一模_18_01` 至 `R_CC0229_2026_东城_一模_18_08` | 调解、惩罚性赔偿、驳回恶意诉讼要分别翻译成司法保护创新的三个动作。   |

## 7. 方法六：满分句要有生成法，不要只有整段范文

旧强框架不只是给答案，更告诉学生答案如何长出来。法律稿也应固定成：

```text
结论句 → 规则句 → 材料句 → 落点句 → 价值句
```

对零基础学生来说，最小可写模型是：

```text
某请求应/不应支持。
依据相关法律，判断……应看……。
本案中，材料显示……。
因此……应承担责任/请求应获支持/应通过某路径维权。
该处理有利于保护……、规范……。
```

证据例子：

| question_id               | rubric_atom_id                                                  | 生成链                              |
| ------------------------- | --------------------------------------------------------------- | -------------------------------- |
| `CC0373_2026_顺义_一模_18`    | `R_CC0373_2026_顺义_一模_18_P01` 至 `R_CC0373_2026_顺义_一模_18_P03`     | 先分别表态闫某、张某诉求，再写规则和事实。            |
| `RECOVER_2026_朝阳_期末_18_1` | `R_RECOVER_2026_朝阳_期末_18_1_01` 至 `R_RECOVER_2026_朝阳_期末_18_1_05` | 要约、承诺、合同成立、合同有效、违约责任、约定违约责任依次生成。 |
| `RECOVER_2024_东城_二模_19_1` | `RECOVER_2024_东城_二模_19_1_R01` 至 `RECOVER_2024_东城_二模_19_1_R03`   | 起诉状题要按请求、事实与理由、证据生成，不能写实体责任散文。   |

## 8. 方法七：价值必须后置，且必须从本案推出

旧强框架最好的价值处理方式是：

```text
先处理本案，再推出价值。
```

法律稿必须禁止学生开头就写：

```text
维护公平正义
弘扬社会主义核心价值观
推进法治中国建设
促进社会和谐
```

正确顺序应是：

```text
本案法律处理 → 保护谁 → 规范谁 → 才能推出价值
```

证据例子：

| question_id            | rubric_atom_id                                                    | 价值必须从哪里推出                               |
| ---------------------- | ----------------------------------------------------------------- | --------------------------------------- |
| `CC0103_2025_丰台_一模_19` | `R_CC0103_2025_丰台_一模_19_01`                                       | 先写技术秘密、惩罚性赔偿、严惩侵权，再写创新保护和司法示范。不能裸写营商环境。 |
| `CC0223_2025_顺义_一模_19` | `R_CC0223_2025_顺义_一模_19_M17_01` 至 `R_CC0223_2025_顺义_一模_19_M17_06` | 先写案例中的相邻关系、企业名称纠纷、多元解纷，再谈定分止争价值。        |
| `CC0045_2024_海淀_二模_19` | `R_CC0045_2024_海淀_二模_19_01`                                       | 先写遗赠扶养协议与养老安排，再谈社会价值，不能直接写德治法治口号。       |

## 9. 方法八：主干和开放容器必须分层

旧强框架有“主干地图”和“容器”意识。法律稿也必须承认：65 题里不是所有题都能进核心满分模板。

当前数据口径：

```text
65 道主观法律题
61 formal
4 reference_only
V6.9 当前候选稿定位：failed/insufficient candidate to critique
```

处理原则：

| 类型                  | 学生版位置  | 证据处理                             |
| ------------------- | ------ | -------------------------------- |
| 高频 formal 核心题       | 主干训练   | 必须有 question_id + rubric_atom_id |
| formal 但低频或未稳题      | 低频保分容器 | 不升母题                             |
| source-check 题      | 保分索引   | 不宣称满分闭合                          |
| reference_only      | 参考练笔   | 不支撑核心节点                          |
| boundary / transfer | 边界保险箱  | 只写法律层最低句或转出                      |

证据例子：

| question_id              | rubric_atom_id                                                  | 应放位置                   |
| ------------------------ | --------------------------------------------------------------- | ---------------------- |
| `CC0011_2024_丰台_二模_17`   | `R_CC0011_2024_丰台_二模_17_01`                                     | 低频生态容器，不作高频母题。         |
| `CC0040_2024_海淀_一模_19`   | `R_CC0040_2024_海淀_一模_19_01`                                     | reference_only，只能参考练笔。 |
| `CC0380_2026_顺义_二模_18_2` | `R_CC0380_2026_顺义_二模_18_2_P01` 至 `R_CC0380_2026_顺义_二模_18_2_P04` | 边界开放题，只能写法律层最低句。       |
| `CC0364_2026_通州_期末_19_1` | `R_CC0364_2026_通州_期末_19_1_01` 至 `R_CC0364_2026_通州_期末_19_1_08`   | 逻辑模块边界，不能塞进法律核心。       |

# WHY_V6_9_STILL_NOT_ENOUGH

## 0. 总体判断

V6.9 比旧版干净，但仍然不够好。

它已经通过了文本卫生扫描和 27 核心小节完整性审计；DOCX 结构和 QuickLook 首屏也通过。但这些只能证明 **文件没明显脏词、核心题数量没丢、首屏能打开**。它不能证明一个零基础聪明高三学生读完就能稳定接近满分。

V6.9 的根本问题是：

```text
它看起来像学生稿，但很多地方仍是“证据抽取结果的学生化改写”。
它还没有真正变成“学生如何从陌生题面生成答案”的训练系统。
```

我最严厉的判断是：

```text
V6.9 可以作为材料底稿。
V6.9 不能作为最终宝典。
```

---

## 1. V6.9 的“材料翻译表”很多不是真翻译

这是最严重的问题。

方法学习包明确要求：

```text
材料事实 → 法律语言 → 得分句
```

但 V6.9 多题的“材料翻译表”把题干开头、背景句、新闻发布会介绍、甚至整段材料片段塞进“材料事实/信号”，再机械配上法律句。这对学生没有训练价值。

### 例 1：`CC0143_2025_朝阳_一模_19`

V6.9 的材料翻译表把：

```text
司法护航，营造在线文旅消费法治环境
某市互联网法院召开新闻发布会
页面显示总价……
```

这类题干背景当成材料信号。真正决定得分的材料信号应是：

```text
王某付款出票
A 公司擅自搭售 10 元外卖红包
王某不能清楚知悉费用细节
王某无法拒绝支付
```

对应证据：

```text
question_id: CC0143_2025_朝阳_一模_19
rubric_atom_id: R_CC0143_2025_朝阳_一模_19_02 至 R_CC0143_2025_朝阳_一模_19_10
```

问题后果：学生会以为“新闻发布会、司法护航、消费环境”也是主要得分材料，考试时会把理由题写成在线文旅消费治理大作文。

---

### 例 2：`CC0063_2024_西城_二模_16`

V6.9 把：

```text
16．（）市民甲购买黄芪薏米饼干……
四次总计付款……
```

这种原材料切片配成法律句，但没有在翻译表里清楚告诉学生：

```text
首单 30 盒 → 正常生活消费需要 → 支持惩罚性赔偿
后续三次大量加购 → 明知问题后扩大购买 → 权利不得滥用
```

对应证据：

```text
question_id: CC0063_2024_西城_二模_16
rubric_atom_id: R_CC0063_2024_西城_二模_16_01
```

问题后果：学生知道要写“惩罚性赔偿”和“不得滥用权利”，但未必能精准区分首单与加购，容易一边倒支持消费者，漏掉法院判决意义的核心分层。

---

### 例 3：`RECOVER_2026_朝阳_期末_18_1`

V6.9 的材料翻译表把：

```text
网络消费成为人们消费的重要方式
张某系某网络店铺的经营者
直播营销中……
```

直接配成：

```text
主播宣称……属于要约
侯某购买并支付……属于承诺
承诺生效后合同成立
```

这不是完全错误，但训练力度不够。学生需要看到的是：

```text
主播“保真”“假一赔十” → 要约内容和合同条款
侯某下单付款 → 承诺
材质不符 → 未按约履行
假一赔十 → 约定违约责任
```

对应证据：

```text
question_id: RECOVER_2026_朝阳_期末_18_1
rubric_atom_id: R_RECOVER_2026_朝阳_期末_18_1_01 至 R_RECOVER_2026_朝阳_期末_18_1_05
```

问题后果：学生能背完整答案，但遇到另一个直播交易题，未必能自己从“承诺保真、假一赔十、材质不符”翻译出合同成立、合同有效和违约责任。

## 2. V6.9 的“材料一句话”经常不是一句话，而是原题粘贴

旧强框架要求：

```text
起因 → 经过 → 结果
```

压成完整小故事。V6.9 仍常把题干和材料大段粘进“材料一句话”。

这会破坏学生的读题训练。学生看到一大段原文，不会学到“怎样压缩争点”。

典型风险题：

| question_id               | rubric_atom_id                                                  | 问题                                                   |
| ------------------------- | --------------------------------------------------------------- | ---------------------------------------------------- |
| `CC0143_2025_朝阳_一模_19`    | `R_CC0143_2025_朝阳_一模_19_02` 至 `R_CC0143_2025_朝阳_一模_19_10`       | “材料一句话”里保留新闻发布会、题干背景、页面金额等大量文本，没有压成“购票付款、隐蔽搭售、起诉维权”。 |
| `CC0063_2024_西城_二模_16`    | `R_CC0063_2024_西城_二模_16_01`                                     | “材料一句话”几乎是题干复述，没有压成“首单正常消费，后续明知加购，法院部分支持”。           |
| `RECOVER_2026_朝阳_期末_18_1` | `R_RECOVER_2026_朝阳_期末_18_1_01` 至 `R_RECOVER_2026_朝阳_期末_18_1_05` | 原材料截断式粘贴，不能教学生自己压缩合同争点。                              |

严厉结论：如果“材料一句话”不能让学生一眼知道争点，它就是伪学生化。

## 3. V6.9 仍把“战场”当主结构，争点还没有真正站到第一层

V6.9 有九个战场：

```text
表格补全
合同成立与履行
消费交易
劳动用工
实体责任
创新竞争
程序救济
AI 风险边界
价值收束
```

这比教材目录好，但还不是最强学生结构。学生真正需要的是：

```text
这题争什么？
第一句判什么？
哪个事实决定输赢？
```

“战场”只是分类。分类不能替代争点判断。

例如：

| question_id            | V6.9 所属战场    | 学生真正第一判断                                |
| ---------------------- | ------------ | --------------------------------------- |
| `CC0244_2026_东城_期末_18` | 合同成立与履行、程序救济 | 先拆两问：第 1 问写合同责任和侵权/产品责任；第 2 问写证据、请求、路径。 |
| `CC0373_2026_顺义_一模_18` | 实体责任         | 先分别判断闫某诉求能否支持、张某诉求能否支持。                 |
| `CC0223_2025_顺义_一模_19` | 价值收束         | 先把两个案例分别压成相邻关系纠纷和名称权/市场主体纠纷，再谈定分止争。     |

对应证据：

```text
CC0244_2026_东城_期末_18: R_CC0244_2026_东城_期末_18_03, R_CC0244_2026_东城_期末_18_05, R_CC0244_2026_东城_期末_18_07, R_CC0244_2026_东城_期末_18_08
CC0373_2026_顺义_一模_18: R_CC0373_2026_顺义_一模_18_P01 至 R_CC0373_2026_顺义_一模_18_P03
CC0223_2025_顺义_一模_19: R_CC0223_2025_顺义_一模_19_M17_01 至 R_CC0223_2025_顺义_一模_19_M17_06
```

结论：V6.9 的“战场”可以保留为后台组织，但学生前台必须先是“设问尾词 + 争点句 + 第一判断”。

## 4. 三句保底容易让学生误以为自己已经会满分

V6.9 每题都有“三句保底答案”，这是进步。但很多三句过于成文，学生可能直接背三句停下，不会继续嵌材料。

典型风险：

| question_id             | rubric_atom_id                                              | 风险                                                    |
| ----------------------- | ----------------------------------------------------------- | ----------------------------------------------------- |
| `CC0025_2024_朝阳_二模_17`  | `R_CC0025_2024_朝阳_二模_17_02` 至 `R_CC0025_2024_朝阳_二模_17_17`   | 三从属性题如果只背“人格、经济、组织从属性”，不嵌派单、签到、奖惩、数据、费用、业务组成，会丢分析分。   |
| `CC0229_2026_东城_一模_18`  | `R_CC0229_2026_东城_一模_18_01` 至 `R_CC0229_2026_东城_一模_18_08`   | 如果只背“法院保护知识产权、维护公平竞争、激励创新”，会漏调解、惩罚性赔偿、驳回恶意诉讼三个具体案例动作。 |
| `RECOVER_2026_通州_一模_20` | `R_RECOVER_2026_通州_一模_20_01` 至 `R_RECOVER_2026_通州_一模_20_06` | 表格题如果只背三句，不按真实列名逐格作答，依然会丢格子分。                         |

V6.9 虽然提醒“三句不是最终满分答案”，但还不够硬。每题应强制分成：

```text
只背这三句能保底
冲满分必须补哪 3 个材料事实
漏哪一个必丢分
```

## 5. V6.9 的 value 处理仍有必修三化诱因

V6.9 已经比早期版本更克制，但一些意义句仍容易诱导学生写空话。

高风险表达包括：

```text
弘扬社会主义核心价值观
促进社会和谐
推动法治中国建设
维护公平正义
增强法治意识
```

这些不是绝对禁用，但必须后置，且必须从本案推出。

典型题：

| question_id            | rubric_atom_id                                                    | 风险                                          |
| ---------------------- | ----------------------------------------------------------------- | ------------------------------------------- |
| `CC0103_2025_丰台_一模_19` | `R_CC0103_2025_丰台_一模_19_01`                                       | 该题细则对表达有区分，不能把“法治中国建设”“优化营商环境”随便写进科技创新意义题。  |
| `CC0223_2025_顺义_一模_19` | `R_CC0223_2025_顺义_一模_19_M17_01` 至 `R_CC0223_2025_顺义_一模_19_M17_06` | 定分止争价值题最容易写成治理大作文，必须先写两个具体案例的权利边界和纠纷化解。     |
| `CC0045_2024_海淀_二模_19` | `R_CC0045_2024_海淀_二模_19_01`                                       | 遗赠扶养协议题不能只写孝老敬亲和核心价值观，必须先写协议安排、养老保障和司法确认价值。 |

严厉结论：只要价值句不能回答“本案保护谁、规范谁、示范什么”，就应删掉或降为尾句。

## 6. V6.9 仍有粗 atom 题被写得过于闭合

部分核心题只有一个粗粒度 rubric atom，但 V6.9 把它写成完整闭合样章。这样做有风险：学生以为整段都稳定，实际上内部采点还需要拆。

典型题：

## 7. 表格题仍然没有完全从“答案段落”切换到“格子训练”

V6.9 在表格战场里已经加入“模拟格子”，这是进步。但从学生训练看仍不够。

表格题必须训练：

```text
列名是什么
示例行是什么
这一格问机制、依据、事实、证据、结果、意义中的哪一个
每格只能写一个功能
```

典型证据：

| question_id             | rubric_atom_id                                              | 必须训练                                    |
| ----------------------- | ----------------------------------------------------------- | --------------------------------------- |
| `RECOVER_2026_通州_一模_20` | `R_RECOVER_2026_通州_一模_20_01` 至 `R_RECOVER_2026_通州_一模_20_06` | 自甘风险、安全保障义务、故意或重大过失、因果关系、责任边界、意义必须按格处理。 |
| `RECOVER_2025_海淀_二模_18` | `R_RECOVER_2025_海淀_二模_18_01` 至 `R_RECOVER_2025_海淀_二模_18_09` | 程序、证据、商标侵权、抗辩、诚信原则、市场秩序、旁听收获必须逐空分开。     |

严厉判断：如果表格题最终仍以完整段落为主，学生在真实卷面上会知道答案内容，却不知道填在哪个格子里。

## 8. V6.9 的非核心区仍像“附录”，没有成为真正的保险箱

V6.9 有 38 非核心保分索引，这是必要结构。但它仍偏像附录题库，保险箱功能不够强。

非核心题应明确四句话：

```text
能写什么
不能写什么
适用什么设问
不适用什么设问
```

现在很多非核心卡还只是“最低三句 + do_not_write”，不够临场。

典型风险：

## 9. V6.9 的逐题演练太长，学生会读懂但未必会迁移

V6.9 每题很完整，但结构偏重：

```text
启动
三句保底
材料一句话
争点
材料翻译表
满分句零件
完整答案
错法改法
迁移提醒
```

这套格式理论上对，但执行上很多题篇幅太长。零基础学生可能出现两种结果：

1. 读完单题，觉得懂。
2. 遇到新题，忘了启动动作。

旧强框架解决这个问题的方法是压缩主线：

```text
总模型
少数动作
主体积累
情境触发
错法改法
```

V6.9 还没有把 27 题浓缩成真正可背的“能力单元”。它更像 27 个样章，而不是一个能反复调用的训练系统。

## 10. V6.9 的首页虽然进步，但还没有把“设问尾词诊断”做到绝对首位

V6.9 首页有“30 秒判题门”，这是正确方向。但学生最需要的首屏应更硬：

```text
先看设问尾词
判断答案产品
第一句怎么写
```

应把“设问尾词诊断树”放到最前，而不是先讲大量“法律主观题考什么”“主体动作”“情境词典”。

学生拿到题时不是先想整本书地图，而是先看：

| 设问尾词     | 第一动作     |
| -------- | -------- |
| 是否支持     | 先表态      |
| 理由依据     | 规则加材料    |
| 意义价值     | 本案处理后推价值 |
| 怎么办维权    | 证据、请求、路径 |
| 表格补全     | 看列名和示例   |
| AI、治理、逻辑 | 先判是否边界   |

V6.9 已经有这些元素，但顺序还不够“考场第一秒”。

## 12. V6.9 最危险的误导

最危险的地方不是它错得离谱。最危险的是它 **看起来已经很像成品**。

学生会以为：

```text
我背了 27 核心和 38 保分，就能覆盖 65 题。
```

但实际风险是：

1. 遇到材料翻译表没有训练过的新材料，仍不会从事实生成法律句。
2. 遇到价值题，仍可能从“法治中国、核心价值观、社会和谐”开写。
3. 遇到表格题，仍可能知道内容却不会按格填。
4. 遇到 reference_only 或 boundary 题，仍可能错当核心模板。
5. 遇到粗 atom 题，仍可能只背整段，漏隐含采点。

## 13. 对 V6.9 的底线裁定

我的底线判断：

```text
V6.9 文本卫生：通过。
V6.9 结构完整：通过。
V6.9 作为方法学习材料：可用。
V6.9 作为最终学生宝典：不通过。
```

原因不是证据少，而是教学前台还没有彻底完成三件事：

```text
争点前置
材料真翻译
错法强改法
```

下一步等你上传 batch 01 到 04 后，应该按批次重建，而不是在 V6.9 上继续润色。V7 不应复制 V6.9 的 27 题样章结构；V7 应先建立“学生如何从陌生题面生成答案”的训练逻辑，再把每批题放进去验证。
