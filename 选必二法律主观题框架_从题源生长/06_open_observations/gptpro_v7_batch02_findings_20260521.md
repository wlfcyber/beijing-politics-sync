## batch_mechanisms

### 机制 1：程序题不是“写维权”，而是先判任务产品

本批的程序救济题反复出现 4 种不同任务产品，不能都放进旧 V6.9 的“排维权步骤”。

| 任务产品      | 学生要产出的答案                           | 代表题                       | 证据                                                                                                              |
| --------- | ---------------------------------- | ------------------------- | --------------------------------------------------------------------------------------------------------------- |
| 调解方案      | 调解结果 + 调解理由 + 合同后果 + 证据意识 + 商家责任边界 | `CC0125_2025_延庆_一模_19`    | `R_CC0125_2025_延庆_一模_19_M17_01` 至 `R_CC0125_2025_延庆_一模_19_M17_05`                                               |
| 起诉状       | 代理方 + 诉讼请求 + 事实理由 + 证据             | `RECOVER_2024_东城_二模_19_1` | `RECOVER_2024_东城_二模_19_1_R01` 至 `RECOVER_2024_东城_二模_19_1_R03`                                                   |
| 维权准备      | 证据准备 + 请求设计 + 路径选择                 | `CC0244_2026_东城_期末_18`    | `R_CC0244_2026_东城_期末_18_03`、`R_CC0244_2026_东城_期末_18_05`、`R_CC0244_2026_东城_期末_18_07`、`R_CC0244_2026_东城_期末_18_08` |
| 违法识别和诉讼请求 | 违法类型 + 权利受害人 + 请求内容 + 事实理由         | `RECOVER_2026_延庆_一模_18_1` | `RECOVER_2026_延庆_一模_18_1_R01` 至 `RECOVER_2026_延庆_一模_18_1_R03`                                                   |

旧 V6.9 节点处理：
`排维权步骤` 必须降级为后台总类。前台应拆成“调解方案”“起诉状”“证据请求路径”“违法识别与请求”四个动作。否则学生看到“怎么办”就只会写协商、调解、仲裁、诉讼，漏具体采点。

### 机制 2：证据题不是“列证据”，而是“证据为请求服务”

本批所有证据采点都服务于具体请求或抗辩，不能孤立列材料。

| 题                         | 证据服务谁              | 证据作用                               | 证据                                                                                  |
| ------------------------- | ------------------ | ---------------------------------- | ----------------------------------------------------------------------------------- |
| `CC0125_2025_延庆_一模_19`    | 杨某主张商品质量不合格        | 商品已丢弃，缺乏证据意识，影响其主张成立               | `R_CC0125_2025_延庆_一模_19_M17_04`                                                     |
| `CC0244_2026_东城_期末_18`    | 陈某主张违约责任、侵权责任、损害赔偿 | 邮件、转账、收货、检测、医疗费、商业损失证明分别服务合同、侵权、损失 | `R_CC0244_2026_东城_期末_18_03`、`R_CC0244_2026_东城_期末_18_05`、`R_CC0244_2026_东城_期末_18_08` |
| `RECOVER_2025_海淀_二模_18`   | 商标侵权或在先权利抗辩        | 包装、书证、物证、证人证言等必须对应商标使用、知名度、在先权利    | `R_RECOVER_2025_海淀_二模_18_02` 至 `R_RECOVER_2025_海淀_二模_18_06`                         |
| `RECOVER_2024_东城_二模_19_1` | 起诉状中的诉讼请求          | 合同、延期沟通、仓储费用、垫付或拒付事实支撑请求           | `RECOVER_2024_东城_二模_19_1_R01` 至 `RECOVER_2024_东城_二模_19_1_R03`                       |

旧 V6.9 节点处理：
“证据准备”不能作为空泛清单。应删掉“保存相关证据”这种孤句，改成“某证据证明某请求”。

---

### 机制 3：请求题必须写“请求内容 + 法律理由 + 材料理由”

本批请求类不是建议题，必须写得像诉讼请求或调解请求。

| 题                         | 请求对象          | 请求内容                    | 证据                                                                                  |
| ------------------------- | ------------- | ----------------------- | ----------------------------------------------------------------------------------- |
| `RECOVER_2024_东城_二模_19_1` | 甲公司或乙公司任选一方代理 | 支付尾款、承担仓储费、继续履行、承担诉讼费用  | `RECOVER_2024_东城_二模_19_1_R01`                                                       |
| `RECOVER_2026_延庆_一模_18_1` | 主持人李某         | 停止侵害、赔偿损失、承担诉讼费用、赔礼道歉   | `RECOVER_2026_延庆_一模_18_1_R02`                                                       |
| `CC0244_2026_东城_期末_18`    | 陈某            | 主张违约损失、人身损害赔偿，区分违约和侵权请求 | `R_CC0244_2026_东城_期末_18_03`、`R_CC0244_2026_东城_期末_18_05`、`R_CC0244_2026_东城_期末_18_08` |

旧 V6.9 节点处理：
`排维权步骤` 里的“协商、调解、仲裁、诉讼”只能作路径尾句，不能替代请求内容。学生必须先写“请求什么”，再写“为什么能请求”。

---

### 机制 4：表格补全题按“格子功能”采点，不按整段答案采点

本批两道表格或补全题，核心都是一格一答。

| 题                       | 格子类型                              | 学生动作                        | 证据                                                          |
| ----------------------- | --------------------------------- | --------------------------- | ----------------------------------------------------------- |
| `RECOVER_2025_海淀_二模_18` | 程序格、事实格、侵权格、证据格、抗辩格、原则格、价值格、旁听收获格 | 先判断每个空格要填程序、事实、证据、规则还是价值    | `R_RECOVER_2025_海淀_二模_18_01` 至 `R_RECOVER_2025_海淀_二模_18_09` |
| `RECOVER_2026_通州_一模_20` | 法律依据格、事实分析格、责任判断格、意义格             | 按民法典依据、张某自身原因、李某尽义务、三层意义分别填 | `R_RECOVER_2026_通州_一模_20_01` 至 `R_RECOVER_2026_通州_一模_20_06` |

旧 V6.9 节点处理：
`一格一答` 保留，但必须改成“看列名、看示例、判格子功能”。不能只给完整段落答案。表格题最容易出现“内容对但填错格”的失分。

---

### 机制 5：归责三档必须区分，不得混写

本批最重要的归责训练有三档：

| 归责档      | 适用场景                       | 不能混淆                    | 证据                                                                                                              |
| -------- | -------------------------- | ----------------------- | --------------------------------------------------------------------------------------------------------------- |
| 合同违约档    | 要约、承诺、合同成立有效、未全面履行         | 不能用侵权要件分析违约             | `R_CC0244_2026_东城_期末_18_03`、`R_CC0244_2026_东城_期末_18_05`、`R_CC0244_2026_东城_期末_18_07`、`R_CC0244_2026_东城_期末_18_08` |
| 一般侵权档    | 销售者刘某造成损害，需分析行为、损害、因果关系、过错 | 不能写成无过错产品责任             | `R_CC0244_2026_东城_期末_18_05`、`R_CC0244_2026_东城_期末_18_08`                                                         |
| 产品责任无过错档 | 对生产者主张产品责任时，重在产品缺陷、损害、因果关系 | 不能写“无过错推定”或“销售者适用无过错责任” | `R_CC0244_2026_东城_期末_18_05`、`R_CC0244_2026_东城_期末_18_06`                                                         |

另有一个责任边界型归责：

| 题                       | 归责结论                                                | 证据                                                          |
| ----------------------- | --------------------------------------------------- | ----------------------------------------------------------- |
| `RECOVER_2026_通州_一模_20` | 李某作为义务组织者已尽安全保障义务，不存在故意或重大过失，张某死亡与李某行为无因果关系，不承担侵权责任 | `R_RECOVER_2026_通州_一模_20_01` 至 `R_RECOVER_2026_通州_一模_20_03` |

旧 V6.9 节点处理：
`实体责任` 必须删除或降级。它太粗，会诱导学生把合同、一般侵权、产品责任、自甘风险混成一段。

---

### 机制 6：公益诉讼题按“主体、方式、效力”三格写

`RECOVER_2026_门头沟_一模_18_1` 不是普通环保价值题，也不是泛泛程序题，而是三层硬采点：

| 层次 | 学生必须写                     | 证据                             |
| -- | ------------------------- | ------------------------------ |
| 主体 | 公益社会组织承担社会责任，提起环境民事公益诉讼   | `RECOVER_2026_门头沟_一模_18_1_R01` |
| 方式 | 通过和解、司法调解、诉讼等多元纠纷解决方式消除风险 | `RECOVER_2026_门头沟_一模_18_1_R02` |
| 效力 | 经法院依法确认的调解协议或民事调解书具有强制执行力 | `RECOVER_2026_门头沟_一模_18_1_R03` |

旧 V6.9 节点处理：
`划风险边界` 可降级为辅助提醒。该题主轴不是 AI 或开放风险，而是“公益诉讼主体、纠纷解决方式、司法确认效力”。

## student_start_moves

### 1. 看到“如何调解并说明理由”

适用：

```text
CC0125_2025_延庆_一模_19
R_CC0125_2025_延庆_一模_19_M17_01 至 R_CC0125_2025_延庆_一模_19_M17_05
```

学生先做：

```text
先判断调解要解决哪三个问题：钱退了，货还不还，证据还在不在。
```

第一步写：

```text
本案标的较小，宜本着以和为贵原则适用调解。
```

接着按四个理由：

```text
消费者维权应遵循诚实信用原则。
退款后买卖合同已经解除，应依法退还货物。
主张商品质量不合格应有证据，杨某已丢弃商品，缺乏证据意识。
经营者应保证商品质量，若商品确有问题则不应追究消费者责任。
```

不能写：

```text
双方应互相体谅，协商解决。
```

这只有态度，没有合同、证据和质量责任。

### 2. 看到“法律责任及法律依据”且材料含交易损害

适用：

```text
CC0244_2026_东城_期末_18
R_CC0244_2026_东城_期末_18_03
R_CC0244_2026_东城_期末_18_05
R_CC0244_2026_东城_期末_18_07
R_CC0244_2026_东城_期末_18_08
```

学生先做：

```text
先拆两问：第 1 问写责任，第 2 问写准备。
```

第 1 问先分三档：

```text
合同档：邮件询价和报价构成要约承诺，合同成立并有效；刘某交付受损展示机，未全面履行，构成违约。
一般侵权档：对刘某主张侵权时，要写行为、损害、因果关系、过错。
产品责任档：对生产者主张产品责任时，适用无过错责任，写产品缺陷、损害和因果关系。
```

第 2 问写：

```text
证据准备：邮件、转账、收货、检测、医疗费、商业拍摄损失证明、协商记录。
请求设计：违约请求和侵权请求分开写。
路径选择：协商未果后可依法起诉。
```

不能写：

```text
刘某欺诈陈某。
```

细则明确提醒本题不构成欺诈，见 `R_CC0244_2026_东城_期末_18_05`。

### 3. 看到“将笔记中划线处补充完整”

适用：

```text
RECOVER_2025_海淀_二模_18
R_RECOVER_2025_海淀_二模_18_01 至 R_RECOVER_2025_海淀_二模_18_09
```

学生先做：

```text
先看每个空在模拟法庭笔记里的位置，再判断要填程序、事实、证据、规则还是价值。
```

逐空动作：

```text
程序空：填回避。
事实空：填被告使用的商品包装。
侵权空：填未经许可使用商标权或侵犯商标权。
证据空：填书证、物证、证人证言或对应材料。
抗辩空：填老字号、知名度、在先权利。
规则空：填恶意抢注、损害在先权利或商标法第三十二条。
原则空：填诚信、公平或公序良俗，且必须是民法原则。
价值空：填市场秩序、营商环境、公平竞争等。
旁听收获：写获得新知、感受司法公正、体会法治价值三层。
```

不能写成完整知识产权作文。

### 4. 看到“完成下表”且材料是组织者责任

适用：

```text
RECOVER_2026_通州_一模_20
R_RECOVER_2026_通州_一模_20_01 至 R_RECOVER_2026_通州_一模_20_06
```

学生先做：

```text
先判断李某是否尽到安全保障义务，张某死亡是否由自身原因造成，二者是否有因果关系。
```

格子作答顺序：

```text
法律依据格：必须写《中华人民共和国民法典》，只写相关法律规定不给分。
事实分析格：张某自愿参加，因自身原因突发意外，并非他人行为造成。
责任判断格：李某事前提醒、事后救助，已尽安全保障义务，不存在故意或重大过失，与死亡无因果关系，不承担侵权责任。
意义格：增强风险意识，合理界定义务组织者责任范围，体现公平原则，鼓励互帮互助。
```

不能写：

```text
李某是义务组织者，所以完全没有责任。
```

要写尽到安全保障义务、无故意或重大过失、无因果关系。

### 5. 看到“选择一方代理并撰写起诉状”

适用：

```text
RECOVER_2024_东城_二模_19_1
RECOVER_2024_东城_二模_19_1_R01 至 RECOVER_2024_东城_二模_19_1_R03
```

学生先做：

```text
先选代理方，再定请求方向。
```

如果代理乙公司：

```text
诉讼请求：请求甲公司支付尾款、继续履行合同，并承担诉讼费用。
事实理由：双方买卖合同有效；延期仓储费没有约定由乙公司承担；甲公司拒付尾款违反全面履行和诚信原则。
证据：买卖合同、延期沟通记录、收货记录、仓储费凭证、催款记录。
```

如果代理甲公司：

```text
诉讼请求：请求乙公司承担全部仓储费用或继续履行相关义务，并承担诉讼费用。
事实理由：乙公司同意延期收货安排，仓储费用争议与合同履行有关。
证据：合同、延期协商记录、仓储缴费凭证、收货和拒付记录。
```

不能两边都代理，也不能只写“请求法院公平处理”。

### 6. 看到“为避免类似问题产生，我们应该……”

适用：

```text
RECOVER_2024_东城_二模_19_2
RECOVER_2024_东城_二模_19_2_R01
RECOVER_2024_东城_二模_19_2_R02
```

学生先做：

```text
先判断纠纷原因是合同内容不清，不是事后谁态度不好。
```

满分方向：

```text
合同内容应清晰明确，尤其要明确双方权利义务、费用承担、履行期限和意思表示。
```

只写：

```text
协商一致
```

最多 1 分。写诚信履行、全面履行不给分，见 `RECOVER_2024_东城_二模_19_2_R02`。

### 7. 看到“涉嫌哪些违法行为，可提出哪些诉讼请求”

适用：

```text
RECOVER_2026_延庆_一模_18_1
RECOVER_2026_延庆_一模_18_1_R01 至 RECOVER_2026_延庆_一模_18_1_R03
```

学生先做：

```text
先按受害对象拆违法线。
```

三条违法线：

```text
对李某：侵犯肖像权、名誉权。
对消费者：侵犯知情权。
对市场竞争秩序：虚假和引人误解的商业宣传，构成不正当竞争。
```

诉讼请求：

```text
停止侵害、赔偿损失、承担诉讼费用、赔礼道歉。
```

理由：

```text
M 公司未经授权利用 AI 合成李某形象进行虚假直播营销，借公众人物声誉和粉丝信任攫取流量，违背诚信原则。
```

不能只写：

```text
侵犯 AI 数字人权益。
```

本案受害主体是李某、消费者和市场秩序。

---

### 8. 看到“如何消除损害风险，维护生态环境公共利益”

适用：

```text
RECOVER_2026_门头沟_一模_18_1
RECOVER_2026_门头沟_一模_18_1_R01 至 RECOVER_2026_门头沟_一模_18_1_R03
```

学生先做：

```text
先判断这是环境民事公益诉讼，不是普通环保意义题。
```

三层启动：

```text
主体层：公益社会组织承担社会责任，提起环境民事公益诉讼。
方式层：通过和解、司法调解、诉讼等多元纠纷解决方式消除风险。
效力层：法院依法确认的调解协议或民事调解书具有强制执行力。
```

不能只写：

```text
保护生态环境，维护公共利益。
```

那会漏主体、方式、效力三个硬点。

```text
AI 数字人违法，应承担法律责任。
```

违法类型和诉讼请求必须具体。

### 规则 8：环境公益诉讼题按“主体、方式、效力”写

适用证据：

```text
RECOVER_2026_门头沟_一模_18_1
RECOVER_2026_门头沟_一模_18_1_R01
RECOVER_2026_门头沟_一模_18_1_R02
RECOVER_2026_门头沟_一模_18_1_R03
```

句式规则：

```text
主体层：谁有资格为公共利益起诉。
方式层：通过哪些纠纷解决方式消除风险。
效力层：法院确认后为什么能落实。
```

本批可写版本：

```text
本案中，公益社会组织承担社会责任，依法提起环境民事公益诉讼，维护生态环境公共利益。从纠纷解决方式看，案件通过和解、司法调解、诉讼等多元方式推动风险消除。法院对首份和解协议不予确认，体现了对公共利益的审查；后经技术论证和协商形成调解协议，并由法院出具民事调解书确认。经人民法院依法确认有效的调解协议具有强制执行力，能够推动生态修复责任真正履行。
```

不能写：

```text
本案保护生态环境，体现绿色发展。
```

这只写价值，漏主体、方式、效力。
