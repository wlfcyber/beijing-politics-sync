# V7_FRAMEWORK_PROPOSAL

## 0. 总判

V7 前台必须从 **学生考场第一动作** 出发，不再沿用 V6.9 的九个战场作学生可见主结构。

V7 的核心对象不是教材知识点，也不是题号分类，而是：

```text
设问要我产出什么答案产品？
第一句判什么？
材料哪句话翻译成哪个法律词？
法律规则用哪一句？
最后落到责任、效力、请求、证据、价值，还是边界降级？
```

证据纪律仍沿用两层结构：formal 核心可训练，reference_only 只练笔，boundary/transfer/source-check 不得升核心。此前工程包也明确要求保留 high-frequency core trunk 与 open/boundary/reference containers 的双层结构，reference_only 不能升 formal support。

## 2. V6.9 前台应删或降级的节点

| V6.9 节点               | V7 处理                                                     |
| --------------------- | --------------------------------------------------------- |
| 九个战场总结构               | 不作学生前台。后台可保留用于审计归档。                                       |

第三句：材料句，把题中决定输赢的事实翻译成法律词。
第四句：落点句，写责任、效力、请求、证据、路径或边界。
