## 一、共同否定

GPTPro 与 Claude 均不建议继续沿用 V6.9 的“九个战场”作为学生前台。

共同理由：

1. “九个战场”容易变成教材目录或法学分类，学生看到陌生题仍不知道第一句怎么写。
2. V6.9 的材料翻译和逐题样例有可用零件，但装配顺序错误。
3. 学生前台应先解决“设问要我交什么答案成品”，再进入法律规则。
4. 价值表达必须由本案法律处理推出，不能作为第一入口。
5. `reference_only` 只能练表达，不得支撑核心框架。

代表证据：

- Claude：`06_open_observations/claude_opus_v7_method_first_rebuild_20260521.md`
- GPTPro：`06_open_observations/gptpro_v7_method_stage_20260521.md`
- GPTPro batch：`gptpro_v7_batch01_findings_20260521.md` 至 `gptpro_v7_batch04_findings_20260521.md`
- GPTPro final：`09_candidate_frameworks/gptpro_v7_framework_proposal_20260521.md`

## 二、共同保留的旧框架方法

两边都确认：用户先前框架可学习的是“教学方法”，不是法律结论。

可迁移方法：

1. 先导页先讲这本书在考什么、学生会死在哪里。
2. 给一个会生成答案的总模型，而不是先给分类目录。
3. 使用“看到……先想……”“先别写……”这种学生脑内操作语言。
4. 每道题必须先写争点一句话。
5. 材料处理要显性呈现：`材料事实 -> 法律语言 -> 得分句`。
6. 教师证据、题号、细则原子放后台，学生前台只看可启动动作。

## 三、共同推荐的 V7 前台主轴

共同主轴应改为“答案产品门”。Claude 提出“六成品 + 边界判断”，GPTPro final 进一步拆成“八个答案产品门”。综合裁定：学生版采用 GPTPro 的八门，因为它把“合同链”“多主体多请求”“创新与 AI 边界”从大类中拆出来，更利于零基础学生第一笔启动。

| 门 2 合同链 | 要约 -> 承诺 -> 成立/有效 -> 履行或违约 | `CC0054`, `CC0244`, `RECOVER_2026_朝阳_期末_18_1` |

## 四、共同推荐的后台生成器

### 1. 三大责任逻辑链

| 逻辑链 | 句子骨架 | 关键证据 |
|---|---|---|
| 违约链 | 合同成立 + 合同有效 + 存在违约行为 -> 违约责任 | `R_CC0244_2026_东城_期末_18_08`, `R_CC0054_2024_石景山_一模_17_M17_01~03` |
| 侵权链 | 权利受保护 + 侵权行为/损害/因果/归责 -> 侵权责任 | `R_CC0244_2026_东城_期末_18_08`, `R_CC0150_2025_朝阳_二模_20_01~07` |
| 不正当竞争链 | 竞争关系/保护对象 + 不正当竞争行为 -> 责任与秩序价值 | `R_CC0206_2025_西城_期末_19_01`, `R_CC0238_2026_东城_二模_19_M17_01` |

### 2. 三档归责卡

| 归责档 | 用在 | 禁错 |
|---|---|---|
| 合同违约档 | 要约、承诺、成立、有效、履行不合约定 | 不能用侵权要件分析违约 |
| 一般过错侵权档 | 销售者、一般损害、人格权/隐私权等 | 不能直接写无过错 |
| 无过错/过错推定档 | 生产者产品责任、饲养动物、建筑物坠落等 | 销售者不是生产者无过错责任；“无过错推定”是错误表述 |

核心证据：`R_CC0244_2026_东城_期末_18_05`、`R_CC0244_2026_东城_期末_18_06`、`R_CC0180_2025_海淀_期末_20_M17_06`、`R_CC0325_2026_石景山_一模_18_02~03`。

## 五、共同要求删除或降级

1. `九个战场`：删除为学生前台主轴，保留为教师后台检索标签。
2. `价值收束/推价值`：降级为尾句生成规则，不做第一入口。
3. `实体责任`：过粗，拆成违约、一般侵权、产品责任、过错推定/无过错。
4. `38 非核心保分索引`：拆成低频规则、综合边界、转出、reference_only 练笔、source-check 保分五类。
5. `CC0150_2025_朝阳_二模_20_12` 及后续周边工作原子：排除出法律框架。
6. `CC0223` 污染材料原子：使用干净题面，不把教师答案当材料。
7. `CC0244` 混合长原子：拆细后才给学生看。

## 六、GPTPro final 对 Claude 方案的增补

GPTPro final 的新增要求：

1. 学生前台用八门而不是六门，尤其单列“合同链”“多主体多请求”“创新与 AI 边界”。
2. 38 道非核心不叫“保分索引”，改成低频、source-check、reference_only、boundary、transfer 五种保险箱。
3. 逐题压测要区分 PASS / PARTIAL / FAIL，并逐题问“学生是否被框架引向正确答案产品”。
4. Codex 需要输出 canonical 证据表、修补 source-card、拆粗 atom、做裸题压测包。

仍需 Codex 裁决：

1. 19 条 `ask_text` 为空行是否在最终逐题宝典前回源补齐；未补齐不得宣称全题满分闭合。
2. `source-check pending` 题只能作为保分索引还是能进入正式逐题样章。
3. V7 最终稿是否采用“前台八门 + 后台责任链 + 证据附录”的三层结构。
