# v8 STEP 04｜8 道金样板题正式选择

本选择基于三份输入共同裁决：

1. 本地 `01_v7_failure_diagnosis.md`：v7.1 判定为 `FAIL_TO_REBUILD`，问题是学生不可启动，不是证据未闭合。
2. GPT-5.5 Pro 输出：`02_model_outputs/gpt55pro_v8_diagnosis_gold_framework.md`。
3. Claude Opus 4.7 输出：`02_model_outputs/claude_opus_v8_diagnosis_gold_framework.md`。

当前语料口径继续锁定为 boundary-patched 53 题：37 PASS，11 PASS_RECOVERED，5 OPEN_OR_REFERENCE。金样板只从 PASS / PASS_RECOVERED 中选；OPEN_OR_REFERENCE 不进入金样板，不支撑核心节点。

## 一、双模型建议对照

| 机制 | GPT 建议 | Claude 建议 | v8 裁决 |
|---|---|---|---|

| 评析行为 | CC0238 | CC0238 | 采用 CC0238。双方一致，是双主体评析标准件。 |
| 权利保护/侵权责任 | CC0305 | CC0305 | 采用 CC0305。虽为 PASS_RECOVERED，但正式细则完整，适合训练“先表态再分层”。 |
| 合同/诚信履约 | CC0143 | CC0054 | 采用 CC0054。CC0054 更纯粹展示合同成立、违约与民法原则，不容易变成消费者三倍赔偿专项。 |
| 知识产权/创新保护 | CC0229 | CC0103 | 采用 CC0103。CC0103 细则直接禁止空泛价值表达，适合训练“价值从规则长出来”。CC0229 保留为后续验证，不做金样板。 |

- CC0373_2026_顺义_一模_18：保留为多主体表态验证题；金样板劳动主样板优先用证据更厚的 CC0025。
- CC0229_2026_东城_一模_18：保留为知识产权/创新保护验证题，但不作为 v8 金样板，避免旧错配表达风险回流。
- CC0340_2026_西城_一模_17：保留为物权绿色发展验证题；制度意义主样板选 CC0002，因为其细则更清楚地区分法理依据与现实意义。
