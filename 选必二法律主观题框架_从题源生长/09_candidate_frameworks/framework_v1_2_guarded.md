# Framework v1.2 Guarded - 选必二法律主观题启动框架

Status: pressure-test framework, not final baodian.

## One-Minute Student Version

1. 先挡边界：国家治理/涉外法治/依法行政，不当选必二核心题写。
2. 再看设问：表格、评析、案例理由、维权建议、意义价值。
3. 案例题先判关系：合同、侵权、相邻、消费欺诈、知识产权/竞争。
4. 满分句按顺序写：定性 -> 规则 -> 材料事实 -> 责任/结果 -> 必要价值收束。
5. 没有重复正式细则支撑的低频题，只进开放容器，不临场造模板。

## Nodes

### FWV1_2_GATE 边界先判

- 节点类别：boundary_gate
- 学生启动：先问这是不是《法律与生活》的私人法律关系、权利义务、责任救济或程序路径；如果是国家治理、涉外法治建设、依法行政，就先挡出去。
- 教师解释：边界 gate 防止 formal 但非选必二核心的法治题污染框架。CC0276 与 RECOVER_2026_西城_二模_18_3 是正式证据，但只能作边界反例。
- 设问触发：运用法治知识谈国家如何建设、国家治理意义、涉外法治建设。
- 材料触发：设问主语是中国/国家/最高法典型案例；答案锚词是涉外法治建设、国家治理能力现代化、依法行政。
- 最小判断：先判断是否离开选必二私法/程序机制，进入必修三或综合法治。
- 满分句生成：本工程不把此类题生成选必二核心满分句；只在边界附录记录。
- 证据：BOUNDARY_FROM_FAIL4_COWORK / CC0276_2026_房山_二模_17|RECOVER_2026_西城_二模_18_3
- 风险：若误收，会把框架必修三化。

### FWV1_2_N02 裁判锚句起手

- 节点类别：core_node
- 学生启动：设问对象是不是法院/仲裁/法官的"裁决/判决/调解结论"；若是，必须先写出"以事实为依据，以法律为准绳"或等价表达"根据《民法典》/相关法律规定"
- 教师解释：凡设问对象是"法院判决/裁判理由/法官调解"且分值 ≥ 5 分的题，"判决以事实为依据、以法律为准绳"是被显式记入评分细则的"必踩点 1 分"。学生只要不点出该锚句（或同义"相关法律规定/民法典规定"）就少 1 分，且该 1 分独立于事实分析分和价值分。
- 设问触发：法院判决/裁判理由/调解结论
- 材料触发：材料中出现"法院判决/判令/裁定/支持/驳回"等司法机关结论性表述
- 最小判断：设问对象是不是法院/仲裁/法官的"裁决/判决/调解结论"；若是，必须先写出"以事实为依据，以法律为准绳"或等价表达"根据《民法典》/相关法律规定"
- 满分句生成：人民法院以事实为根据、以法律为准绳，依据具体法律……
- 证据：CODE_COWORK_003 / CC0002_2024_丰台_一模_17|CC0119_2025_丰台_期末_19|CC0251_2026_丰台_一模_20
- 风险：必修三化风险：GPT: 高。只写维护公平正义、促进社会和谐，不写法律规则和材料触发事实，无法拿满分。; Claude: 中等——若仅停在锚句而无后续法律规则定性会被卡在 1 分上限；法考化风险：GPT: 中。可以写构成要件，但必须服务高中评分点，不能扩展到法考式多层抗辩。; Claude: 低；边界：GPT: 低。三个题均有明确法律与生活或民法劳动法触发。; Claude: 低（属选必二裁判合法性表述）

### FWV1_2_N03 评析先表态

- 节点类别：core_node
- 学生启动：立场是否明确——CC0373_P01"对闫某请求先表态支持"、CC0373_P04"对张某请求先表态不支持"、CC0305_P01"先作出支持原告诉讼请求的结论"、CC0317_01"该合同条款无效"为答案首句、RECOVER_2026_房山_一模_17_1_R01"奖励结论判断"
- 教师解释：评析类（含"判断…是否有效/能否得到法院支持/是否构成××"）题，得分有强制顺序：第一步先做表态（支持/不支持/有效/无效/构成/不构成），第二步分别给出"法律依据 + 事实依据"。表态缺失即使后续论证正确也扣 1–2 分。
- 设问触发：是否支持/是否有效/是否构成/能否得到法院支持
- 材料触发：材料中存在两方诉求或两方行为对立，且题目要求选边
- 最小判断：立场是否明确——CC0373_P01"对闫某请求先表态支持"、CC0373_P04"对张某请求先表态不支持"、CC0305_P01"先作出支持原告诉讼请求的结论"、CC0317_01"该合同条款无效"为答案首句、RECOVER_2026_房山_一模_17_1_R01"奖励结论判断"
- 满分句生成：先表态，再写法律依据，再写材料事实依据，最后复结论。
- 证据：CODE_COWORK_005 / CC0305_2026_海淀_一模_18_3|CC0373_2026_顺义_一模_18
- 风险：必修三化风险：GPT: 高。先写意义、后补规则，容易漏掉判决结论和请求层次。; Claude: 低；法考化风险：GPT: 中。细化请求和主体是必要的，过度展开无关责任形态会偏题。; Claude: 中——CC0373 多主体题易过度推理（如夫妻共同财产侵权法理延伸），需克制于教材范畴；边界：GPT: 低到中。CC0305 设问待 OCR 回补，但评分细则清晰指向选必二民事权利与消费者保护。; Claude: 低

### FWV1_2_N04 私法案例四链

- 节点类别：core_node
- 学生启动：第一步选定权利或行为类型（侵权？违约？不正当竞争？）；第二步把材料事实拆为规则要件；CC0244_08 把这条逻辑写成口令：合同成立+合同有效+违约行为→违约责任；权利受保护+侵权要件齐备→侵权责任；存在竞争关系+不正当竞争行为→构成不正当竞争
- 教师解释：凡设问形如"分析上述案例/分析判决结果并说明理由"且案例核心是侵权或合同纠纷时，评分按"①定性（权利名/行为性质）→②确权（援引法律规则）→③解释（材料事实嵌入规则要件）→④责任（承担方式）"四步分点给分，缺任一步当步即扣。
- 设问触发：分析案例/判决理由；材料是合同、侵权、相邻、消费欺诈、责任承担
- 材料触发：材料具体行为细节（拍摄角度、商品瑕疵、商品被损、用户协议条款）需要被援引到要件位
- 最小判断：第一步选定权利或行为类型（侵权？违约？不正当竞争？）；第二步把材料事实拆为规则要件；CC0244_08 把这条逻辑写成口令：合同成立+合同有效+违约行为→违约责任；权利受保护+侵权要件齐备→侵权责任；存在竞争关系+不正当竞争行为→构成不正当竞争
- 满分句生成：定性权利或行为 -> 援引规则 -> 嵌入材料事实 -> 落责任/结果。
- 证据：CODE_COWORK_004|CODE_COWORK_006 / CC0150_2025_朝阳_二模_20|CC0244_2026_东城_期末_18|RECOVER_2026_西城_二模_18_1|CC0238_2026_东城_二模_19|CC0364_2026_通州_期末_19_1|CC0143_2025_朝阳_一模_19|CC0317_2026_海淀_期末_18|RECOVER_2026_朝阳_期末_18_1
- 风险：必修三化风险：GPT: 中。侵权题价值话可作收束，但责任类型错误会造成核心失分。; Claude: 低；法考化风险：GPT: 高。容易把高中要求扩成法考构成要件清单，或错套过错推定、无过错责任。; Claude: 中——CC0244_08 显式提醒"债权与物权不要混同，这道题不涉及物权变动的分析"；学生过度细化诉权细节属过度法考化；边界：GPT: 低。侵权责任是选必二核心。; Claude: 低

### FWV1_2_N05 知识产权竞争保护链

- 节点类别：core_node
- 学生启动：判断行为属于知识产权侵权、不正当竞争、商业诋毁、恶意诉讼或权利滥用，并识别法院司法手段
- 教师解释：面对知识产权、不正当竞争、恶意诉讼、惩罚性赔偿、调解或驳回等创新保护案例，学生先判断行为性质，再援引规则，再落到司法手段与创新/竞争秩序意义。该 code 只使用裁剪后的六题核心支撑。
- 设问触发：知识产权、不正当竞争、恶意诉讼、惩罚性赔偿、司法保护创新

- 最小判断：判断行为属于知识产权侵权、不正当竞争、商业诋毁、恶意诉讼或权利滥用，并识别法院司法手段

### FWV1_2_N06A 法律边界识别与合规措施

- 满分句生成：风险事实 -> 法律边界 -> 合规措施，一项风险对应一项措施。

### FWV1_2_N07 意义收束三层

- 节点类别：core_node
- 学生启动：题目要求的"对象"是判决/案例/制度→对应的受益主体（个人/司法或行业/社会）；切分三个角度时主语必须不同
- 教师解释：凡设问含"现实意义/社会价值/典型意义/意义"且独立赋分 ≥ 3 分的题，意义层按"个人/当事人合法权益—司法或行业秩序—社会公平/核心价值观"三个角度分点给分，每个角度 1 分，互不重复；只答其中两个角度封顶 2 分。
- 设问触发：现实意义/社会价值/典型意义/意义

### FWV1_2_OPEN 开放容器

- 节点类别：open_container
- 学生启动：能看出法律信号但样本不足或边界偏综合时，不硬套核心节点。
- 教师解释：开放容器用于压力测试和后续样本积累，不允许生成新核心节点。
- 设问触发：AI法律风险/治理，或判决对产业发展和治理影响。
- 材料触发：AI责任边界、开源智能体法律风险、裁判标准、行业有序发展、企业创新活力。
- 最小判断：主语是判决对产业的外部影响，不是当事人责任承担。
- 满分句生成：作为开放示范保留，不支撑核心节点；不得把综合治理或AI开放题升为稳定满分模板。
- 证据：OPEN_CONTAINER_POLICY / RECOVER_2026_西城_二模_18_2|CC0380_2026_顺义_二模_18_2
- 风险：强行升核心会把意义层写成泛营商环境话术。

## Guardrails

- reference_only 题只作弱示范，不支撑节点。
- open-container 题可压测，不生成核心模板。
