## Claude Opus 给出的结构骨架

### 一、设问成品分流器

| 成品 | 设问信号 | 第一笔 |
|---|---|---|
| 判断型 | 能否支持、是否有效、是否构成、判决结果 | XX 请求应/不应支持，因为…… |
| 理由依据型 | 理由、法律依据、为什么这样判 | 依据 XX 规则，本案中 XX，所以…… |
| 意义价值型 | 意义、价值、作用、典型案例 | 本案通过 XX 处理，保护了……规范了…… |
| 评析型 | 评析、谈认识、辨析 | 逐主体/逐案例分别表态 |
| 维权建议型 | 怎么办、维权、需做哪些工作、起诉状 | 围绕 XX 收集证据，提出 XX 请求，走 XX 路径 |
| 表格补全型 | 完成下表、补全笔记、划线处 | 照抄真实列名，逐格判功能，一格一句 |
| 边界判断 | AI/治理/逻辑/哲学混合 | 先判是否纯法律，不是就进容器 |

### 二、三大责任逻辑链

- 违约链：合同成立 + 合同有效 + 存在违约行为 -> 违约责任。
- 侵权链：权利受保护 + 存在侵权行为/满足侵权要件 -> 侵权责任。
- 不正当竞争链：竞争关系 + 不正当竞争行为 -> 不正当竞争。

核心证据：`R_CC0244_2026_东城_期末_18_08`。

### 三、三档归责卡

- 过错责任：一般侵权、销售者产品责任。
- 过错推定：个人信息处理者、动物园动物、林木折断、高空坠物、建筑物坠落。
- 无过错责任：生产者产品责任、环境污染、饲养动物。

硬警告：销售者不是无过错责任；生产者不是一般过错责任；“无过错推定”是错误表述。

### 四、四类容器

- 低频规则容器：三句保底，不升母题。
- 综合边界容器：法律层最低句 + 其他模块另开段。
- 转出容器：逻辑/哲学主任务题只答独立法律小问。
- reference_only 练笔容器：只练表达，不支撑满分闭环。
