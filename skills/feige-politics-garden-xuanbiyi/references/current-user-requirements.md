# 当前用户硬性要求

每次继续做飞哥政治庄园选必一《当代国际政治与经济》主观题文档前，先读本文件，再读 `xuanbiyi-term-protocol.md`。

## 内容生成权威链 V2

- 用户最新要求：主体内容必须由 Codex 与 ClaudeCode 双生产线独立生成厚稿，再交给 GPT Pro 做主融合。不能把 GPT Pro 只当终稿外审，也不能把 ClaudeCode 降级为结构诊断员。
- 每个五题批次必须先形成同一源包，然后分别产出：
  1. `Codex 独立厚稿`：由 Codex 按细则、设问、材料触发、答案句完整展开。
  2. `ClaudeCode 独立厚稿`：由 ClaudeCode 在不读取 Codex 厚稿结论的前提下，用同一源包独立生成完整厚稿。
  3. `差异对照包`：列出二者术语、归桶、材料触发、答案句、证据边界差异。
  4. `GPT Pro 主融合稿`：把 Codex 厚稿和 ClaudeCode 厚稿同时交给 GPT Pro，由 GPT Pro 选择、合并、改写、保留或驳回，并输出可直接进入学生宝典的融合稿。
- Codex 在 GPT Pro 融合之后只做三件事：证据回查、格式落盘、Governor 校验。Codex 不得用自己的偏好覆盖 GPT Pro 融合稿，除非 GPT Pro 的说法与原始细则或材料证据冲突；冲突必须写入裁决日志。
- Claude Opus 4.7 Adaptive 可作为融合后教学化审阅/语言润色/误用风险审查，但不得替代 GPT Pro 的主融合地位。
- 最终审计必须明确记录：Codex 厚稿路径、ClaudeCode 厚稿路径、GPT Pro 融合稿路径、Codex 证据回查记录、Claude Opus 如有的审阅记录。
- 如果没有完成“Codex 独立厚稿 + ClaudeCode 独立厚稿 + GPT Pro 主融合”，不得宣称主体内容由三方共同生成；只能说是 Codex 主体生成、外部模型审阅。

## 固定结构

- 正文主文档必须以六大要素为主线：时代背景、理论、经济全球化、政治多极化、中国、联合国。
- 六大要素下面放术语/框架容器，再把各大考题插入对应术语下。
- 六桶索引必须先做“同类项合并”，再做“表述积累”。同一采分核心只能出现一次，不能因为题面写法不同拆成多条。
- 同类项合并必须看“细则表述是否接近、学生卷面上是否能互相替代、细则是否明示为同一点可选表述”，不能只看抽象本质是否相同。
- 经济全球化桶尤其禁止把 `开放型世界经济`、`开放型经济`、`创新型、开放型世界经济`、`全球经济治理和规则制定`、`多边贸易体系`、`普惠包容的经济全球化`、`贸易自由化` 等只因都属于开放合作就塞入一个核心点；表述不可替代的要拆开，细则明示同点可选的保留原细则组合为独立节点。
- 例如：`时代主题`、`时代主题：和平与发展`、`中国的做法符合和平与发展时代主题` 都归入 `和平与发展仍是时代主题`；`中国的做法符合和平与发展时代主题` 作为该核心点的表述积累。
- 合并后的“核心采分点”必须保留完整核心信息，优先使用最高信息量的细则原词，不能抽象成空泛标签。例如：应写 `推动经济全球化朝着更加开放、包容、普惠、平衡、共赢方向发展`，不能只写 `经济全球化正确方向`。
- 例如：`推动经济全球化朝着更加开放、包容、普惠、平衡、共赢方向发展` 与 `经济全球化方向：普惠、平衡、共赢` 归入同一核心，核心行名保留完整的 `开放、包容、普惠、平衡、共赢`；`合作共赢的新型国际关系` 的不同题面写法也归入同一核心。
- 合并不是删差异：每个核心点下保留 `来源题目 / 材料触发差异 / 可用表述积累 / 答案句变体`，让学生背一个点、会多种写法。
- 正文主文档每个答题点/术语后面用括号标注出现频次；单独的术语频次统计文档继续保留详细出现次数和出现卷子。
- 正文不得退化成只有“材料触发”和“细则/术语”两项。

## 每个正文条目必须保留

1. 术语：评分细则、评标或阅卷总结原词，不写自造概括。
2. 完整设问：写题目要求学生作答的完整设问句。
3. 细则位置：写清套卷、题号、评分细则/评标中的点位、分值或证据层级。
4. 来源：写清年份、区、题号。
5. 材料触发：解释为什么这个材料关系会触发该术语，不能只复述材料。
6. 答案句：给学生可直接落笔的句子。

## 材料触发硬规则

- 材料触发不是摘抄材料，也不是后台制作说明；必须写清题目追问什么、材料形成什么关系、为什么这组关系需要该术语。
- 禁止出现“材料明确写到”“先写”“要写”“落到”“v7漏了”“细则要求”“关键词最稳”等懒表述或复查话。
- 遇到“政治多极化、经济全球化深入发展”这类表述，不能只说材料写了这些词，而要说明它们如何构成外交变化、开放合作或全球治理的背景。
- 材料触发要写真表述：每条都必须能回答“为什么这道题的材料关系触发这个答题点”，不能写成“材料明确写到……”这种废话。

## 答案句硬规则

- 答案句就是考生在答题纸上写的那一条答案，必须按“教材/细则术语 + 本题材料事实 + 因果/作用/结论”写成卷面作答句。
- 答案句禁止写成“要落到……采分点”“并结合材料说明……”这类制作说明，也禁止出现“设问要求”“细则要求”“本题需要”“v7漏了”等后台复查语言。
- 答案句里不要写“材料中”这个笨前缀；直接把材料事实写进答案，例如写“全球治理倡议回应世界乱象和治理赤字”，不要写“材料中全球治理倡议……”。
- 答案句可以短，但必须包含术语本身和本题材料事实，不能只复述标题、不能只喊口号、不能只有“顺应时代潮流/体现大国担当”这类空泛判断。
- 细则出现“总说/分说”“宏观概括/细化解释”“角度+材料”等分层时，必须完整复查总说和各条分说；不能只摘总说术语。
- 如果总说不单独赋分、只作替代或等级条件，要在细则位置里说明证据层级，不能混成逐点术语。

## 版式与复查硬规则

- Word 正文不要再使用术语、完整设问等字段前面的丑矩形/项目符号；使用清爽讲义式标签排版。
- 正文每个答题点/术语后面标注出现频次，例如“（出现N次）”；频次统计文档另行保留详细卷子列表。
- 每次生成六桶索引或学生迁移表前，必须先跑一遍同类项复查：同一教材核心、同一采分功能、同一答案作用的条目必须合并；相似表述进入“表述积累”，不得并列成多个术语行。
- 每次接受新要求后，必须回看本文件，避免“拆了西墙补东墙”：新要求不能覆盖旧要求。
- 若用户指出某套题的细则结构，要把该结构作为硬校验项，不能只修局部表述。

## 重点错例钉死

- 2026-05-17 GPT Pro 源包审阅后的终稿补丁钉死：
  - 理论桶不得再写“合作共赢等理论点”。理论桶只放国家利益、共同利益、综合国力、国际竞争实质等解释性原理；`共同利益是合作的基础` 可解释合作成立。`独立自主和平外交政策`、`独立自主的基本立场`、`和平共处五项原则` 属于中国外交政策链，归入 `中国`；技术、产业链和安全发展语境中的 `独立自主 / 自力更生 / 自主可控 / 发展主动权` 也归入 `中国`，不能放在理论桶。
  - 含 `新型国际关系`、国际秩序、全球治理方向的合作共赢统一归入 `政治多极化`；含贸易、市场、产业链供应链、资源流动、开放平台的合作共赢归入 `经济全球化`；含中国担当、共享机遇、发展中国家民生的合作共赢归入 `中国`。
  - `正确义利观` 只有在细则明确列入国家利益角度、理论逻辑或合作基础可选表达时才放入 `理论`；若设问重心是中国行动、中国担当、发展合作、共享机遇或发展中国家民生，归入 `中国`。
  - `积极参与全球经济治理和规则制定` 必须按功能拆分：企业出海规则环境、国际高标准经贸规则/国际标准/规则话语权、发展中国家代表性和发言权、维护多边贸易体系等不能压成一个宽节点；经济规则和标准话语权优先放 `经济全球化`，只有主问全球治理公平或国际秩序公正合理时才转入 `政治多极化`。
  - `推动经济全球化朝着更加开放、包容、普惠、平衡、共赢方向发展` 不得吞并 `中国市场红利与全球经济包容性增长`、`推进普惠包容的经济全球化/更加开放包容的全球经济格局` 等不同细则落点；但 `推动普惠包容的经济全球化 / 建设开放型世界经济 / 促进贸易自由化` 若细则明确同组“任意1点1分”，保留为独立合并节点。
  - `官方参考答案来源`、`教师版参考答案，未定位正式评分细则`、普通 `参考答案来源` 必须在厚版中标注来源等级。来源等级B/C只作迁移参考，不冒充正式稳定采分点；字段名用 `参考答案术语`，不能写成 `细则术语`。
